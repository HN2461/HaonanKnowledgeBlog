# 第二篇：常见四类图表：折线图、柱状图、饼图、散点图怎么选

> 主人如果已经知道 `ECharts` 是什么了，下一步最重要的就不是背配置，而是先学会 **选图**。  
> 图选对了，数据表达就已经成功了一半。

这一篇我按 **2026-05-03** 查到的 Apache ECharts 官方 `line`、`bar`、`pie`、`scatter`、`dataset` 与交互文档来讲。

目标很直接：

- 先告诉你这四类图分别适合什么数据
- 再告诉你最小配置怎么写
- 最后告诉你项目里最容易犯的选择错误

---

## 一、先记住一个最实用的选图原则

你以后碰到一组数据，先别急着选图。

先问自己这三个问题：

1. 我要看的是趋势，还是对比，还是占比，还是关系
2. 数据是离散分类，还是连续数值，还是两维数值关系
3. 这张图是给自己看，还是给别人看，还是给老板/用户快速扫一眼看

然后再选：

- 看趋势，先想到 `line`
- 看对比，先想到 `bar`
- 看占比，先想到 `pie`
- 看关系，先想到 `scatter`

这四个先掌握，基本就能覆盖大部分后台图表。

---

## 二、折线图 `line` 适合看趋势

官方 `Basic Line` 和 `Smoothed Line` 文档里，最核心的意思就是：

**折线图最适合看连续变化和趋势。**

比如：

- 日活
- 周销售额
- 访问量
- 接口响应时间
- 温度变化

### 最小写法

```js
option = {
  xAxis: {
    data: ['Mon', 'Tue', 'Wed', 'Thu', 'Fri']
  },
  yAxis: {},
  series: [
    {
      type: 'line',
      data: [10, 22, 28, 23, 19]
    }
  ]
}
```

### 常见加法

- `smooth: true`
  - 让线更顺滑

- `areaStyle: {}`
  - 变成面积图

- `stack: 'x'`
  - 变成堆叠折线

- `step: 'start' | 'middle' | 'end'`
  - 变成阶梯线

### 什么时候别乱用

折线图不适合拿来表达纯分类排名。

如果你只是想看“谁高谁低”，  
柱状图通常比折线图更清楚。

---

## 三、柱状图 `bar` 适合看对比

官方 `Basic Bar` 文档说得很直白：

**柱状图就是拿来比较离散数据的。**

比如：

- 各部门业绩
- 各城市订单量
- 各产品销量
- 各渠道注册数

### 最小写法

```js
option = {
  xAxis: {
    data: ['Mon', 'Tue', 'Wed', 'Thu', 'Fri', 'Sat', 'Sun']
  },
  yAxis: {},
  series: [
    {
      type: 'bar',
      data: [23, 24, 18, 25, 27, 28, 25]
    }
  ]
}
```

### 常见加法

- 多系列柱状图
  - 用来比较两组或多组数据

- `stack`
  - 做堆叠柱状图

- `barWidth`
  - 控制柱子宽度

- `barGap`
  - 控制系列之间的间距

- `barCategoryGap`
  - 控制分类之间的间距

- `barMinHeight`
  - 让很小的数据也能看到一点高度

### 什么时候别乱用

如果你要表达的是“时间趋势”，别总想着柱状图。

柱状图更适合：

- 离散比较
- 类别排名
- 多组并列对比

---

## 四、饼图 `pie` 适合看占比

官方 `Basic Pie` 和 `Doughnut` 文档的重点很明确：

**饼图适合表达部分与整体的关系。**

### 最小写法

```js
option = {
  series: [
    {
      type: 'pie',
      data: [
        { value: 335, name: 'Direct' },
        { value: 234, name: 'Union Ad' },
        { value: 1548, name: 'Search' }
      ]
    }
  ]
}
```

### 最常见的两个变体

- 环形图
  - 用 `radius: ['40%', '70%']`

- 南丁格尔玫瑰图
  - 用 `roseType`

### 什么时候别乱用

饼图最容易被误用。

如果分类太多、差值又很接近，  
饼图会变得非常难读。

这时候更适合：

- 柱状图
- 条形图
- 排序后的列表

### 实战建议

饼图最好控制在：

- 分类不多
- 占比差异明显
- 一眼看得出谁大谁小

---

## 五、散点图 `scatter` 适合看关系

官方 `Basic Scatter` 文档里最关键的一句话就是：

**散点图不是拿来排队的，是拿来观察两个数值之间关系的。**

比如：

- 身高和体重
- 年龄和消费额
- 曝光和转化
- 流量和订单

### 最小写法

```js
option = {
  xAxis: {},
  yAxis: {},
  series: [
    {
      type: 'scatter',
      data: [
        [10, 5],
        [0, 8],
        [6, 10],
        [2, 12],
        [8, 9]
      ]
    }
  ]
}
```

### 常见加法

- `symbolSize`
  - 控制点大小

- `visualMap`
  - 让颜色或大小跟某个值联动

- 自定义 `symbol`
  - 圆点、图片、SVG 路径都能用

### 什么时候特别有用

如果你想看：

- 数据是否聚集
- 有没有离群点
- 两个指标之间是否相关

散点图就很合适。

## 六、四类图的核心区别，其实就一张表

| 图表 | 最适合 | 最不适合 |
|------|--------|---------|
| 折线图 | 趋势、变化 | 纯分类排名 |
| 柱状图 | 对比、排名 | 连续趋势表达 |
| 饼图 | 占比、构成 | 分类太多的比较 |
| 散点图 | 关系、分布、离群点 | 单纯展示总量 |

主人以后选图，先拿这张表做第一判断。

---

## 七、业务需求到图表配置速查

如果主人后面是在真实项目里接需求，可以直接先用这张速查表做判断。

| 业务问题 | 优先图表 | 关键配置 | 注意点 |
|----------|----------|----------|--------|
| 最近 30 天销售额涨跌 | 折线图 | `type: 'line'`、`tooltip.trigger: 'axis'`、`dataZoom` | 时间跨度长时别把所有标签都挤出来 |
| 各城市订单量排名 | 柱状图 / 条形图 | `type: 'bar'`、排序后的 `dataset` | 城市名很长时优先横向条形图 |
| 渠道成交占比 | 饼图 / 环形图 | `type: 'pie'`、`radius`、`tooltip.trigger: 'item'` | 分类超过 6-8 个时优先改条形图 |
| 曝光量和转化率关系 | 散点图 | `type: 'scatter'`、二维数值、`visualMap` | 不要拿散点图展示单一指标总量 |
| 订单量和转化率同屏看 | 柱线组合图 | 多个 `series`、双 `yAxis`、`tooltip.trigger: 'axis'` | 两个 y 轴必须写清单位 |
| 很多天的数据需要局部查看 | 折线图 / 柱状图 | `dataZoom: [{ type: 'inside' }, { type: 'slider' }]` | 图太密时先缩放，不要只缩小字体 |

这张表背后的原则很简单：

**先判断用户要回答什么问题，再决定图表类型，不要先看哪个图好看。**

## 八、项目开发时更推荐用 `dataset` 组织四类图

官方示例常用 `series.data`，这很适合入门。

但如果要快速进入项目开发，主人最好早点熟悉这个骨架：

```js
const source = [
  { date: '05-01', visits: 1200, orders: 86, rate: 0.071 },
  { date: '05-02', visits: 1320, orders: 94, rate: 0.071 },
  { date: '05-03', visits: 1180, orders: 76, rate: 0.064 }
]

option = {
  dataset: {
    source
  },
  tooltip: {
    trigger: 'axis'
  },
  legend: {},
  xAxis: {
    type: 'category'
  },
  yAxis: {
    type: 'value'
  },
  series: [
    {
      name: '访问量',
      type: 'bar',
      encode: {
        x: 'date',
        y: 'visits'
      }
    },
    {
      name: '订单数',
      type: 'line',
      encode: {
        x: 'date',
        y: 'orders'
      }
    }
  ]
}
```

这个写法比把 `xAxis.data`、`series[0].data`、`series[1].data` 分开手动 `map` 更适合业务页面。

原因是：

- 接口字段和图表字段关系更清楚
- 新增系列时只改 `series` 和 `encode`
- 后面接筛选、排序、tooltip 字段展示更顺
- 数据源可以被多张图复用

主人以后可以用这个心法：

**demo 可以直接写 `series.data`，业务页优先整理成 `dataset.source + encode`。**

## 九、组合图是后台里很常见的下一步

看完四类基础图，下一步你很快就会碰到“组合图”。

最典型的是：

- 柱状图 + 折线图
- 双 `yAxis`
- 同一张图里既看量，又看率

比如：

- 柱状图看订单量
- 折线图看转化率

这种图适合后台分析页。

### 组合图的关键点

- `series` 里写多个系列
- 每个系列的 `type` 可以不同
- 必要时配两个 `yAxis`
- 注意 `tooltip.trigger = 'axis'`

这一步是后面进阶图表的基础。

## 十、几个很实用的实战判断

### 1. 趋势和对比不要混着想

如果问题是“涨了还是跌了”，就先想折线图。

如果问题是“谁更高”，就先想柱状图。

### 2. 饼图不是默认答案

很多人一想到“占比”就画饼图。

但如果分类太碎，饼图会很累。

这时条形图往往更清楚。

### 3. 散点图是被低估的图

它不如折线图、柱状图常见，  
但在分析“相关性”和“分布”时非常有价值。

### 4. 先把坐标轴想明白

只要是 `line`、`bar`、`scatter` 这类图，  
你就要先想：

- 横轴是什么
- 纵轴是什么
- 有没有多轴
- 有没有缩放

---

## 十一、项目里最容易犯的三个错

### 1. 容器没高度

图不显示，不一定是配置错了，  
有时候只是 `div` 高度没给。

### 2. 一上来就堆太多配置

刚开始不要一张图里把：

- 主题
- 渐变
- 阴影
- 多轴
- 数据缩放
- 复杂 label

全塞进去。

先把基本骨架跑通。

### 3. 图表类型选错

这个最常见。

比如：

- 用饼图展示几十个分类
- 用折线图展示纯排名
- 用柱状图展示特别长的连续时间

都容易让图变乱。

---

## 十二、开发前 30 秒检查清单

每次写 ECharts 页面前，主人可以先快速过一遍：

1. 这张图回答的是趋势、对比、占比还是关系
2. 容器有没有明确高度
3. 需不需要 `dataset + encode`
4. 多系列图有没有 `legend`
5. 多系列趋势或组合图有没有 `tooltip.trigger: 'axis'`
6. 数据点很多时有没有 `dataZoom`
7. 饼图分类是否太多，是否应该改成条形图
8. 双轴图有没有写清单位和格式化
9. 数据来自外部时有没有避免直接拼 HTML
10. 页面销毁或组件卸载时有没有处理实例释放

这 10 条过完，很多项目坑会直接少一半。

---

## 十三、这一篇看完后，主人应该记住什么

就记四句话：

1. 看趋势，优先折线图
2. 看对比，优先柱状图
3. 看占比，优先饼图
4. 看关系，优先散点图
