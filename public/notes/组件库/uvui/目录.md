# uv-ui 目录

## 为什么单独开这个目录

`uv-ui` 和 `Element Plus` 的使用场景不一样。

它更偏：

- `uni-app` 项目
- `H5` + `App` + 小程序多端复用
- 移动端页面、表单、弹窗、导航、上传、请求封装这类高频业务

所以这里单独开一个专题，把 `uv-ui` 这条线系统收起来。

## 基于 2026-05-03 查到的几个关键结论

1. 官方更新日志页顶部版本是 `1.1.20`，日期是 `2024-01-20`，`npm` 公开最新标签也是 `1.1.20`。这更像一套“已经可用、但要重视真机验证”的稳定型框架。
2. 官方安装文档依然优先推荐 `HBuilderX + uni_modules`，如果走 `npm`，就一定要补 `easycom`，`Vue 2 CLI` 场景还要补 `transpileDependencies`。
3. 官方文档把组件分成 `最新组件`、`基础组件`、`表单组件`、`数据组件`、`反馈组件`、`布局组件`、`导航组件` 和 `其他组件` 八条主线，适合按业务场景学习，而不是一个个零散点着看。
4. `uv-row`、`uv-grid`、`uv-tabbar` 这类依赖父子组合和虚拟节点行为的组件，在部分非微信小程序平台会有兼容性差异，不能简单理解成“只要写了 uni-app 就所有小程序都完全一样”。

---

## 建议阅读顺序

1. 先看《第一篇：快速认识 uv-ui：安装、扩展配置、组件地图与项目开发路线》，先把全局地图建立起来。
2. 再看《第二篇：uv-ui 入门安装与小程序配置》，把安装、主题和基础样式配顺。
3. 再看《第三篇：uv-ui 请求封装与使用指南》，把请求层统一收口。
4. 真正做页面时，优先按“表单流 -> 反馈流 -> 列表流 -> 导航流”的顺序补组件。

---

## 项目起步时最值得先掌握的组件

- 表单流
  - `uv-form`、`uv-input`、`uv-textarea`、`uv-picker`、`uv-datetime-picker`、`uv-upload`

- 反馈流
  - `uv-toast`、`uv-notify`、`uv-popup`、`uv-modal`、`uv-loading-page`、`uv-empty`

- 列表流
  - `uv-cell`、`uv-list`、`uv-load-more`、`uv-badge`

- 导航流
  - `uv-navbar`、`uv-tabs`、`uv-tabbar`、`uv-sticky`

- 业务增强
  - `uv-drop-down`、`uv-vtabs`、`uv-waterfall`、`uv-qrcode`

---

## 这个专题后面适合继续补什么

- 表单体系进阶：校验、地址联动、上传回显、复杂筛选
- 列表页组合：搜索、筛选、分页加载、空态、骨架屏
- 微信小程序专题：授权按钮、隐私协议、滚动穿透、缓存与真机排错
- 复杂业务组件：`uv-drop-down`、`uv-vtabs`、`uv-waterfall`、`uv-parse`
