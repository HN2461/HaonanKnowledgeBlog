# MongoDB 详解第六篇：索引优化与事务

> 索引是数据库性能的关键，事务是数据一致性的保障。这篇把这两个重要但容易被忽视的知识点讲透。

---

## 一、为什么需要索引

### 1.1 没有索引时发生了什么

假设 users 集合有 100 万条文档，你执行：

```javascript
await User.findOne({ email: 'zhangsan@example.com' })
```

没有索引时，MongoDB 必须**逐条扫描**所有 100 万条文档，直到找到匹配的那条。这叫**全集合扫描（Collection Scan）**，时间复杂度 O(n)。

有了索引，MongoDB 用 **B-tree（B树）** 数据结构，可以快速定位到目标文档，时间复杂度 O(log n)。

100 万条数据：
- 无索引：可能扫描 100 万次
- 有索引：大约只需 20 次比较（log₂(1,000,000) ≈ 20）

### 1.2 用 explain 查看执行计划

```javascript
// explain('executionStats') 返回查询的执行统计信息
const result = await User.find({ email: 'test@example.com' })
  .explain('executionStats')

console.log(result.executionStats.totalDocsExamined)  // 扫描了多少文档
console.log(result.executionStats.totalKeysExamined)  // 扫描了多少索引键
console.log(result.executionStats.executionTimeMillis) // 执行时间（毫秒）

// 关键指标：
// COLLSCAN（全集合扫描）→ 没有用到索引，需要优化
// IXSCAN（索引扫描）→ 用到了索引，性能好
console.log(result.queryPlanner.winningPlan.stage)
// 'COLLSCAN' 或 'IXSCAN'
```

## 二、索引类型详解

### 2.1 单字段索引

```javascript
const userSchema = new Schema({
  email: {
    type: String,
    unique: true,   // 唯一索引（最常用）
    index: true     // 普通索引
  },
  username: {
    type: String,
    index: true
  }
})

// 也可以在 Schema 外单独定义
userSchema.index({ email: 1 })   // 1 = 升序索引
userSchema.index({ email: -1 })  // -1 = 降序索引（单字段索引方向影响不大）
```

**唯一索引 vs 普通索引**：
- `unique: true`：唯一索引，保证字段值不重复，同时也是索引
- `index: true`：普通索引，只加速查询，不限制重复

### 2.2 复合索引（多字段联合索引）

```javascript
// 复合索引：多个字段组合成一个索引
userSchema.index({ role: 1, createdAt: -1 })
// 这个索引可以加速：
// - 按 role 查询
// - 按 role + createdAt 查询
// - 按 role 查询后按 createdAt 排序

// 另一个例子
articleSchema.index({ author: 1, status: 1, createdAt: -1 })
// 加速：查某个作者的已发布文章，按时间倒序
```

**最左前缀原则**（非常重要）：

```
索引 { a: 1, b: 1, c: 1 } 可以支持以下查询：
✅ 查询 a
✅ 查询 a + b
✅ 查询 a + b + c
✅ 查询 a，排序 b

❌ 只查询 b（没有 a）
❌ 只查询 c（没有 a）
❌ 只查询 b + c（没有 a）

原因：复合索引按字段顺序排列，必须从最左边的字段开始匹配。
```

实际例子：

```javascript
// 索引：{ role: 1, isActive: 1, createdAt: -1 }

// ✅ 能用到索引
User.find({ role: 'user' })
User.find({ role: 'user', isActive: true })
User.find({ role: 'user', isActive: true }).sort({ createdAt: -1 })

// ❌ 用不到这个索引（没有从 role 开始）
User.find({ isActive: true })
User.find({ createdAt: { $gte: someDate } })
```

### 2.3 文本索引（全文搜索）

```javascript
// 文本索引：支持全文搜索
articleSchema.index({ title: 'text', content: 'text' })

// 使用文本搜索
const articles = await Article.find({
  $text: { $search: 'MongoDB 教程' }
})

// 按相关度排序
const articles = await Article.find(
  { $text: { $search: 'MongoDB 教程' } },
  { score: { $meta: 'textScore' } }  // 投影里加上相关度分数
).sort({ score: { $meta: 'textScore' } })  // 按相关度排序

// 注意：
// - 一个集合只能有一个文本索引
// - 文本索引不支持中文分词（中文搜索需要 Atlas Search 或 Elasticsearch）
// - 搜索词之间是 OR 关系，加 - 前缀表示排除
// $text: { $search: 'MongoDB -教程' }  → 包含 MongoDB 但不包含 教程
```

### 2.4 TTL 索引（自动过期删除）

TTL（Time To Live）索引可以让文档在指定时间后自动删除，非常适合：
- 验证码（5分钟过期）
- 临时 token
- 会话数据
- 日志（保留30天）

```javascript
// 方式一：expireAfterSeconds — 文档创建后多少秒删除
const verificationSchema = new Schema({
  email: String,
  code: String,
  createdAt: { type: Date, default: Date.now }
})

// createdAt 字段上建 TTL 索引，300秒（5分钟）后自动删除
verificationSchema.index({ createdAt: 1 }, { expireAfterSeconds: 300 })

// 方式二：expireAt 字段 — 指定具体的过期时间
const sessionSchema = new Schema({
  userId: Schema.Types.ObjectId,
  token: String,
  expireAt: Date  // 存储具体的过期时间
})

// expireAfterSeconds: 0 — 到达 expireAt 时间时删除
sessionSchema.index({ expireAt: 1 }, { expireAfterSeconds: 0 })

// 创建时指定过期时间
await Session.create({
  userId: user._id,
  token: generateToken(),
  expireAt: new Date(Date.now() + 7 * 24 * 60 * 60 * 1000)  // 7天后过期
})
```

> **注意**：TTL 索引的删除不是精确到秒的，MongoDB 后台每 60 秒扫描一次，所以实际删除时间可能比设定时间晚最多 60 秒。

### 2.5 稀疏索引

```javascript
// 稀疏索引：只索引有该字段的文档，没有该字段的文档不进入索引
// 适合：可选字段，大量文档没有该字段的情况

userSchema.index({ phone: 1 }, { sparse: true })
// 没有 phone 字段的文档不会出现在这个索引里
// 节省索引空间，避免大量 null 值占用索引
```

### 2.6 部分索引

```javascript
// 部分索引：只索引满足条件的文档
// 比稀疏索引更灵活

// 只索引活跃用户的 email（减少索引大小）
userSchema.index(
  { email: 1 },
  { partialFilterExpression: { isActive: true } }
)

// 只索引已发布的文章
articleSchema.index(
  { createdAt: -1 },
  { partialFilterExpression: { status: 'published' } }
)
```

## 三、索引使用原则

### 3.1 什么时候建索引

```
✅ 适合建索引：
- 经常作为查询条件的字段（WHERE 子句）
- 经常用于排序的字段（ORDER BY）
- 外键字段（关联查询的 localField）
- 唯一性约束字段（如 email、username）
- 数据量大（超过 1 万条）且查询频繁的字段

❌ 不适合建索引：
- 很少用于查询的字段
- 数据量很小的集合（全扫描反而更快）
- 写多读少的字段（索引会降低写入性能）
- 基数很低的字段（如 Boolean，只有 true/false，索引效果差）
- 频繁更新的字段（每次更新都要维护索引）
```

### 3.2 索引对写入性能的影响

```
每个索引都有代价：
- 插入文档时，需要更新所有相关索引
- 更新文档时，如果修改了索引字段，需要更新索引
- 删除文档时，需要从索引中移除

一般原则：
- 读多写少的集合：可以建较多索引
- 写多读少的集合：尽量少建索引
- 生产环境不要超过 5-6 个索引（除非有充分理由）
```

### 3.3 查看和管理索引

```javascript
// 在 mongosh 里查看集合的所有索引
db.users.getIndexes()

// 删除索引
db.users.dropIndex('email_1')  // 按索引名删除
db.users.dropIndex({ email: 1 })  // 按字段删除

// 在 Mongoose 里，Model 同步索引
// Mongoose 连接后会自动调用 ensureIndexes，确保 Schema 里定义的索引存在
// 生产环境建议关闭自动同步，手动管理索引
mongoose.set('autoIndex', false)  // 全局关闭
// 或在 Schema 选项里：
const schema = new Schema({ ... }, { autoIndex: false })
```

## 四、事务（Transactions）

### 4.1 为什么需要事务

考虑这个场景：用户 A 向用户 B 转账 100 元。

```javascript
// 没有事务的写法（危险！）
await User.findByIdAndUpdate(fromId, { $inc: { balance: -100 } })
// 如果这里服务器崩溃了...
await User.findByIdAndUpdate(toId, { $inc: { balance: 100 } })
// A 的钱扣了，B 的钱没加上，100 元凭空消失！
```

事务保证：**要么两个操作都成功，要么都不执行**，不会出现中间状态。

### 4.2 MongoDB 事务的前提条件

```
MongoDB 4.0+ 支持多文档事务，但有要求：
- 必须是副本集（Replica Set）或分片集群（Sharded Cluster）
- 单机 mongod 不支持事务

本地开发时，可以把单机 mongod 转换为单节点副本集：
mongod --replSet rs0

然后在 mongosh 里初始化：
rs.initiate()
```

> **MongoDB Atlas（云数据库）默认就是副本集，直接支持事务。**

### 4.3 事务基本写法

```javascript
const mongoose = require('mongoose')

async function transfer(fromId, toId, amount) {
  // 第一步：开启 session
  const session = await mongoose.startSession()

  try {
    // 第二步：开启事务
    session.startTransaction()

    // 第三步：在事务内执行操作（每个操作都要传 session）
    const fromUser = await User.findByIdAndUpdate(
      fromId,
      { $inc: { balance: -amount } },
      { new: true, session }  // ← 必须传 session
    )

    if (!fromUser) throw new Error('转出用户不存在')
    if (fromUser.balance < 0) throw new Error('余额不足')

    const toUser = await User.findByIdAndUpdate(
      toId,
      { $inc: { balance: amount } },
      { new: true, session }  // ← 必须传 session
    )

    if (!toUser) throw new Error('转入用户不存在')

    // 记录转账流水
    await Transaction.create([{
      from: fromId,
      to: toId,
      amount,
      type: 'transfer',
      createdAt: new Date()
    }], { session })  // ← create 传 session 要用数组形式

    // 第四步：提交事务（所有操作生效）
    await session.commitTransaction()
    console.log('转账成功')
    return { success: true }

  } catch (err) {
    // 第五步：出错时回滚（所有操作撤销）
    await session.abortTransaction()
    console.error('转账失败，已回滚:', err.message)
    throw err

  } finally {
    // 第六步：结束 session（无论成功失败都要执行）
    session.endSession()
  }
}
```

### 4.4 withTransaction — 更简洁的写法

Mongoose 提供了 `withTransaction` 方法，自动处理提交和回滚：

```javascript
async function transfer(fromId, toId, amount) {
  const session = await mongoose.startSession()

  try {
    // withTransaction 自动处理：
    // - 开启事务
    // - 成功时提交
    // - 失败时回滚
    // - 遇到可重试错误时自动重试
    const result = await session.withTransaction(async () => {
      const fromUser = await User.findByIdAndUpdate(
        fromId,
        { $inc: { balance: -amount } },
        { new: true, session }
      )

      if (!fromUser || fromUser.balance < 0) {
        throw new Error('余额不足')
      }

      await User.findByIdAndUpdate(
        toId,
        { $inc: { balance: amount } },
        { session }
      )

      await Transaction.create([{
        from: fromId,
        to: toId,
        amount
      }], { session })

      return { success: true }
    })

    return result

  } finally {
    session.endSession()
  }
}
```

### 4.5 实战：库存扣减（防超卖）

```javascript
// 下单时扣减库存，防止超卖
async function createOrder(userId, productId, quantity) {
  const session = await mongoose.startSession()

  try {
    await session.withTransaction(async () => {
      // 查询商品（加 session，在事务内读取）
      const product = await Product.findById(productId).session(session)

      if (!product) throw new Error('商品不存在')
      if (product.stock < quantity) throw new Error('库存不足')

      // 扣减库存（用 $inc 原子操作）
      await Product.findByIdAndUpdate(
        productId,
        { $inc: { stock: -quantity } },
        { session }
      )

      // 创建订单
      const order = await Order.create([{
        user: userId,
        product: productId,
        quantity,
        price: product.price * quantity,
        status: 'pending'
      }], { session })

      return order[0]
    })

  } finally {
    session.endSession()
  }
}
```

### 4.6 事务注意事项

```
⚠️ 使用事务时要注意：

1. 事务有时间限制
   默认最长 60 秒，超时自动回滚
   可以通过 maxTimeMS 调整

2. 事务内的操作会锁定文档
   其他操作需要等待事务完成
   事务越短越好，不要在事务内做耗时操作（如发邮件）

3. create 在事务内要用数组形式
   await Model.create([doc], { session })  ← 注意是数组
   await Model.create(doc, { session })    ← 这样不行！

4. 事务不支持某些操作
   - 不能创建集合（collection）
   - 不能创建索引
   - 不能使用 $out 聚合阶段

5. 本地开发需要副本集
   单机 mongod 不支持事务
   可以用 Docker 启动副本集，或者用 MongoDB Atlas
```

## 五、小结

**索引**：

| 索引类型 | 用法 | 适用场景 |
|----------|------|----------|
| 单字段索引 | `index: true` | 常用查询字段 |
| 唯一索引 | `unique: true` | email、username 等唯一字段 |
| 复合索引 | `schema.index({ a: 1, b: -1 })` | 多字段组合查询 |
| 文本索引 | `schema.index({ title: 'text' })` | 全文搜索（英文） |
| TTL 索引 | `{ expireAfterSeconds: n }` | 验证码、session、日志 |
| 稀疏索引 | `{ sparse: true }` | 可选字段，大量文档无此字段 |

**事务**：

| 步骤 | 代码 |
|------|------|
| 开启 session | `const session = await mongoose.startSession()` |
| 开启事务 | `session.startTransaction()` |
| 操作时传 session | `{ session }` 选项 |
| 提交 | `await session.commitTransaction()` |
| 回滚 | `await session.abortTransaction()` |
| 结束 session | `session.endSession()` |
| 简洁写法 | `session.withTransaction(async () => { ... })` |
