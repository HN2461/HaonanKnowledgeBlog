# MongoDB 详解第五篇：关联查询与聚合管道

> 这篇讲两个进阶但非常实用的功能：populate（关联查询）和 aggregate（聚合管道）。前者解决"查文章时顺便把作者信息带出来"的问题，后者解决"统计、分组、复杂数据转换"的问题。

---

## 一、关联查询（populate）

### 1.1 为什么需要关联

MongoDB 是文档型数据库，理论上可以把所有数据嵌套在一个文档里。但实际项目中，有些数据需要独立存储、被多处引用，比如：

- 文章有作者，作者信息存在 users 集合
- 订单有商品，商品信息存在 products 集合
- 评论有用户，用户信息存在 users 集合

这时候就需要"关联"——在一个集合里存另一个集合文档的 `_id`，查询时再把关联文档填充进来。

### 1.2 定义关联关系

```javascript
// src/models/Article.js
const { Schema, model } = require('mongoose')

const articleSchema = new Schema({
  title: {
    type: String,
    required: true
  },
  content: String,
  // 关联字段：存储 User 的 _id
  author: {
    type: Schema.Types.ObjectId,
    ref: 'User',      // ref 指向关联的 Model 名称，populate 时用到
    required: true
  },
  // 多个标签（关联 Tag 集合）
  tags: [{
    type: Schema.Types.ObjectId,
    ref: 'Tag'
  }],
  views: { type: Number, default: 0 },
  status: {
    type: String,
    enum: ['draft', 'published'],
    default: 'draft'
  }
}, { timestamps: true, versionKey: false })

const Article = model('Article', articleSchema)
module.exports = Article
```

### 1.3 基础 populate

```javascript
const Article = require('./models/Article')

// 不用 populate：author 字段只是一个 ObjectId 字符串
const article = await Article.findById(id)
console.log(article.author)
// ObjectId('6642a1b2c3d4e5f678901234')  ← 只有 ID，没有用户信息

// 用 populate：自动把 author 的 ObjectId 替换成完整的 User 文档
const article = await Article.findById(id).populate('author')
console.log(article.author)
// {
//   _id: ObjectId('6642a1b2c3d4e5f678901234'),
//   username: '张三',
//   email: 'zhangsan@example.com',
//   role: 'user',
//   ...
// }
```

### 1.4 populate 选择字段

通常不需要把关联文档的所有字段都带出来，用第二个参数指定：

```javascript
// 只填充 username 和 email（_id 默认包含）
const article = await Article.findById(id)
  .populate('author', 'username email')

// 排除某些字段
const article = await Article.findById(id)
  .populate('author', '-password -isDeleted')

// 同时 populate 多个字段
const article = await Article.findById(id)
  .populate('author', 'username email')
  .populate('tags', 'name color')
```

### 1.5 populate 配置对象写法

需要更多控制时，用对象写法：

```javascript
const article = await Article.findById(id).populate({
  path: 'author',           // 要填充的字段
  select: 'username email', // 选择字段
  match: { isActive: true }, // 只填充满足条件的文档（不满足则为 null）
  options: {
    sort: { username: 1 }   // 填充结果排序（数组关联时有用）
  }
})
```

### 1.6 多级 populate（嵌套关联）

```javascript
// 假设 User 还关联了 Department
const article = await Article.findById(id).populate({
  path: 'author',
  select: 'username department',
  populate: {               // 嵌套 populate
    path: 'department',
    select: 'name'
  }
})

console.log(article.author.department.name)  // 部门名称
```

### 1.7 在 find 结果上 populate

```javascript
// 查询列表时 populate
const articles = await Article.find({ status: 'published' })
  .populate('author', 'username avatar')
  .sort({ createdAt: -1 })
  .limit(10)
  .lean()
```

### 1.8 populate 的性能注意事项

```
populate 的原理：
1. 先查询主集合（如 articles）
2. 收集所有关联字段的 ID
3. 用 $in 查询关联集合（如 users）
4. 把结果填充回去

所以 populate 实际上是两次查询，不是 JOIN。
对于大量数据，要注意：
- 不要在循环里 populate（N+1 问题）
- 列表页只 populate 必要字段
- 考虑用 aggregate + $lookup 替代（一次查询）
```

## 二、聚合管道（Aggregation Pipeline）

### 2.1 什么是聚合管道

聚合管道是 MongoDB 最强大的功能，可以对数据进行复杂的统计、转换和分析。

**类比理解**：就像工厂流水线，原材料（文档）依次经过多道工序（阶段），最终产出成品（结果）。

```
原始文档集合
    ↓
  $match（过滤，类似 WHERE）
    ↓
  $group（分组统计，类似 GROUP BY）
    ↓
  $sort（排序，类似 ORDER BY）
    ↓
  $project（选择/计算字段，类似 SELECT）
    ↓
  最终结果
```

### 2.2 基础语法

```javascript
// Model.aggregate(pipeline) → Promise<Array>
// pipeline 是一个数组，每个元素是一个阶段对象

const result = await User.aggregate([
  { $match: { isActive: true } },   // 阶段1：过滤
  { $group: { _id: '$role', count: { $sum: 1 } } },  // 阶段2：分组
  { $sort: { count: -1 } }          // 阶段3：排序
])
```

## 三、核心聚合阶段详解

### 3.1 $match — 过滤文档

```javascript
// 类似 find() 的查询条件，放在管道最前面可以减少后续处理的数据量
{ $match: { isActive: true, age: { $gte: 18 } } }

// 支持所有查询操作符
{ $match: {
  role: { $in: ['admin', 'moderator'] },
  createdAt: { $gte: new Date('2026-01-01') }
} }
```

> **性能提示**：`$match` 尽量放在管道最前面，越早过滤数据，后续阶段处理的数据量越少。

### 3.2 $group — 分组统计

```javascript
// $group 是聚合管道的核心
// _id 是分组依据，null 表示不分组（对所有文档统计）

// 按角色分组，统计每组数量和平均年龄
{ $group: {
  _id: '$role',                    // 按 role 字段分组
  count: { $sum: 1 },              // 每组文档数
  avgAge: { $avg: '$age' },        // 平均年龄
  maxAge: { $max: '$age' },        // 最大年龄
  minAge: { $min: '$age' },        // 最小年龄
  totalScore: { $sum: '$score' },  // 分数总和
  names: { $push: '$username' }    // 把每组的 username 收集成数组
} }

// 不分组，统计所有文档
{ $group: {
  _id: null,
  total: { $sum: 1 },
  avgAge: { $avg: '$age' }
} }

// 按多个字段组合分组
{ $group: {
  _id: { role: '$role', isActive: '$isActive' },
  count: { $sum: 1 }
} }
```

**$group 常用累加器**：

| 累加器 | 说明 | 示例 |
|--------|------|------|
| `$sum` | 求和（传 1 则计数） | `{ $sum: '$price' }` |
| `$avg` | 平均值 | `{ $avg: '$age' }` |
| `$max` | 最大值 | `{ $max: '$score' }` |
| `$min` | 最小值 | `{ $min: '$score' }` |
| `$push` | 收集到数组（允许重复） | `{ $push: '$name' }` |
| `$addToSet` | 收集到数组（去重） | `{ $addToSet: '$tag' }` |
| `$first` | 每组第一个值 | `{ $first: '$name' }` |
| `$last` | 每组最后一个值 | `{ $last: '$name' }` |
| `$count` | 计数（MongoDB 5.0+） | `{ $count: {} }` |

### 3.3 $sort — 排序

```javascript
// 1 升序，-1 降序
{ $sort: { count: -1, avgAge: 1 } }
```

### 3.4 $project — 选择/计算字段

```javascript
// 类似 SELECT，控制输出哪些字段，以及计算新字段
{ $project: {
  username: 1,                    // 保留
  email: 1,                       // 保留
  _id: 0,                         // 排除 _id
  password: 0,                    // 排除 password

  // 计算新字段
  fullName: { $concat: ['$firstName', ' ', '$lastName'] },

  // 条件字段
  ageGroup: {
    $cond: {
      if: { $gte: ['$age', 18] },
      then: '成年',
      else: '未成年'
    }
  },

  // 字符串操作
  emailDomain: {
    $arrayElemAt: [{ $split: ['$email', '@'] }, 1]
  },

  // 日期格式化
  createdYear: { $year: '$createdAt' },
  createdMonth: { $month: '$createdAt' }
} }
```

### 3.5 $limit 和 $skip

```javascript
{ $limit: 10 }   // 只取前 10 条
{ $skip: 20 }    // 跳过前 20 条

// 分页：第 2 页，每页 10 条
[
  { $skip: 10 },
  { $limit: 10 }
]
```

### 3.6 $count — 统计数量

```javascript
// 统计管道到此阶段的文档数量
{ $count: 'total' }
// 输出：[{ total: 150 }]
```

### 3.7 $addFields — 添加新字段

```javascript
// 在原有字段基础上添加新字段（不影响原有字段）
{ $addFields: {
  fullName: { $concat: ['$firstName', ' ', '$lastName'] },
  isAdult: { $gte: ['$age', 18] },
  // 计算文章总字数（假设 content 是字符串）
  wordCount: { $strLenCP: '$content' }
} }
```

## 四、$lookup — 关联查询

`$lookup` 是聚合管道里的关联查询，类似 SQL 的 LEFT JOIN。

### 4.1 基础用法

```javascript
// 查询文章，同时关联作者信息
await Article.aggregate([
  {
    $lookup: {
      from: 'users',          // 关联的集合名（注意：是集合名，不是 Model 名！）
                              // 'User' Model → 集合名 'users'
      localField: 'author',   // 本集合的字段（存的是 ObjectId）
      foreignField: '_id',    // 关联集合的字段
      as: 'authorInfo'        // 结果存放的字段名（是一个数组！）
    }
  }
])

// 结果：
// {
//   title: '文章标题',
//   author: ObjectId('...'),
//   authorInfo: [{ _id: ..., username: '张三', ... }]  ← 数组
// }
```

### 4.2 $unwind — 展开数组

`$lookup` 的结果是数组，一对一关联时用 `$unwind` 展开：

```javascript
await Article.aggregate([
  {
    $lookup: {
      from: 'users',
      localField: 'author',
      foreignField: '_id',
      as: 'authorInfo'
    }
  },
  // 展开 authorInfo 数组（一对一关联）
  { $unwind: '$authorInfo' },
  // 现在 authorInfo 是对象而不是数组了
  {
    $project: {
      title: 1,
      'authorInfo.username': 1,
      'authorInfo.email': 1
    }
  }
])
```

`$unwind` 的另一个用途——展开文档自身的数组字段：

```javascript
// 文档：{ name: '张三', tags: ['js', 'node', 'vue'] }
// $unwind 后变成三条文档：
// { name: '张三', tags: 'js' }
// { name: '张三', tags: 'node' }
// { name: '张三', tags: 'vue' }

// 统计所有标签的使用频率
await Article.aggregate([
  { $unwind: '$tags' },
  { $group: { _id: '$tags', count: { $sum: 1 } } },
  { $sort: { count: -1 } },
  { $limit: 10 }
])
```

### 4.3 $lookup 高级用法（pipeline）

```javascript
// 关联时加条件过滤（MongoDB 3.6+）
await Article.aggregate([
  {
    $lookup: {
      from: 'users',
      let: { authorId: '$author' },  // 定义变量
      pipeline: [
        {
          $match: {
            $expr: { $eq: ['$_id', '$$authorId'] },  // $$变量名 引用 let 里的变量
            isActive: true  // 只关联活跃用户
          }
        },
        { $project: { username: 1, email: 1 } }  // 只取需要的字段
      ],
      as: 'authorInfo'
    }
  }
])
```

## 五、$facet — 多维度统计

`$facet` 可以在一次查询里同时执行多个子管道，返回多个统计结果：

```javascript
// 一次查询同时获取：数据列表 + 总数 + 分类统计
const [result] = await Article.aggregate([
  { $match: { status: 'published' } },  // 先过滤
  {
    $facet: {
      // 子管道1：分页数据
      data: [
        { $sort: { createdAt: -1 } },
        { $skip: 0 },
        { $limit: 10 },
        {
          $lookup: {
            from: 'users',
            localField: 'author',
            foreignField: '_id',
            as: 'authorInfo'
          }
        },
        { $unwind: '$authorInfo' }
      ],

      // 子管道2：总数
      total: [
        { $count: 'count' }
      ],

      // 子管道3：按状态分组统计
      byStatus: [
        { $group: { _id: '$status', count: { $sum: 1 } } }
      ]
    }
  }
])

const { data, total, byStatus } = result
const totalCount = total[0]?.count || 0
```

## 六、实战：常见聚合场景

### 6.1 分页 + 总数（一次查询）

```javascript
async function getArticles(page = 1, limit = 10, filter = {}) {
  const [result] = await Article.aggregate([
    { $match: { status: 'published', ...filter } },
    {
      $facet: {
        data: [
          { $sort: { createdAt: -1 } },
          { $skip: (page - 1) * limit },
          { $limit: limit },
          {
            $lookup: {
              from: 'users',
              localField: 'author',
              foreignField: '_id',
              as: 'author'
            }
          },
          { $unwind: { path: '$author', preserveNullAndEmptyArrays: true } },
          {
            $project: {
              title: 1,
              views: 1,
              createdAt: 1,
              'author.username': 1,
              'author.avatar': 1
            }
          }
        ],
        total: [{ $count: 'count' }]
      }
    }
  ])

  return {
    list: result.data,
    pagination: {
      page,
      limit,
      total: result.total[0]?.count || 0,
      pages: Math.ceil((result.total[0]?.count || 0) / limit)
    }
  }
}
```

### 6.2 数据统计报表

```javascript
// 统计每天的新增用户数（最近 30 天）
const stats = await User.aggregate([
  {
    $match: {
      createdAt: {
        $gte: new Date(Date.now() - 30 * 24 * 60 * 60 * 1000)
      }
    }
  },
  {
    $group: {
      _id: {
        year: { $year: '$createdAt' },
        month: { $month: '$createdAt' },
        day: { $dayOfMonth: '$createdAt' }
      },
      count: { $sum: 1 }
    }
  },
  { $sort: { '_id.year': 1, '_id.month': 1, '_id.day': 1 } }
])
```

### 6.3 标签热度排行

```javascript
// 统计所有文章中标签的使用频率
const tagStats = await Article.aggregate([
  { $match: { status: 'published' } },
  { $unwind: '$tags' },
  {
    $group: {
      _id: '$tags',
      count: { $sum: 1 },
      articles: { $push: '$title' }  // 收集使用该标签的文章标题
    }
  },
  { $sort: { count: -1 } },
  { $limit: 20 },
  {
    $project: {
      tag: '$_id',
      count: 1,
      _id: 0
    }
  }
])
```

### 6.4 用户活跃度分析

```javascript
// 统计每个用户的文章数、总阅读量、平均阅读量
const userStats = await Article.aggregate([
  { $match: { status: 'published' } },
  {
    $group: {
      _id: '$author',
      articleCount: { $sum: 1 },
      totalViews: { $sum: '$views' },
      avgViews: { $avg: '$views' }
    }
  },
  {
    $lookup: {
      from: 'users',
      localField: '_id',
      foreignField: '_id',
      as: 'userInfo'
    }
  },
  { $unwind: '$userInfo' },
  {
    $project: {
      username: '$userInfo.username',
      articleCount: 1,
      totalViews: 1,
      avgViews: { $round: ['$avgViews', 0] }  // 四舍五入
    }
  },
  { $sort: { totalViews: -1 } },
  { $limit: 10 }
])
```

## 七、populate vs aggregate $lookup 怎么选

| 场景 | 推荐方案 | 原因 |
|------|----------|------|
| 简单关联，查单条文档 | populate | 代码简洁，够用 |
| 列表页关联查询 | populate | 代码简洁，Mongoose 自动优化 |
| 需要同时统计/分组 | aggregate + $lookup | 一次查询，灵活 |
| 关联时需要过滤条件 | aggregate + $lookup pipeline | populate 的 match 有限制 |
| 性能要求极高 | aggregate + $lookup | 一次数据库往返 |
| 多级嵌套关联 | populate（嵌套） | 代码更清晰 |

## 八、小结

| 知识点 | 核心要点 |
|--------|----------|
| populate 定义 | 字段加 `ref: 'ModelName'`，查询时 `.populate('字段名')` |
| populate 选字段 | `.populate('author', 'username email')` |
| populate 配置对象 | `{ path, select, match, options }` |
| aggregate 语法 | `Model.aggregate([阶段1, 阶段2, ...])` |
| $match | 过滤，放管道最前面 |
| $group | 分组统计，`_id` 是分组依据 |
| $sort | 排序，1 升序 -1 降序 |
| $project | 选择/计算字段 |
| $lookup | 关联查询，结果是数组 |
| $unwind | 展开数组字段 |
| $facet | 一次查询多个子管道 |
