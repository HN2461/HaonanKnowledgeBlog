# Node.js 从零到实战系列

> 更新时间：2026-04-18

本系列共 8 篇，基于 B 站课程《Node.js零基础教程，nodejs新手到高手》（BV1gM411W7ex）与配套课件整理，面向**有前端基础、初次接触后端**的开发者，覆盖 Node.js 后端开发的完整学习路径。

> **官方核对说明**：
> 这套文章已按 2026-04-18 的官方资料做过一次版本核对。学习时优先以 **Node.js 24 LTS + Express 5 + MongoDB 社区版 + Mongoose 9** 这一组合为准；如果你为了兼容老课件暂时停留在 Node 20/22，大多数示例仍然能跑，但请以文中的“版本提示”和“废弃提醒”为准。

## 前端转后端学习里程碑

1. **先把运行环境跑通**：第一篇和第五篇一起看，先装好 Node.js、nvm，理解 `process`、模块系统和 npm 基本命令。
2. **补齐服务端文件与路径意识**：第二篇把 `fs`、`path`、Stream 吃透，这是后端处理日志、上传、配置文件的基础。
3. **建立请求-响应模型**：第三篇用原生 `http` 自己写服务端，真正理解浏览器请求到了服务器后发生了什么。
4. **开始写可维护的接口项目**：第四篇和第六篇连着看，先会拆模块，再上 Express 的路由、中间件和错误处理。
5. **把数据持久化接上**：第七篇用 MongoDB + Mongoose 建模、校验、CRUD、聚合，开始具备完整后端能力。
6. **把登录态和安全补上**：第八篇把 Cookie、Session、JWT、密码哈希、限流和参数校验补齐，形成最小可上线后端闭环。

## 建议的练手项目顺序

1. `hello-node`：输出环境信息，练习命令行运行和调试。
2. `file-tool`：做一个本地文件扫描/整理脚本，练习 `fs`、`path`、Stream。
3. `mini-http-server`：不用框架，手写 GET/POST 接口。
4. `express-notes-api`：Express + 路由拆分 + 统一错误处理。
5. `auth-api`：MongoDB + Mongoose + JWT + bcrypt，完成注册、登录、鉴权。

## 覆盖主题速览

**运行环境与基础**

- Node.js 架构：V8 引擎 + libuv + 内置模块
- Buffer：二进制数据处理，与字符串互转
- 全局对象：`__dirname`、`__filename`、`process`、`global`
- 编码注意事项：UTF-8、CRLF/LF、BOM 处理

**核心内置模块**

- `fs`：同步/异步读写、流式读写、目录操作、`fs.promises`
- `path`：`join`、`resolve`、`dirname`、`basename`、`extname`
- `http`：`createServer`、请求对象、响应对象、GET/POST 处理

**模块化与包管理**

- CommonJS：`require` / `module.exports` / `exports`
- ES6 模块：`import` / `export`，`.mjs` 与 `"type": "module"`
- npm：`install`、`uninstall`、`scripts`、`semver` 版本规则
- nvm：多 Node.js 版本管理，`.nvmrc` 锁定版本

**Express 框架**

- 路由：GET/POST/PUT/DELETE，路由参数，Router 模块化
- 中间件：全局/路由级/错误处理，`next()` 调用链
- 静态资源：`express.static`，注意事项
- 模板引擎：ejs 语法，`res.render`，`express-generator`

**数据库**

- MongoDB：文档型数据库，集合与文档概念
- Mongoose：Schema 定义，Model CRUD，数据校验
- 聚合查询：`populate`，`aggregate`

**接口与会话**

- RESTful 设计规范，状态码语义
- apipost 接口测试工具
- Cookie：设置、读取、删除，`httpOnly`、`maxAge`
- Session：`express-session`，与 Cookie 的区别
- JWT：无状态认证，适用场景对比
