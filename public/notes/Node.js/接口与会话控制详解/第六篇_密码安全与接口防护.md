# 接口与会话控制详解第六篇：密码安全与接口防护

> 功能做完了，安全不能忽视。这篇把密码存储、参数校验、限流、安全响应头这几个最基础的安全措施讲透。

## 一、密码安全：bcrypt

### 1.1 为什么不能明文存密码

```
❌ 明文存储：password = '123456'
  数据库泄露 → 所有用户密码直接暴露

❌ MD5/SHA1 哈希：password = md5('123456')
  彩虹表攻击：攻击者预先计算了大量常见密码的哈希值
  md5('123456') = 'e10adc3949ba59abbe56e057f20f883e'
  查彩虹表 → 立刻知道原始密码是 '123456'

✅ bcrypt 哈希：
  每次哈希结果不同（加了随机 salt）
  计算慢（故意的，防暴力破解）
  无法反向解密
```

### 1.2 bcrypt 的工作原理

```
bcrypt 做了两件事：

1. 加盐（Salt）：
   每次哈希时生成一个随机字符串（salt）
   把 salt 和密码混合后再哈希
   所以同一个密码每次哈希结果都不同
   
   '123456' → $2b$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhWy
   '123456' → $2b$10$Ks8Ld3Ld3Ld3Ld3Ld3LdOeIjZAgcfl7p92ldGxad68LJZdL17lhWy
   （两次结果不同，但都能验证 '123456' 是正确密码）

2. 慢哈希（Cost Factor）：
   saltRounds（也叫 cost factor）控制计算次数
   saltRounds = 10 → 计算 2^10 = 1024 次
   saltRounds = 12 → 计算 2^12 = 4096 次
   
   普通哈希（MD5）：每秒可以计算几十亿次
   bcrypt（saltRounds=10）：每秒只能计算几十次
   → 暴力破解的成本极高
```

### 1.3 安装与基础使用

```bash
npm install bcrypt
```

```javascript
const bcrypt = require('bcrypt')

// ─── 加密密码 ───
// bcrypt.hash(plainPassword, saltRounds) → Promise<string>
const saltRounds = 10  // 推荐值：10-12
const plainPassword = '123456'

const hashedPassword = await bcrypt.hash(plainPassword, saltRounds)
console.log(hashedPassword)
// $2b$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhWy
// 格式：$2b$<cost>$<22位salt><31位hash>

// ─── 验证密码 ───
// bcrypt.compare(plainPassword, hashedPassword) → Promise<boolean>
const isMatch = await bcrypt.compare('123456', hashedPassword)
console.log(isMatch)  // true

const isMatch2 = await bcrypt.compare('wrong', hashedPassword)
console.log(isMatch2)  // false
```

### 1.4 saltRounds 怎么选

```
saltRounds = 10：约 100ms，推荐（开发和大多数生产环境）
saltRounds = 12：约 400ms，高安全要求的场景
saltRounds = 14：约 1.5s，极高安全要求（登录会明显变慢）

原则：在用户可接受的响应时间内，尽量大
一般选 10，如果服务器性能好可以选 12
```

### 1.5 在 Mongoose 中集成

```javascript
// models/User.js
const bcrypt = require('bcrypt')

userSchema.pre('save', async function() {
  // 只有 password 字段被修改时才重新加密
  // 避免每次 save 都重新加密（如只改了 username）
  if (!this.isModified('password')) return

  this.password = await bcrypt.hash(this.password, 10)
})

// 验证密码的实例方法
userSchema.methods.comparePassword = async function(plainPassword) {
  return bcrypt.compare(plainPassword, this.password)
}
```

## 二、参数校验：express-validator

### 2.1 为什么需要参数校验

```javascript
// 没有校验时，用户可以传任何数据
router.post('/register', async (req, res) => {
  const user = await User.create(req.body)
  // 如果 req.body 是 { username: '', email: 'not-an-email', password: '1' }
  // 数据库里就存了垃圾数据
  // 或者触发 Mongoose 校验错误，但错误信息不友好
})
```

在路由层做校验，可以：
- 提前拦截非法请求
- 返回友好的错误信息
- 防止 SQL 注入、XSS 等攻击

### 2.2 安装

```bash
npm install express-validator
```

### 2.3 基础使用

```javascript
const { body, param, query, validationResult } = require('express-validator')

// 定义校验规则（中间件数组）
const registerRules = [
  body('username')
    .trim()                                          // 去除首尾空格
    .notEmpty().withMessage('用户名不能为空')
    .isLength({ min: 2, max: 20 }).withMessage('用户名长度 2-20 个字符')
    .matches(/^[\w\u4e00-\u9fa5]+$/).withMessage('用户名只能包含字母、数字、下划线和中文'),

  body('email')
    .trim()
    .notEmpty().withMessage('邮箱不能为空')
    .isEmail().withMessage('邮箱格式不正确')
    .normalizeEmail(),                               // 标准化邮箱（转小写等）

  body('password')
    .notEmpty().withMessage('密码不能为空')
    .isLength({ min: 6, max: 50 }).withMessage('密码长度 6-50 个字符')
    .matches(/\d/).withMessage('密码必须包含数字')
]

// 在路由中使用
router.post('/register', registerRules, async (req, res) => {
  // 检查校验结果
  const errors = validationResult(req)
  if (!errors.isEmpty()) {
    // errors.array() 返回所有错误的数组
    return res.status(400).json({
      code: 400,
      message: errors.array().map(e => e.msg).join('；'),
      data: errors.array()  // 可以把详细错误也返回给前端
    })
  }

  // 校验通过，继续处理
  const { username, email, password } = req.body
  // ...
})
```

### 2.4 常用校验方法

```javascript
// 字符串
body('name').isString()
body('name').isLength({ min: 2, max: 20 })
body('name').matches(/^[a-zA-Z]+$/)
body('name').trim().notEmpty()

// 数字
body('age').isInt({ min: 0, max: 150 })
body('price').isFloat({ min: 0 })
body('count').isNumeric()

// 邮箱 / URL / 手机号
body('email').isEmail()
body('website').isURL()
body('phone').isMobilePhone('zh-CN')  // 中国手机号

// 枚举值
body('role').isIn(['user', 'admin', 'moderator'])

// 可选字段（不传时跳过校验）
body('bio').optional().isLength({ max: 200 })

// 自定义校验
body('confirmPassword').custom((value, { req }) => {
  if (value !== req.body.password) {
    throw new Error('两次密码不一致')
  }
  return true
})

// 路由参数校验
param('id').isMongoId().withMessage('ID 格式不正确')

// 查询参数校验
query('page').optional().isInt({ min: 1 }).withMessage('页码必须是正整数')
query('limit').optional().isInt({ min: 1, max: 100 })
```

### 2.5 封装校验中间件

```javascript
// middleware/validate.js
const { validationResult } = require('express-validator')

// 通用校验结果处理中间件
function validate(req, res, next) {
  const errors = validationResult(req)
  if (!errors.isEmpty()) {
    return res.status(400).json({
      code: 400,
      message: errors.array()[0].msg,  // 只返回第一个错误
      data: null
    })
  }
  next()
}

module.exports = validate
```

使用：

```javascript
const validate = require('../middleware/validate')

router.post('/register',
  registerRules,  // 校验规则
  validate,       // 处理校验结果
  async (req, res) => {
    // 到这里参数一定是合法的
    const { username, email, password } = req.body
    // ...
  }
)
```

## 三、限流：express-rate-limit

### 3.1 为什么需要限流

```
没有限流时：
  攻击者可以每秒发几千次请求
  暴力破解密码：每秒尝试 1000 个密码组合
  DDoS 攻击：大量请求把服务器打垮
  爬虫：大量抓取数据
```

### 3.2 安装

```bash
npm install express-rate-limit
```

### 3.3 基础使用

```javascript
const rateLimit = require('express-rate-limit')

// 全局限流：每 15 分钟最多 100 次请求
const globalLimiter = rateLimit({
  windowMs: 15 * 60 * 1000,  // 时间窗口：15分钟
  max: 100,                   // 最大请求次数
  message: {
    code: 429,
    message: '请求过于频繁，请 15 分钟后再试'
  },
  standardHeaders: true,      // 在响应头里返回限流信息（RateLimit-*）
  legacyHeaders: false        // 禁用旧版 X-RateLimit-* 头
})

app.use(globalLimiter)
```

### 3.4 针对敏感接口的严格限流

```javascript
// 登录接口：每 15 分钟最多 5 次（防暴力破解）
const loginLimiter = rateLimit({
  windowMs: 15 * 60 * 1000,
  max: 5,
  message: {
    code: 429,
    message: '登录尝试次数过多，请 15 分钟后再试'
  },
  // 按 IP 限流（默认）
  keyGenerator: (req) => req.ip
})

// 注册接口：每小时最多 3 次（防批量注册）
const registerLimiter = rateLimit({
  windowMs: 60 * 60 * 1000,
  max: 3,
  message: {
    code: 429,
    message: '注册过于频繁，请 1 小时后再试'
  }
})

// 发送验证码：每分钟最多 1 次
const smsCooldown = rateLimit({
  windowMs: 60 * 1000,
  max: 1,
  message: {
    code: 429,
    message: '发送太频繁，请 1 分钟后再试'
  }
})

// 应用到路由
app.post('/api/auth/login', loginLimiter, loginHandler)
app.post('/api/auth/register', registerLimiter, registerHandler)
app.post('/api/auth/send-code', smsCooldown, sendCodeHandler)
```

### 3.5 跳过某些请求

```javascript
const limiter = rateLimit({
  windowMs: 15 * 60 * 1000,
  max: 100,
  // 跳过内网 IP（如健康检查）
  skip: (req) => {
    const ip = req.ip
    return ip === '127.0.0.1' || ip === '::1'
  }
})
```

## 四、安全响应头：helmet

### 4.1 为什么需要安全响应头

浏览器有很多安全机制，通过响应头来控制。不设置这些头，网站容易受到各种攻击。

### 4.2 安装与使用

```bash
npm install helmet
```

```javascript
const helmet = require('helmet')

// 一行代码，自动设置多个安全响应头
app.use(helmet())
```

### 4.3 helmet 设置了哪些头

```
X-Content-Type-Options: nosniff
  → 禁止浏览器猜测 MIME 类型（防 MIME 嗅探攻击）

X-Frame-Options: SAMEORIGIN
  → 禁止页面被嵌入 iframe（防点击劫持）

X-XSS-Protection: 0
  → 禁用旧版 XSS 过滤器（现代浏览器已不需要，反而可能有问题）

Strict-Transport-Security: max-age=15552000; includeSubDomains
  → 强制使用 HTTPS（HSTS）

Content-Security-Policy: ...
  → 内容安全策略，控制页面可以加载哪些资源

Referrer-Policy: no-referrer
  → 控制 Referer 头的发送策略

Cross-Origin-Opener-Policy: same-origin
  → 防止跨域窗口访问
```

### 4.4 自定义配置

```javascript
// 如果默认配置影响了正常功能，可以单独配置
app.use(helmet({
  // 关闭 CSP（如果你的页面需要加载外部资源）
  contentSecurityPolicy: false,

  // 自定义 CSP
  contentSecurityPolicy: {
    directives: {
      defaultSrc: ["'self'"],
      scriptSrc: ["'self'", 'cdn.example.com'],
      imgSrc: ["'self'", 'data:', 'https:']
    }
  },

  // 允许 iframe 嵌入（如果需要）
  frameguard: { action: 'sameorigin' }
}))
```

## 五、其他安全实践

### 5.1 不暴露服务器信息

```javascript
// Express 默认在响应头里暴露服务器信息
// X-Powered-By: Express

// 关闭它
app.disable('x-powered-by')
// 或者用 helmet（helmet 默认会关闭这个头）
```

### 5.2 防止 MongoDB 注入

```javascript
// MongoDB 注入：攻击者传入对象而不是字符串
// 如：{ "email": { "$gt": "" } }  → 匹配所有文档

// 方案1：用 express-mongo-sanitize 过滤
const mongoSanitize = require('express-mongo-sanitize')
app.use(mongoSanitize())  // 自动移除 $ 和 . 开头的键

// 方案2：用 express-validator 校验类型
body('email').isEmail()  // 确保是字符串格式的邮箱
```

## 六、小结

| 知识点 | 核心要点 |
|--------|----------|
| bcrypt 加密 | `bcrypt.hash(password, 10)` — saltRounds 推荐 10 |
| bcrypt 验证 | `bcrypt.compare(plain, hashed)` → boolean |
| 为什么用 bcrypt | 加盐防彩虹表，慢哈希防暴力破解 |
| express-validator | `body('field').isEmail()` + `validationResult(req)` |
| 常用校验 | isEmail / isLength / isInt / isIn / matches / custom |
| express-rate-limit | `rateLimit({ windowMs, max, message })` |
| 限流场景 | 登录（5次/15分钟）、注册（3次/小时）、发验证码（1次/分钟） |
| helmet | `app.use(helmet())` — 自动设置多个安全响应头 |
| 环境变量 | dotenv + .env 文件，密钥不硬编码 |
