> 整理日期：2026-04-21
> 说明：本文基于官方文档核验，修正了原 AIGC 版本中的多处错误，可作为实现依据使用。

## 一、核心概念澄清

### OAuth2.0 授权原理（通用）

无论钉钉还是企业微信，OAuth2.0 登录的本质流程一致：

```
1. 前端构建授权链接（含 appid、redirect_uri、state 等参数）
2. 跳转到第三方授权页面
3. 用户同意授权（或静默自动同意）
4. 回调 redirect_uri，URL 携带 code 参数
5. 前端把 code 发给后端
6. 后端拿 code + appSecret → 换取 access_token
7. 后端拿 access_token → 调用 API 获取用户信息
8. 后端验证/创建本地用户，返回业务 Session
```

**关键点**：code 是临时的、一次性的，有效期通常 5 分钟。

## 二、钉钉登录实现

### 2.1 两种登录场景

| 场景                       | 登录方式                       | 是否需要 dd.config |
| -------------------------- | ------------------------------ | ------------------ |
| 钉钉内部 H5 微应用（免登） | `dd.ready` → `requestAuthCode` | ❌ 免登不需要      |
| 普通浏览器 / App 端        | OAuth 跳转授权                 | ❌ 不需要          |

> ⚠️ 重要澄清：`dd.config` 是用来声明通讯录、分享等**通用业务 JSAPI** 权限的，和免登是两回事。
> `requestAuthCode` 不需要先做 `dd.config`，只需要加载 SDK 并等待 `dd.ready`。
> 只有当页面还要继续调用扫码、选人等通用 JSAPI 时，才需要额外做 `dd.config`。

### 2.2 钉钉内部 H5 微应用（免登）

**特点**：

- 钉钉内置浏览器环境（UA 含 `DingTalk`）
- 加载 `dingtalk-jsapi` SDK，等 `dd.ready`，直接调 `requestAuthCode`
- 不需要 `dd.config`，不需要后端签名

**完整流程**：

```
1. 判断钉钉内部 H5 环境（UA 检测 DingTalk）
2. 获取学校 ID
3. 调用后端接口获取钉钉配置（corpId 等）
4. 懒加载钉钉 JS-SDK（单例 Promise，避免重复加载）
5. 等待 dd.ready（5 秒超时保护）
6. dd.runtime.permission.requestAuthCode({ corpId }) → 获取授权码
7. 携带 authCode + schoolID 请求后端登录接口
8. 会话落盘，跳转首页
```

**代码示例**：

```javascript
// 懒加载钉钉 SDK（单例 Promise）
let ddPromise = null;

function loadDdSdk() {
  if (window.dd?.runtime?.permission) return Promise.resolve(window.dd);
  if (ddPromise) return ddPromise;

  ddPromise = new Promise((resolve, reject) => {
    const existing = document.getElementById("app-dingtalk-sdk");
    if (existing) {
      existing.addEventListener("load", () => resolve(window.dd));
      existing.addEventListener("error", () =>
        reject(new Error("SDK 加载失败")),
      );
      return;
    }

    const script = document.createElement("script");
    script.id = "app-dingtalk-sdk";
    script.src =
      "https://g.alicdn.com/dingding/dingtalk-jsapi/2.11.0/dingtalk.open.js";
    script.async = true;
    script.onload = () => resolve(window.dd);
    script.onerror = () => {
      ddPromise = null;
      reject(new Error("SDK 加载失败"));
    };
    document.head.appendChild(script);
  });

  return ddPromise;
}

// 获取免登授权码（不需要 dd.config）
async function getDingTalkAuthCode(corpId) {
  const dd = await loadDdSdk();

  return new Promise((resolve, reject) => {
    const timer = setTimeout(() => reject(new Error("获取授权码超时")), 5000);

    dd.ready(() => {
      dd.runtime.permission.requestAuthCode({
        corpId,
        onSuccess: (res) => {
          clearTimeout(timer);
          resolve(res.code || res.authCode);
        },
        onFail: (err) => {
          clearTimeout(timer);
          reject(err);
        },
      });
    });

    dd.error((err) => {
      clearTimeout(timer);
      reject(err);
    });
  });
}
```

### 2.3 普通浏览器 / App 端（OAuth 跳转）

**特点**：

- 不在钉钉容器内
- 两阶段：首次进入跳转授权页，回调后从 URL 取 code

```javascript
async function dingTalkOAuthLogin(schoolID, corpId, clientId) {
  // 检查 URL 中是否已有 code（回调返回的）
  const code = getCodeFromUrl(); // 兼容 code 和 authCode 两种字段名
  if (code) {
    // 校验 state 防 CSRF
    if (!validateState(getStateFromUrl())) throw new Error("state 校验失败");
    clearOAuthCallbackParams();
    return await loginWithCode({ schoolID, code });
  }

  // 首次进入，生成 state 并跳转
  const state = generateState();
  saveState(state);

  const redirectUri = encodeURIComponent(buildCurrentPageUrl());
  const authUrl = `https://login.dingtalk.com/oauth2/auth?client_id=${clientId}&redirect_uri=${redirectUri}&response_type=code&scope=openid&state=${state}`;

  window.location.replace(authUrl);
}
```

## 三、企业微信登录实现

### 3.1 场景分类（重要）

| 场景                            | 登录方式                               | 是否需要 SDK |
| ------------------------------- | -------------------------------------- | ------------ |
| 企业微信内置浏览器（H5 微应用） | OAuth `snsapi_base` 跳转               | ❌ 不需要    |
| 系统浏览器 PC 端（Web 登录）    | `ww.createWWLoginPanel` 或构造登录链接 | ✅ 需要      |
| App 原生端                      | WebView 弹出 OAuth 授权页，拦截回调    | ❌ 不需要    |

> ⚠️ 重要：`ww.createWWLoginPanel` 和 `login.work.weixin.qq.com` 登录链接**只适用于系统浏览器（PC 端）**。
> 官方文档明确说明：如果需要在企业微信内置 Webview 进行登录，请使用 OAuth 授权登录。
> 两者不能混用。

### 3.2 企业微信内置浏览器（推荐：纯 URL 跳转，不需要 SDK）

**特点**：

- 企业微信内置浏览器（UA 含 `wxwork`）
- `snsapi_base` 静默授权，用户无感知
- 不需要任何 SDK，代码最简单
- 和 App 端 OAuth 逻辑可以完全复用

**OAuth 授权 URL 格式**（官方文档 `91022`）：

```
https://open.weixin.qq.com/connect/oauth2/authorize
  ?appid=CORPID
  &redirect_uri=REDIRECT_URI（需 URLEncode）
  &response_type=code
  &scope=snsapi_base
  &state=STATE
  &agentid=AGENTID
  #wechat_redirect
```

**注意**：

- `snsapi_privateinfo` 时 `agentid` 必填，否则系统视为 `snsapi_base`
- `state` 最长 128 字节，只能用 `a-zA-Z0-9`

**代码示例**：

```javascript
async function weComH5Login(schoolID, corpId, agentId) {
  // 检查 URL 中是否已有 code（回调返回的）
  const code = getCodeFromUrl();
  if (code) {
    if (!validateState(getStateFromUrl())) throw new Error("state 校验失败");
    clearOAuthCallbackParams();
    return await loginWithCode({ schoolID, code, clientType: "H5" });
  }

  // 首次进入，构造静默授权 URL 并跳转
  const state = generateState();
  saveState(state);

  const redirectUri = encodeURIComponent(buildCurrentPageUrl());
  const authUrl = `https://open.weixin.qq.com/connect/oauth2/authorize?appid=${corpId}&redirect_uri=${redirectUri}&response_type=code&scope=snsapi_base&agentid=${agentId}&state=${state}#wechat_redirect`;

  window.location.replace(authUrl);
}
```

### 3.3 系统浏览器 PC 端（Web 登录组件）

**特点**：

- 系统浏览器打开的 PC 端管理后台
- 内嵌二维码，支持企业微信桌面端快速登录
- 需要 `@wecom/jssdk >= 2.3.2`
- 必须 HTTPS

**快速登录显示条件（全部满足才显示）**：

1. 企业微信桌面端版本 ≥ 3.1.23，已运行并登录
2. 开发者网页使用 HTTPS
3. `login_type=CorpApp` 且当前用户在应用可见范围内
4. 网页在桌面端系统浏览器中打开（不是企微内置 Webview）
5. `@wecom/jssdk >= 2.3.2`（Chrome 142+ 必须）

**代码示例**：

```javascript
import * as ww from "@wecom/jssdk";

ww.createWWLoginPanel({
  el: "#wx-login-container",
  params: {
    login_type: "CorpApp",
    appid: "ww1234567890abcdef", // 企业 corpId
    agentid: "1000002",
    redirect_uri: encodeURIComponent("https://your-domain.com/callback"),
    state: "random_state_string",
    lang: "zh",
  },
  onCheckWeComLogin({ isWeComLogin }) {
    // 判断企业微信桌面端是否已登录，可据此决定是否直接显示快速登录面板
    console.log("企业微信桌面端登录状态：", isWeComLogin);
  },
  onLoginSuccess({ code }) {
    // 拿到 auth code，发给后端换取用户信息
    loginWithCode(code);
  },
  onLoginFail(err) {
    console.error("登录失败：", err);
  },
});
```

**也可以不用 SDK，直接构造登录链接跳转**（官方文档 `98152`）：

```
https://login.work.weixin.qq.com/wwlogin/sso/login
  ?login_type=CorpApp
  &appid=CORPID
  &agentid=AGENTID
  &redirect_uri=REDIRECT_URI（需 URLEncode）
  &state=STATE
```

### 3.4 App 原生端（WebView 拦截）

**特点**：

- `plus.webview.create` 打开 OAuth 授权页
- `overrideUrlLoading` 拦截回调 URL 提取 code
- 和 H5 端共用同一个后端登录接口，`clientType` 传 `'APP'`

### 3.5 关于"能不能用 SDK 在企微内置浏览器里登录"

**结论：不能，也没必要。**

`@wecom/jssdk` 里没有类似钉钉 `requestAuthCode` 那样能直接返回授权码的接口。翻遍官方文档，`ww.getLoginInfo`、`ww.login` 这类 API 根本不存在，这是平台设计上的差异，绕不过去。

企微内置浏览器里获取用户身份，官方唯一的方式就是 OAuth 跳转（`snsapi_base` 静默授权），用户完全无感知，体验和钉钉 `requestAuthCode` 一样好，没有理由非要用 SDK。

**SDK 在企微内置浏览器里的正确用途是调用 JS 能力（扫码、选人等），不是登录。** 详见第九节。

## 四、钉钉 vs 企业微信 对照表

| 对比维度                    | 钉钉                                                     | 企业微信                                         |
| --------------------------- | -------------------------------------------------------- | ------------------------------------------------ |
| 内部 H5 获取 code           | `dd.ready` → `requestAuthCode`（SDK 直接返回，无需跳转） | OAuth `snsapi_base` 跳转（无需 SDK，静默无感知） |
| 内部 H5 是否需要 JSAPI 鉴权 | ❌ 免登不需要 dd.config                                  | ❌ 不需要任何 SDK                                |
| SDK 能否直接拿授权码        | ✅ 可以（requestAuthCode）                               | ❌ 没有这个接口                                  |
| 外部浏览器 OAuth            | ✅ 支持                                                  | ✅ 支持                                          |
| PC 端内嵌登录组件           | 需额外组件                                               | `ww.createWWLoginPanel`（官方支持）              |
| PC 端快速登录               | 无                                                       | ✅ 企业微信桌面端一键登录                        |
| 用户标识                    | userId                                                   | userid（小写，企业内唯一）                       |
| 企业标识                    | corpId                                                   | corpId（`ww` 开头）                              |
| 应用标识                    | agentId / clientId                                       | agentId（纯数字）                                |

## 六、后端统一处理逻辑

无论前端用哪种方式获取 code，后端处理逻辑完全一致：

```
1. 接收前端传来的 code
2. 调用钉钉/企业微信 API，用 code 换取用户信息
3. 查询/创建本地用户记录
4. 生成业务 Session/JWT
5. 返回给前端
```

### 企业微信后端接口（官方文档核验）

```
# 获取 access_token
GET https://qyapi.weixin.qq.com/cgi-bin/gettoken
  ?corpid=CORPID&corpsecret=SECRET

# 通过 code 获取用户信息（注意是 GET，不是 POST）
GET https://qyapi.weixin.qq.com/cgi-bin/auth/getuserinfo
  ?access_token=ACCESS_TOKEN&code=CODE

# 返回（企业成员）
{
  "errcode": 0,
  "errmsg": "ok",
  "userid": "zhangsan",          // 小写，企业内唯一
  "user_ticket": "USER_TICKET"   // snsapi_privateinfo 且在可见范围内时返回
}

# 返回（非企业成员/外部联系人）
{
  "errcode": 0,
  "errmsg": "ok",
  "openid": "OPENID",
  "external_userid": "EXTERNAL_USERID"
}
```

### 钉钉后端接口

```
# 获取 access_token
GET https://oapi.dingtalk.com/gettoken
  ?appkey=APPKEY&appsecret=APPSECRET

# 通过 code 获取用户信息
POST https://oapi.dingtalk.com/topapi/v2/user/getuserinfo
Body: { "code": "xxx" }
Headers: x-acs-dingtalk-access-token: ACCESS_TOKEN

# 返回
{
  "result": {
    "name": "张三",
    "unionId": "xxx",
    "userid": "xxx"
  }
}
```

## 八、注意事项

### 企业微信配置要点

1. 可信域名：授权回调域名必须与访问链接域名完全一致（含端口）
2. HTTPS：Web 登录组件的快速登录功能必须使用 HTTPS
3. 可信 IP：2022.6.20 后新建应用必须配置，否则服务端接口调用被拦截
4. `snsapi_base` 只返回 userid，`snsapi_privateinfo` 返回详细信息（agentid 必填）

### 钉钉配置要点

1. 可信域名：必须在钉钉开放平台配置
2. 应用权限：必须在管理后台配置通讯录读权限，否则后端接口报权限错误
3. 应用发布：配置完成后必须发布，否则配置不生效

### 通用注意事项

1. code 有效期 5 分钟，一次性使用
2. state 必须校验，防 CSRF 攻击
3. H5 端 OAuth 回调后必须清除 URL 中的 code/state 参数，避免刷新重复消费
4. access_token 和 jsapi_ticket 有效期 7200 秒，后端必须缓存，不能每次都重新请求

## 九、企业微信 JSSDK 能力调用（登录之后）

### 9.1 登录与 JS 能力是两件独立的事

登录（获取用户身份）和调用企业微信 JS 能力（扫码、选人等）是完全独立的两个步骤，不要混为一谈：

```
用户打开 H5 页面
  │
  ├─ 步骤一：登录（OAuth 跳转，不需要 SDK）
  │     → 拿到 code → 后端换 userid → 登录完成
  │
  └─ 步骤二：初始化 JSSDK（登录完成后，需要调用 JS 能力时）
        → 加载 @wecom/jssdk
        → ww.register（传企业签名 + 应用签名回调）
        → 注册完成后直接调用 ww.scanQRCode / ww.invoke 等
```

两步串行，先登录，再按需初始化 SDK。

### 9.2 ww.register 初始化（新版 SDK，推荐）

```javascript
import * as ww from "@wecom/jssdk";

// 登录完成后，需要用到 JS 能力时调用一次
ww.register({
  corpId: "ww1234567890abcdef",
  agentId: 1000002,
  jsApiList: ["scanQRCode", "selectEnterpriseContact", "previewFile"],
  async getConfigSignature(url) {
    // 调后端接口，传当前页面 URL，返回企业签名
    const res = await uni.$uv.http.post("/api/wxwork/getWxConfig", { url });
    return res.data.data; // { timestamp, nonceStr, signature }
  },
  async getAgentConfigSignature(url) {
    // 调后端接口，返回应用签名（使用应用 jsapi_ticket，与企业 ticket 不同）
    const res = await uni.$uv.http.post("/api/wxwork/getAgentConfig", { url });
    return res.data.data; // { timestamp, nonceStr, signature }
  },
});

// 注册后直接调用，SDK 内部自动处理时序，不需要等 ready
ww.scanQRCode({
  needResult: 1,
  success(res) {
    console.log("扫码结果：", res.resultStr);
  },
});
```

### 9.3 旧版 SDK 写法（wx.config + wx.agentConfig）

旧版需要两步，且 `agentConfig` 必须在 `wx.config` 的 `ready` 回调内调用，iOS/Android 调用方式还不同：

```javascript
// 第一步：企业签名
wx.config({
  beta: true, // 必须，否则 wx.invoke 形式的 jsapi 有问题
  debug: false,
  appId: "ww1234567890abcdef", // corpId
  timestamp: "...",
  nonceStr: "...",
  signature: "...",
  jsApiList: ["scanQRCode", "previewFile"],
});

wx.ready(() => {
  // 第二步：应用签名（必须在 ready 里调用）
  const ua = navigator.userAgent;
  const isAndroid = /Android|Linux/i.test(ua);

  if (isAndroid) {
    // 安卓用 wx.invoke 方式
    wx.invoke(
      "agentConfig",
      {
        corpid: "ww1234567890abcdef",
        agentid: "1000002",
        timestamp: "...",
        nonceStr: "...",
        signature: "...",
        jsApiList: ["selectEnterpriseContact"],
      },
      (res) => {
        /* 回调 */
      },
    );
  } else {
    // iOS / PC 用 wx.agentConfig 方式
    wx.agentConfig({
      corpid: "ww1234567890abcdef",
      agentid: "1000002",
      timestamp: "...",
      nonceStr: "...",
      signature: "...",
      jsApiList: ["selectEnterpriseContact"],
      success() {
        /* 初始化成功 */
      },
      fail(res) {
        if (res.errMsg.indexOf("function not exist") > -1) {
          alert("企业微信版本过低，请升级");
        }
      },
    });
  }
});
```

> 官方强烈建议迁移到新版 `ww.register`，旧版不再新增接口。

### 9.4 两种 ticket 的区别（后端必须区分）

| ticket 类型       | 用途                                                          | 获取接口                                    |
| ----------------- | ------------------------------------------------------------- | ------------------------------------------- |
| 企业 jsapi_ticket | `wx.config` / `ww.register` 的 `getConfigSignature`           | `GET /cgi-bin/get_jsapi_ticket`             |
| 应用 jsapi_ticket | `wx.agentConfig` / `ww.register` 的 `getAgentConfigSignature` | `GET /cgi-bin/ticket/get?type=agent_config` |

两个 ticket 接口不同，不能混用。都有严格的调用频率限制，后端必须缓存（有效期 7200 秒）。

### 9.6 iOS URL 签名问题

iOS 企业微信中，SPA 路由变化后 URL 不变，签名 URL 与实际 URL 不一致会导致 `invalid signature`。

**旧版 SDK 需要手动处理**：

```javascript
// 记录首次进入的 URL，每次签名都用这个 URL
if (!window.__firstPageUrl) {
  window.__firstPageUrl = window.location.href.split("#")[0];
}
const signUrl = isIOS()
  ? window.__firstPageUrl
  : window.location.href.split("#")[0];
```

**新版 `ww.register` 不需要处理**，SDK 内部自动解决了这个问题。这也是推荐迁移新版的原因之一。
