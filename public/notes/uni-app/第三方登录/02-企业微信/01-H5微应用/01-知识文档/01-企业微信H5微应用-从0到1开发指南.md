# 企业微信H5微应用-从0到1开发指南

> 官方文档：https://developer.work.weixin.qq.com/document/path/91335

---

## 1. H5 微应用是什么

企业微信 H5 微应用是在企业微信内置浏览器中打开的普通 H5 页面，通过 OAuth2 授权获取企业成员身份，并可调用企业微信 JSSDK 实现扫码、选人等能力。

**适用场景**：

- 现有 uni-app 项目编译 H5 后直接接入，改动最小
- 不需要小程序审核，发布更灵活
- 需要调用企业微信 JS 能力（扫码、选人、预览文件等）

## 2. 管理后台配置

### 2.1 创建应用

1. 登录企业微信管理后台：https://work.weixin.qq.com/
2. 「应用管理」→「自建」→「创建应用」
3. 填写应用名称、Logo、可见范围
4. 记录 **corpId**、**agentId**、**Secret**

### 2.2 配置应用主页

1. 进入应用详情 → 「功能」→「网页」
2. 设置「企业应用主页」为 H5 页面地址
3. 格式：`https://your-domain.com/path`

### 2.3 配置可信域名

1. 应用详情 → 「开发者接口」→「网页授权及JS-SDK」
2. 添加「可信域名」（必须与 H5 页面域名完全一致，含端口）
3. 下载域名校验文件，放到服务器根目录

### 2.4 配置可信 IP

1. 应用详情 → 「开发者接口」→「企业可信IP」
2. 添加后端服务器 IP（2022.6.20 后新建应用必须配置）

## 3. OAuth2 授权登录流程

### 3.1 流程图

```
用户在企业微信中打开 H5 页面
    │
    ▼
前端检测是否有 code 参数
    │
    ├── 有 code → 发给后端换取用户信息 → 登录成功
    │
    └── 无 code → 构造 OAuth2 授权 URL → 重定向到企业微信授权页
                                              │
                                              ▼
                                    用户确认授权（或静默授权）
                                              │
                                              ▼
                                    重定向回 redirect_uri?code=CODE
```

### 3.2 构造授权 URL

```javascript
// util/auth/wxworkOAuth.js

/**
 * 构造企业微信 OAuth2 授权 URL
 * @param {string} corpId - 企业 corpId
 * @param {string} agentId - 应用 agentId（snsapi_privateinfo 时必填）
 * @param {string} redirectUri - 授权后回调地址（需 URL encode）
 * @param {string} scope - snsapi_base（静默）或 snsapi_privateinfo（手动，可获取手机号等）
 * @param {string} state - 自定义状态参数（可选，a-zA-Z0-9，最长128字节）
 */
export const buildOAuthUrl = ({
  corpId,
  agentId,
  redirectUri,
  scope = "snsapi_base",
  state = "STATE",
}) => {
  const encodedUri = encodeURIComponent(redirectUri);
  // 注意：snsapi_privateinfo 时 agentid 必填，否则会报错
  const agentParam = agentId ? `&agentid=${agentId}` : "";
  return `https://open.weixin.qq.com/connect/oauth2/authorize?appid=${corpId}&redirect_uri=${encodedUri}&response_type=code&scope=${scope}${agentParam}&state=${state}#wechat_redirect`;
};

/**
 * 触发 OAuth2 授权跳转
 */
export const redirectToOAuth = (config) => {
  const url = buildOAuthUrl(config);
  window.location.href = url;
};
```

**参数说明**：

- `scope=snsapi_base`：静默授权，可获取成员 userid
- `scope=snsapi_privateinfo`：手动授权，可获取手机号、邮箱等敏感信息，此时 `agentid` 必填，且成员必须在应用可见范围内
- `state`：重定向后会带上该参数，可用于防 CSRF 攻击

### 3.3 前端登录入口

```javascript
// util/auth/wxworkLogin.js

import { buildOAuthUrl } from "./wxworkOAuth.js";

const CORP_ID = "ww1234567890abcdef"; // 从配置文件读取
const AGENT_ID = "1000002"; // 从配置文件读取

/**
 * 企业微信 H5 微应用登录
 * 1. 检查 URL 中是否有 code
 * 2. 有 code → 发给后端换取 token
 * 3. 无 code → 跳转 OAuth2 授权
 */
export const wxworkH5Login = async () => {
  const urlParams = new URLSearchParams(window.location.search);
  const code = urlParams.get("code");

  if (code) {
    // 有 code，发给后端换取用户信息
    try {
      const res = await uni.$uv.http.post("/api/wxwork/h5Login", { code });
      if (res.data.code === 200) {
        uni.setStorageSync("token", res.data.data.token);
        uni.setStorageSync("userInfo", res.data.data.userInfo);
        // 清除 URL 中的 code 参数，避免刷新重复使用
        const cleanUrl = window.location.href
          .replace(/[?&]code=[^&]+/, "")
          .replace(/[?&]state=[^&]+/, "");
        window.history.replaceState({}, "", cleanUrl);
        return res.data.data;
      }
    } catch (err) {
      console.error("H5 登录失败：", err);
      throw err;
    }
  } else {
    // 无 code，跳转授权
    redirectToOAuth({
      corpId: CORP_ID,
      agentId: AGENT_ID,
      redirectUri: window.location.href,
      scope: "snsapi_base",
    });
  }
};
```

## 4. 后端接口说明

### 4.1 用 code 换取用户信息

**接口地址**：

```
GET https://qyapi.weixin.qq.com/cgi-bin/auth/getuserinfo
  ?access_token=ACCESS_TOKEN
  &code=CODE
```

**返回值（企业成员）**：

```json
{
  "errcode": 0,
  "errmsg": "ok",
  "userid": "zhangsan",
  "user_ticket": "USER_TICKET" // scope=snsapi_privateinfo 且成员在应用可见范围内时返回
}
```

**返回值（非企业成员/外部联系人）**：

```json
{
  "errcode": 0,
  "errmsg": "ok",
  "openid": "OPENID",
  "external_userid": "EXTERNAL_USERID" // 仅当用户是企业客户且跟进人在应用可见范围内时返回
}
```

**注意事项**：

- code 有效期 5 分钟，只能使用一次
- 跳转的域名须完全匹配应用的可信域名，否则返回 50001 错误

### 4.2 用 user_ticket 换取敏感信息（手动授权）

**接口地址**：

```
POST https://qyapi.weixin.qq.com/cgi-bin/auth/getuserdetail
  ?access_token=ACCESS_TOKEN
Body: { "user_ticket": "USER_TICKET" }
```

**返回值**：

```json
{
  "errcode": 0,
  "errmsg": "ok",
  "userid": "zhangsan",
  "gender": "1",
  "avatar": "http://shp.qpic.cn/bizmp/xxxxxxxxxxx/0",
  "qr_code": "https://open.work.weixin.qq.com/wwopen/userQRCode?vcode=xxx",
  "mobile": "13800138000",
  "email": "zhangsan@gzdev.com",
  "biz_mail": "zhangsan@qyycs2.wecom.work",
  "address": "广州市海珠区新港中路"
}
```

**字段说明**：

- `gender`：0=未定义，1=男性，2=女性
- `mobile`、`email`、`biz_mail`、`address`：第三方应用不可获取
- 敏感字段需要管理员在应用详情里选择，且成员 OAuth2 授权时确认后才返回

**权限要求**：

- 成员必须在应用的可见范围内
- scope 必须为 `snsapi_privateinfo`

## 5. 授权注意事项

### 5.1 手动授权 30 天免重复弹窗

用户首次手动授权后，30 天内再次进入应用不会再弹出授权页，默认沿用上次授权的敏感信息。仅适用于 2022.6.20 后新建的自建应用、代开发应用，以及所有第三方应用。

### 5.2 官方推荐的缓存方案

频繁调用 OAuth2 接口有性能开销，官方建议：

1. 企业应用 URL 直接填写自己的页面地址
2. 用户进入页面时，后台检查是否有标识成员身份的 cookie
3. 没有 cookie → 重定向到 OAuth2 授权链接，获取 userId 后由后台植入 cookie
4. 有 cookie → 直接进入页面

> 本项目用 token 替代 cookie 实现同样效果，逻辑一致。

### 5.3 与钉钉 H5 微应用的主要差异

| 对比项       | 钉钉 H5 微应用        | 企业微信 H5 微应用                       |
| ------------ | --------------------- | ---------------------------------------- |
| 授权方式     | `dd.getAuthCode`      | OAuth2 重定向                            |
| 用户标识     | userId                | userid（企业内账号，小写）               |
| JSAPI 初始化 | `dd.config`           | `wx.config` + `wx.agentConfig`           |
| 扫码         | `dd.scan`             | `wx.scanQRCode`                          |
| 选人         | `dd.chooseStaffForPC` | `wx.invoke('selectEnterpriseContact')`   |
| 静默授权     | 支持                  | 支持（snsapi_base）                      |
| 手动授权     | 支持                  | 支持（snsapi_privateinfo，agentid 必填） |

## 6. 常见问题

### Q1：redirect_uri 参数错误

**原因**：回调域名未在管理后台配置「可信域名」，或域名不完全一致。

**处理**：

1. 检查管理后台「可信域名」配置
2. 确认域名（含端口）与 redirect_uri 完全一致
3. 确认域名校验文件已放到服务器根目录

### Q2：code 只能使用一次

**原因**：OAuth2 code 是一次性的，5 分钟内有效。

**处理**：

- 获取 code 后立即发给后端换取 token
- 换取成功后清除 URL 中的 code 参数

### Q3：iOS 中 JSSDK 签名失败

**原因**：iOS 微信/企业微信中，SPA 路由变化后 URL 不变，签名 URL 与实际 URL 不一致。

**处理**：记录首次进入的 URL，每次签名都用这个 URL（见 JSSDK 接入文档）。

### Q4：在非企业微信浏览器中打开

**处理**：

```javascript
// 检测是否在企业微信浏览器中
const isWxworkBrowser = () => {
  return /wxwork/i.test(navigator.userAgent);
};

if (!isWxworkBrowser()) {
  // 提示用户在企业微信中打开，或展示二维码
  uni.showToast({ title: "请在企业微信中打开", icon: "none" });
}
```
