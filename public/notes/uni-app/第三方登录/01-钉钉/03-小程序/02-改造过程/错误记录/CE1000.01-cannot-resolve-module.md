# CE1000.01 - cannot resolve module

## 出现阶段

阶段 1 - 编译链路接入，补完 `mp-dingtalk` 编译脚本后首次编译时触发。

## 错误信息

```
error[CE1000.01]: pagesContactList\groupChat\index.js:2397:29
cannot resolve module '@/configInfo/index.js'

error[CE1000.01]: pagesContactList\privateChat\index.js:2741:27
cannot resolve module '@/configInfo/index.js'

error[CE1000.01]: teacherApp\appSettings\index.js:220:24
cannot resolve module '@/util/websocket/storage/chatStorageManager.js'
```

## 原因

三处代码在函数体内使用了 `require('@/xxx')` 动态引入模块。

钉钉小程序走 `mp-alipay` 编译通道，编译器只能静态分析顶部 `import` 语句来解析模块依赖，函数体内的 `require()` 属于运行时动态引入，编译期无法解析带 `@/` 别名的路径，导致报错。

## 影响文件

| 文件 | 问题写法 |
|------|---------|
| `pagesContactList/groupChat/index.vue` | 函数体内 `require('@/configInfo/index.js').default` |
| `pagesContactList/privateChat/index.vue` | 函数体内 `require('@/configInfo/index.js').default`（顶部已有静态 import，重复引入） |
| `teacherApp/appSettings/index.vue` | 函数体内 `require('@/util/websocket/storage/chatStorageManager.js').default` |

## 解决方式

把函数体内的 `require()` 全部改为文件顶部静态 `import`，函数体内直接使用已导入的变量。

### groupChat

顶部补：
```js
import configInfo from '@/configInfo/index.js'
```
删掉函数体内：
```js
const configInfo = require('@/configInfo/index.js').default  // 删除
```

### privateChat

顶部已有 `import configInfo from '@/configInfo/index.js'`，只删掉函数体内重复的：
```js
const configInfo = require('@/configInfo/index.js').default  // 删除
```

### appSettings

顶部补：
```js
import chatStorage from '@/util/websocket/storage/chatStorageManager.js'
```
删掉函数体内：
```js
// ✅ 改为 require()，避免微信小程序动态导入问题
const chatStorage = require('@/util/websocket/storage/chatStorageManager.js').default  // 删除
```

## 结论

微信 / 支付宝 / 钉钉小程序均不支持函数体内 `require()` 动态引入带 `@/` 别名的模块。统一规范：模块引入一律写在文件顶部 `import`，不在函数体内使用 `require()`。
