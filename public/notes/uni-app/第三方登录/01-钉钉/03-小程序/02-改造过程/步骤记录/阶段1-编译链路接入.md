# 阶段 1 - 编译链路接入

## 目标

让项目能在钉钉开发者工具里跑起来，首页不白屏。

---

## 步骤 1：补 package.json 的 mp-dingtalk 编译脚本

### 背景

`package.json` 没有 `scripts` 字段，也没有 `uni-app.scripts`，项目完全没有钉钉构建入口。这是阶段 1 的第一个硬缺口，不补这个就无法编译出钉钉产物。

### 改动文件

`package.json`

### 改动内容

新增以下两块：

```json
"scripts": {
  "dev:mp-dingtalk": "uni -p mp-alipay",
  "build:mp-dingtalk": "uni build -p mp-alipay"
},
"uni-app": {
  "scripts": {
    "mp-dingtalk": {
      "title": "钉钉小程序",
      "env": {
        "UNI_PLATFORM": "mp-alipay"
      },
      "define": {
        "MP-DINGTALK": true
      }
    }
  }
}
```

### 说明

- 钉钉小程序在 uni-app 中没有原生平台，走 `mp-alipay` 编译通道。
- `define` 里注入 `MP-DINGTALK: true`，后续用 `#ifdef MP-DINGTALK` 做条件编译隔离钉钉专属逻辑。

### 验收

在 HBuilderX 顶部菜单 → 运行 → 运行到小程序模拟器，自定义脚本 `mp-dingtalk` 出现在列表，选择后编译产物可导入钉钉开发者工具。

---

## 步骤 2：修复编译报错 CE1000.01

编译后出现三处 `cannot resolve module` 报错，见 [错误记录/CE1000.01-cannot-resolve-module.md](../错误记录/CE1000.01-cannot-resolve-module.md)，修复后首页成功打开。

---

## 结论

阶段 1 完成，编译链路已打通，首页可在钉钉开发者工具中正常打开。
