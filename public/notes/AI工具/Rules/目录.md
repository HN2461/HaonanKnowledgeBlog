# AI 工具规则文档系列目录

> 更新时间：2026-04-11

规则文档是"给 AI 看的项目 README"——把每次都要重复说的技术栈、代码规范、边界约束写进一个文件，AI 工具在每次会话开始时自动读取，从第一行代码起就遵守你的规范。

本系列共 5 篇，覆盖从概念理解到各工具实战配置的完整路径。

## 覆盖主题

**AGENTS.md 开放标准**
- OpenAI 2025-08 发起，Linux 基金会 AAIF 治理
- 四段式结构：Stack / Conventions / Boundaries / Verification
- 多级目录与子目录作用域隔离
- 全局个人配置 `~/.codex/AGENTS.md`
- 各工具支持情况与优先级规则

**CLAUDE.md 与 Claude Code 记忆系统**
- 四层文件层级：组织级 / 项目级 / 用户级 / 本地私有
- `@import` 语法引用外部文件
- `.claude/rules/` 路径作用域规则
- Auto Memory 自动记忆机制
- `/memory` 命令管理与调试

**Cursor Rules**
- `.cursorrules`（legacy）→ `.cursor/rules/`（MDC 格式）演进
- 四种规则类型：Always / Auto Attached / Agent Requested / Manual
- YAML frontmatter：`alwaysApply` / `globs` / `description`
- 全局规则与项目规则优先级

**Windsurf Rules**
- `.windsurfrules`（旧）→ `.windsurf/rules/*.md`（新）
- 四种激活模式：`always_on` / `glob` / `model_decision` / `manual`
- 全局规则 `~/.codeium/windsurf/memories/global_rules.md`
- 6000 token 预算管理
- Memories 自动记忆 vs Rules 手动规则

**Kiro Steering**
- `.kiro/steering/*.md` 目录结构
- 四种 inclusion 模式：always / fileMatch / manual / auto
- 文件引用语法 `#[[file:path]]`
- 一键生成基础 Steering 文件

**Gemini CLI GEMINI.md**
- 三层加载：全局 / 工作区 / JIT（即时）
- `/memory show` / `/memory reload` / `/memory add` 命令
- `@file.md` 模块化导入语法
- 自定义文件名配置

**GitHub Copilot**
- `.github/copilot-instructions.md`（原有格式）
- `.github/instructions/*.instructions.md`（新格式，支持 `applyTo` 路径作用域）
- AGENTS.md 支持（2025-08 起）
