# 第一篇：Cursor 上手、迁移与索引配置（2026-04）

> 本篇依据 Cursor 官方 `Installation`、`Quickstart`、`VS Code`、`Codebase Indexing`、`Ignore files` 文档整理，资料检索时间：2026-04-14。

[[toc]]

---

## 1. 为什么现在把入门只保留这一篇

真正决定 Cursor 上手体验的，不是“会不会按快捷键”，而是这三件事有没有先配对：

1. 你的旧编辑器习惯有没有顺利迁过来
2. 仓库有没有被正确索引
3. 不该进上下文的噪声和敏感文件有没有先挡住

所以这次不再把安装、迁移、索引、ignore 拆成很多小稿，而是合并成一篇开箱清单。

---

## 2. 首次安装后先做什么

Cursor 官方安装文档给的路径很直接：

- 去 `cursor.com/downloads` 下载对应平台安装包
- 首次启动时完成快捷键、主题、终端偏好的 onboarding
- 如果之后想重新跑一遍初始化流程，可以在命令面板搜索 `Cursor: Start Onboarding`

第一次启动时，我更建议主人按这个顺序来：

1. 先登录账号，确保 AI 功能可用
2. 先迁移常用快捷键和主题，减少操作摩擦
3. 直接打开一个真实项目，不要先在空目录里试
4. 立刻看索引是否开始跑
5. 先配 ignore，再开始让 Agent 接手任务

---

## 3. 从 VS Code 迁移时，优先迁什么

Cursor 官方明确支持从 VS Code 一键导入：

- `Extensions`
- `Themes`
- `Settings`
- `Keybindings`

入口在 `Cursor Settings` 的 `General > Account` 下的 `VS Code Import`。如果是跨机器迁移，官方也支持用 VS Code 的 `Profile Export / Import` 手动搬配置。

这里有两个很实用的官方细节：

- Cursor 基于 VS Code 代码库，所以迁移成本通常很低
- 为了稳定性，Cursor 常常会落后于最新版 VS Code 一点点，不要把“不是最新 VS Code 内核”误判成安装异常

如果主人不习惯默认横向的 Activity Bar，也可以按官方文档把 `workbench.activityBar.orientation` 改回 `vertical`。

---

## 4. Codebase Indexing 比 prompt 更优先

官方安装页和索引页都强调了同一件事：打开项目后，Cursor 会自动开始做 `Codebase Indexing`，这一步决定后面 Agent、Ask、搜索和相关文件召回准不准。

几个关键点：

- 索引是自动启动的，不需要手动触发
- 项目大小不同，耗时大约在 `1-15` 分钟
- 新文件会增量索引
- 团队可以共享索引，减少重复等待
- 可以在 `Ctrl+Shift+J -> Indexing & Docs` 查看进度
- 还能通过 `View included files` 打开一份实际被索引文件的清单

如果主人发现 Cursor 只会围着当前文件打转，优先检查的不是模型，而是：

1. 索引有没有完成
2. 被索引的文件范围对不对
3. 有没有把大量无关生成物也一起喂进去

## 5. `.cursorignore` 和 `.cursorindexingignore` 怎么分工

这是官方文档里最容易被忽略、但实际非常关键的边界。

### 5.1 `.cursorignore`

`.cursorignore` 会阻止这些文件进入：

- Codebase indexing
- Tab、Agent、Inline Edit 可访问的代码
- `@` 符号引用可访问的代码

但官方也明确提醒了一件重要的事：Agent 发起的工具调用，例如终端命令和 MCP server，本身不能被 `.cursorignore` 完全隔绝。所以它是强约束，但不是“绝对保险箱”。

### 5.2 `.cursorindexingignore`

`.cursorindexingignore` 只把文件排除在索引之外，这些文件仍可能被其他 AI 功能访问。它更适合处理“噪声很多、但偶尔还想读”的内容。

### 5.3 一个更稳的起手示例

```gitignore
dist/
coverage/
node_modules/
*.log
.env
.env.*
secrets/
```

如果是大仓库，还可以打开官方提到的 `Hierarchical Cursor Ignore`，让父目录里的 `.cursorignore` 也参与生效。

---

## 6. 第一天最值得配好的清单

主人如果只想记最少的动作，照这个清单做就够了：

1. 装好 Cursor 后先完成 onboarding，再导入 VS Code 配置
2. 打开真实项目，确认索引已经开始
3. 去 `Indexing & Docs` 看是否有明显不该被纳入的目录
4. 新建 `.cursorignore`，先挡住密钥、构建产物、日志和大体量噪声目录
5. 用 `View included files` 抽查索引结果
6. 确认常用快捷键和终端设置已经迁过来

当这几步做完，后面的 Rules、Agent 和工作流文章才真正值得看。
