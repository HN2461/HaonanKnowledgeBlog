# 第二十八篇：Cursor 常见问题排障与网络诊断（2026-04）

> 本篇基于 Cursor 官方 Common Issues 文档整理，并结合中文排障习惯做归纳，资料快照时间：2026-04-14。

[[toc]]

---

## 1. 遇到问题先从网络诊断开始

官方文档建议，先到：

```text
Cursor Settings -> Network -> Run Diagnostics
```

做网络诊断。

这一步很重要，因为很多“AI 不工作”“索引失败”“更新失败”的根因，其实是网络层问题，而不是模型或项目本身的问题。

---

## 2. HTTP/2 与公司代理

官方文档明确提到，Cursor 的 AI 功能依赖 HTTP/2 来更高效地处理流式响应。

如果你的网络环境：

- 不支持 HTTP/2
- 有企业代理
- 使用 VPN
- 使用像 Zscaler 这样的代理产品

就可能出现：

- 索引失败
- AI 功能异常
- Tab / Inline Edit 失效

官方给出的解决方案是：在应用级 Settings 中关闭 HTTP/2，强制回退到 HTTP/1.1。

设置方式：

1. 打开应用设置
2. 搜索 `HTTP/2`
3. 开启 `Disable HTTP/2`

---

## 3. GitHub 登录 / 退出问题

官方 Common Issues 文档提到，如果 GitHub 登录有问题，或者想退出 GitHub，可以通过：

```text
Sign Out of GitHub
```

这个 Command Palette 命令来处理。

另外，文档也提到，如果你刚升级 Pro，但应用里仍显示免费状态，可以尝试重新登录刷新状态。

---

## 4. Remote SSH 问题

官方文档提到：

- GitHub Codespaces 目前不支持
- SSH 到 Mac 或 Windows 机器不支持

如果是 Windows 上出现：

```text
SSH is only supported in Microsoft versions of VS Code
```

官方建议：

1. 卸载 `Remote-SSH`
2. 安装 `Anysphere Remote SSH`
3. 关闭所有 VS Code SSH 连接
4. 重启 Cursor 再试

这类问题很像“Cursor 不支持 SSH”，但其实常常是扩展装错了。

---

## 5. Tab / Inline Edit 在代理后失效

官方文档明确说，Tab 和 Inline Edit 在企业代理后可能失效，原因通常还是 HTTP/2。

如果出现这类问题，优先：

1. 跑网络诊断
2. 回退 HTTP/1.1
3. 再验证 Tab 和 Inline Edit

不要一开始就重装应用。

---

## 6. CPU / RAM 异常

官方文档也提到资源占用问题：

- CPU 过高
- RAM 过高

这类问题会拖慢机器甚至导致告警。遇到时应先排查：

- 是否开了过多大型项目
- 是否有索引和多轮 Agent 同时运行
- 是否存在后台扩展异常

官方 Troubleshooting Guide 还给了几条更直接的定位方法：

1. 打开 `Developer: Open Process Explorer`，先看是 `window`、`extensionHost` 还是 `ptyHost` 在吃资源
2. 如果怀疑是扩展问题，可以先用 `cursor --disable-extensions` 启动一次做对照
3. 需要细查扩展性能时，先在 `Settings -> Application -> Experimental` 开启 `Extension Monitor`，重启后再执行 `Developer: Open Extension Monitor`

资源问题未必是 Cursor 本身，也可能是项目和扩展叠加造成的。

---

## 7. 历史记录丢失

官方文档提到，如果磁盘空间不足，更新时可能导致聊天或 Composer 历史数据丢失。

因此建议：

1. 更新前留足磁盘空间
2. 定期清理系统垃圾文件
3. 重要对话及时沉淀到文档或规则，不要只留在聊天里

这也再次说明：重要信息别只放在历史对话里。

---

## 8. 看到更新日志但客户端还没更新

官方 Common Issues 页面还提到，如果你在 changelog 里已经看到新版本，但本地客户端还没收到更新，这通常是 staged rollout 的正常现象。先等一段时间再检查，通常比反复重装更稳。

---

## 9. 一句话总结

Cursor 排障最常见的主线是：先查网络，再看代理与 HTTP/2，再排查登录、扩展和资源问题。很多表面像“AI 坏了”的问题，本质其实是环境问题。
