## 这组资料适合谁

如果你遇到的是下面这种场景，这组资料基本就是对口的：

1. 外部 PowerShell / CMD 里 `node` 正常
2. 只有 Codex 终端里 `node -e`、`npm`、`vite`、`vitest` 异常
3. 最近刚用 `CC Switch` 切过 Codex 中转或同步过配置
4. 报错里出现 `ncrypto::CSPRNG(nullptr, 0)`、`NTE_PROVIDER_DLL_FAIL`、`0x8009001d`

## 以后自己排查时先看什么

每次在 `CC Switch` 里切完 Codex 中转后，优先做这两步：

```powershell
Select-String -Path C:\Users\HN246\.codex\config.toml -Pattern 'shell_environment_policy|inherit = "core"|inherit = "all"'
pwsh -File scripts/checkNodeRuntime.ps1
```

如果看到：

```toml
[shell_environment_policy]
inherit = "core"
```

而且 `node -e` 又开始失败，就优先回看主文里的修复步骤，不要先去卸载重装 Node。
