# 企业微信开发从0到1\_结合企业微信开放平台

> 官方文档入口：https://developer.work.weixin.qq.com/document/path/90664

---

## 1. 企业微信应用类型速览

| 类型                 | 说明                               | 本项目关联 |
| -------------------- | ---------------------------------- | ---------- |
| 自建应用（小程序）   | 企业内部开发，绑定企业微信小程序   | 主要场景   |
| 自建应用（H5微应用） | 企业内部 H5 页面，在企业微信内打开 | 次要场景   |
| 第三方应用           | 服务商代开发，多企业复用           | 暂不涉及   |

---

## 2. 核心概念

### 2.1 企业微信 vs 普通微信

| 对比项        | 普通微信小程序          | 企业微信小程序                            |
| ------------- | ----------------------- | ----------------------------------------- |
| 登录 API      | `wx.login`              | `wx.qy.login`                             |
| code 换取信息 | `code2Session` → openId | `code2Session` → **userId**（企业内唯一） |
| 用户标识      | openId / unionId        | **userId**（企业账号，非 openId）         |
| JSSDK         | `wx.config`             | `wx.config` + `wx.agentConfig`（双层）    |
| 应用配置      | appId                   | corpId + agentId                          |

### 2.2 两种开发模式

**模式一：企业微信小程序**

- 在企业微信客户端内运行的小程序
- 调用 `wx.qy.login` 获取 code，后端用 code2Session 换 userId
- 适合需要深度集成企业通讯录、审批等能力的场景

**模式二：企业微信 H5 微应用**

- 普通 H5 页面，在企业微信内置浏览器中打开
- 通过 OAuth2 授权获取 userId
- 需要接入 JSSDK（`wx.config` + `wx.agentConfig`）才能调用企业微信 JS 接口
- uni-app 编译 H5 后接入，改动最小

## 3. 开发前准备

### 3.1 管理后台配置

1. 登录企业微信管理后台：https://work.weixin.qq.com/
2. 进入「应用管理」→「自建」→「创建应用」
3. 记录以下关键信息：
   - **corpId**：「我的企业」→「企业信息」→「企业ID」
   - **agentId**：应用详情页的「AgentId」
   - **Secret**：应用详情页的「Secret」（需在企业微信客户端接收）

### 3.2 小程序关联配置

- 在应用详情页 → 「关联小程序」，绑定对应的微信小程序 appId
- 配置「可信域名」（H5 微应用必须）
- 配置「可信 IP」（2022.6.20 后新建应用必须，否则服务端接口调用会被拦截）

### 3.3 uni-app manifest.json 配置

企业微信小程序本质上还是微信小程序，`manifest.json` 中无需特殊平台配置，但需要注意：

```json
// manifest.json
{
  "mp-weixin": {
    "appid": "wx企业微信关联的小程序appid",
    "setting": {
      "urlCheck": false
    }
  }
}
```

> 注意：企业微信小程序的 appId 是微信小程序的 appId，不是 corpId。

## 4. 服务端接口调用流程

```
1. 获取 access_token
   GET https://qyapi.weixin.qq.com/cgi-bin/gettoken?corpid=CORPID&corpsecret=SECRET

2. 用 access_token 调用各业务接口
   - 通讯录：/cgi-bin/user/get
   - 消息推送：/cgi-bin/message/send
   - 获取 jsapi_ticket：/cgi-bin/get_jsapi_ticket
   - 获取应用 jsapi_ticket：/cgi-bin/ticket/get?type=agent_config

3. access_token 有效期 7200秒，必须后端缓存，不能频繁获取
```

## 5. 前端开发关键约束

- 企业微信小程序内，`wx.qy.login` 替代 `wx.login`，其余 wx API 基本兼容
- H5 微应用内，需要引入企业微信 JSSDK（`jweixin-1.x.x.js` + `jwxwork-1.x.x.js`）
- 签名必须在服务端生成，前端只负责传当前页面 URL 给后端换签名参数
- `agentConfig` 必须在 `wx.config` 的 `ready` 回调内调用，顺序不能乱
- iOS 和 Android 调用 `agentConfig` 的方式不同（见 JSSDK 接入文档）

## 6. 与钉钉开发的主要差异

| 对比项         | 钉钉                  | 企业微信                               |
| -------------- | --------------------- | -------------------------------------- |
| 登录           | `dd.getAuthCode`      | `wx.qy.login`                          |
| 用户标识       | userId / unionId      | userId（企业内唯一）                   |
| JSAPI 初始化   | `dd.config`           | `wx.config` + `wx.agentConfig`         |
| 扫码           | `dd.scan`             | `wx.scanQRCode`                        |
| 选人           | `dd.chooseStaffForPC` | `wx.invoke('selectEnterpriseContact')` |
| 消息推送       | 钉钉工作通知          | 企业微信应用消息                       |
| 小程序平台标识 | `MP-DINGTALK`         | `MP-WEIXIN`（企业微信复用微信小程序）  |
