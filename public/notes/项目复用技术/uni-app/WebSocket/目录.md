# uni-app WebSocket 专题目录

这一组文章专门服务一个目标：

`以后别的项目要接 WebSocket / STOMP 时，可以直接按目录一篇篇研究、模仿和复制。`

## 推荐阅读顺序

1. [01-uni-app-WebSocket与STOMP总览_分层设计与接入顺序.md](./01-uni-app-WebSocket与STOMP总览_分层设计与接入顺序.md)
   先建立全局地图，知道这一整套为什么要分层。

2. [02-uni-app-STOMP帧封装实战_命令拼装拆帧心跳与粘包处理.md](./02-uni-app-STOMP帧封装实战_命令拼装拆帧心跳与粘包处理.md)
   先把最底层的协议帧看明白，不然后面客户端层不好理解。

3. [03-uni-app-StompClient封装实战_connect订阅心跳与回调分发.md](./03-uni-app-StompClient封装实战_connect订阅心跳与回调分发.md)
   继续看客户端层如何把 `uni.connectSocket` 和 STOMP 协议真正接起来。

4. [04-uni-app-连接管理与自动重连封装实战_状态机退避策略与用户决策.md](./04-uni-app-连接管理与自动重连封装实战_状态机退避策略与用户决策.md)
   再看最关键的连接状态机、自动重连、暂停重连和用户决策。

5. [05-uni-app-消息解析与服务层封装实战_消息标准化订阅分发与页面薄消费.md](./05-uni-app-消息解析与服务层封装实战_消息标准化订阅分发与页面薄消费.md)
   看后端消息怎么标准化成前端稳定结构，以及服务层怎么广播给页面。

6. [06-uni-app-MessageServiceAdapter与页面接入实战_统一入口事件桥接与初始化时机.md](./06-uni-app-MessageServiceAdapter与页面接入实战_统一入口事件桥接与初始化时机.md)
   最后看如何把整套复杂底层收口成页面层统一入口。

## 这组文章最适合什么时候回来看

1. 新项目要从零接 WebSocket
2. 旧项目的实时消息逻辑已经变得很乱
3. 想把聊天、通知、在线状态统一到一套底座里
4. 想把“页面里直接 connectSocket”改造成可维护方案

## 一句话提醒

如果你只记住一句话：

`先把协议层和状态机搭稳，再谈页面层消息体验。`
