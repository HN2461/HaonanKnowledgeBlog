# 3.4-9、组件通讯
## 3.4.1、props配置
props配置项，让组件接收外部传过来的数据

一般是父组件给子组件传递，props配置在子组件内接收父组件传递的数据，也可以让父元素接受到子元素的数据

父组件===》子组件 传递数据 用的比较多

子组件==> 父组件 传递数据 用的比较少

### 3.4.1.1、传递数据（父组件里传递给子组件）
```html
<Demo name="李四" sex='女' :age="18" :ceshi='ceshi'/>
```

这里的age，测试前面加：，也就是v-bind，表明里面是表达式

### 3.4.1.2、接收数据（子组件内接收数据）
接收一：数组方式（只接收）

```javascript
props: ["name", "age", "sex", "ceshi"],
```

接收二 ：对象方式（限制类型）

```javascript
props:{
    name:String,
    age:Number,
    sex:String
    ceshi：Object
}
```

接收三：对象方式（限制类型，限制必要性，指定默认值）

```javascript
props: {
    name: {
      type: String, //name的类型是字符串
      required: true, //name是必要的
    },
    age: {
      type: Number,
      default: 99, //默认值
    },
    sex: {
      type: String,
      required: true,
    },
    ceshi: {
      type: Object,
      required: true,
    },
}
```

### 3.4.1.3、注意：
props是只读的，Vue底层会监测你对props的修改，如果进行了修改，就会发出警告，若业务需求确实需要修改，请复制props的内容到data中，然后去修改data中的数据

对于传递的复杂数据类型，是无法监测到内容值的改变，除非修改地址值

```javascript
<script>
export default {
  name: "Student",
  data() {
    console.log(this);
    return {
      msg: "我是学生",
      myAge: this.age,//复制props接收到的数据到myAge
    };
  },
  methods: {
    updateAge() {
      this.myAge++;//
    },
  },
  //接收一：简单声明
  props: ["name", "age", "sex", "ceshi"],
}
</script>

```

### 3.4.1.4、子传递数据给父
1、在父组件中定义方法，去接收，去处理子组件传递的数据

```javascript
//1、 接受子组件传递的数据
handleGirlInfo(value){
   console.log(`接受到了girl组件传递的数据${value}`);
}
```

2、将父组件的方法，传递给子组件

```html
<!-- 2、将方法传递给子组件 -->
<girl msg="hello,girl" a="666" :m="m" :handleGirlInfo="handleGirlInfo"/>
```

3、子组件接收父组件传递的方法，并且调用这个方法，调用的同时，传递数据

```javascript
// 接受方式一
props: ['msg', 'a', 'm', 'handleGirlInfo'],
data() {
    return {
        girlInfo: '我是girl组件'
    }
},
methods: {
    // 3、子组件触发父组件传递的方法，同时传递实参过去
    sendInfo() {
        this.handleGirlInfo(this.girlInfo)
    }
}
```

## 3.4.3、组件自定义事件
click、keydown、keyup，这些事件都是内置事件。

Vue也支持给组件添加自定义事件。

包括两种方式：

第一种方式：直接在组件标签上绑定事件

第二种方式：通过代码来给组件绑定事件

### 3.4.3.1 直接在组件标签上绑定事件
1.关于内置事件的实现步骤。

第一步：提供事件源（以下这个按钮就是一个事件源）

第二步：给事件源绑定事件 v-on:事件名 或者 @事件名

第三步：编写回调函数，将回调函数和事件进行绑定

第四步：等待事件的触发，只要事件触发，则执行回调函数。

2.关于组件的自定义事件，实现步骤：

第一步：提供事件源（这个事件源是一个组件）

第二步：给组件绑定事件 v-on:事件名 或者 @事件名

第三步：编写回调函数，将回调函数和事件进行绑定

第四步：等待事件的触发，只要事件触发，则执行回调函数。

对于组件自定义事件来说，要想让事件发生，需要去执行一段代码。这段代码负责去触发这个事件，让这个事件发生。

这段代码在哪里写？

事件绑定在A组件上，则触发这个事件的代码要在A组件当中编写。

```html
<div id="app" class="bg">
    <h1>App组件</h1>
    <girl @resiveMsg="resiveMsg" />
    <son />
</div>

```

```javascript
methods: {
    resiveMsg(msg) {
      console.log('接受子组件传递的数据：', msg);
    }
}
```

```html
<div class="bg">
    <h2>girl组件</h2>
    <button @click="sendMsg">传递数据</button>
</div>

```

```javascript
methods: {
    sendMsg() {
        this.$emit('resiveMsg', 666)
    }
}
```

### 3.4.3.2 通过代码给组件绑定事件
通过ref标记来获取元素，然后用$on给父组件绑定自定义事件

触发还是在子组件中

```html
<template>
    <div class="app">
        <h1>app父组件</h1>
        <!-- 第二种方式、通过代码绑定自定义事件 -->
        <helloworld ref="hello" />
    </div>
</template>

```

```javascript
<script>
export default {
    name: 'app',
    //在组件挂载完毕后，去绑定自定义事件
    mounted() {
        this.$refs.hello.$on('event2', this.fun2)
    },
    methods: {
        fun2(msg, ...params) {
            console.log('自定义事件触发了····,', msg, params);
        }
    },
}
</script>

```

触发还是一样在子组件中触发

这种方式有个小坑，需要注意的

this.$ refs.hello. $on('自定义事件名', 函数)，函数不同的写法，this指向不一样

```javascript
mounted() {
    // 此处有点个小坑，this的指向问题
    this.$refs.hello.$on('event2', () => {
        // 此时的this是app组件
        console.log(this);
    })
}
```

methods 中的函数默认绑定组件实例。

普通函数写法中 this 指向绑定事件的目标组件（如 hellowrod），箭头函数写法中 this 继承外层组件（如 app 组件）。

### 3.4.3.3 解绑事件
哪个组件绑定的就找哪个组件解绑：例如：app组件给mygirl组件绑定事件，到mygirl组件上解绑事件

```javascript
methods: {
    unbinding() {
        this.$off('event2')//解绑1个事件
        this.$off(['event2', 'event1'])//解绑多个事件
        this.$off()//解绑所有事件
    }
}
```

注意：vm和vc销毁的时候，所有组件以及子组件当中的事件会全部解绑。

## 3.4.4、provide / inject
provide：可以让我们指定想要提供给后代组件的数据或方法

inject：在任何后代组件中接收想要添加在这个组件上的数据或方法，不管组件嵌套多深都可以直接拿来用

app.vue

```javascript
provide() {
    return {
        name: "jack",
        age: this.num,
        someMethod: this.someMethod  // methods 中的方法
    }
},
methods: {
    someMethod() {
        console.log("这是注入的方法")
    }
}
```

child.vue

```javascript
<script>
export default {
    name: 'MyChild',
    // 孙组件：获取数据
    inject: ["name", 'age', 'someMethod'],
}
</script>

```

注意事项：

单向数据流：provide 和 inject 主要用于单向的数据传递，也就是从祖先组件向后代组件传递。虽然后代组件可以修改接收到的数据，但这并不推荐，因为会破坏单向数据流的原则，使数据流向变得复杂，不利于代码的维护。

响应式问题：如果 provide 提供的数据是响应式的（例如来自 data 选项），那么后代组件接收到的基本类型数据也是响应式的，但修改会报错。如果 provide 提供的是复杂数据类型，修改这个值不会报错，但没有响应式。

依赖关系：使用 provide 和 inject 会使组件之间产生一定的依赖关系，因为后代组件依赖于祖先组件提供的数据。所以在使用时要注意组件的结构和数据的流向，避免过度依赖导致代码难以维护。

## 3.4.5、全局事件总线
原理：给项目中所有的组件找一个共享的vc对象。把这个共享的对象vc叫做全局事件总线。

所有的事件都可以绑定到这个共享对象上。所有组件都通过这个全局事件总线对象来传递数据。

这种方式可以完美的完成兄弟组件之间传递数据。这样的共享对象必须具备两个特征：

(1) 能够让所有的vc共享。

(2) 共享对象上有$ on、 $off、$emit等方法。

在main.js中创建: 两种方案，推荐第二种

```javascript

        Vue.prototype.$bus = this;
    },
```

## 3.4.6、消息订阅与发布
<!-- 这是一张图片，ocr 内容为： -->
![](https://i-blog.csdnimg.cn/direct/9c960fa2923a4bd9a178b6ba028d160e.png)

使用pubsub-js库完成消息订阅与发布。该库可以在任意前端框架中实现消息的订阅与发布。

安装 pubsub-js： npm i pubsub-js

程序中引入pubsub：import pubsub from 'pubsub-js'

**<u>引入了一个pubsub对象，通过调用该对象的subscribe进行消息订阅，调用publish进行消息发布。</u>**

需求：兄弟组件的传参

helloworld.vue

```javascript
<template>
    <div class="hello">
        <h2>helloworld组件</h2>
    </div>
</template>
<script>
import pubsub from 'pubsub-js'
export default {
    name: 'hellowrod',
    data() {
        return {
            msg: '我是helloworld组件！'
        }
    },
    mounted() {
                    // 消息订阅
        this.pid = pubsub.subscribe('msgName', function (msgName, msg) {
            console.log('收到消息了,消息名字：', msgName, '消息是', msg);
        })
    },
    beforeDestroy() {
        pubsub.unsubscribe(this.pid)
    }
}
</script>

```

hellochild.vue

```javascript
<template>
    <div class="child">
        <h2>hellochild组件</h2>
        <button @click="sendmsg">发送数据给兄弟</button>
    </div>
</template>
<script>
import pubsub from 'pubsub-js'
export default {
    name: 'hellochild',
    data() {
        return {
            msg: '我是hellochild组件'
        }
    },
    methods: {
        sendmsg() {
              // 消息发布
            pubsub.publish('msgName', this.msg)
        }
    }
}
</script>

```

## 3.4.8、$ children /  $parent/$root
$children：获取到一个包含所有子组件(不包含孙子组件)的 VueComponent 对象数组，可以直接拿到子组件中所有数据和方法等

$parent：获取到一个父节点的 VueComponent 对象，同样包含父节点中所有数据和方法等

$root 可以拿到 vm 里的数据和方法

```javascript
// Parent.vue
export default{
    mounted(){
        this.$children[0].someMethod() // 调用第一个子组件的方法
        this.$children[0].name // 获取第一个子组件中的属性
    }
}

// Child.vue
export default{
    mounted(){
        this.$parent.someMethod() // 调用父组件的方法
        this.$parent.name // 获取父组件中的属性
    }
}
```

## 3.4.9、$ attrs /  $listeners ( 非父子组件通信 )
### 1、基本理解
$attrs 是一个包含了父组件传递给子组件但未在子组件的 props 选项中声明的所有属性的对象。通过它，我们可以实现非 props 属性的透传，避免在中间组件中重复定义这些属性。

$listeners 是一个包含了父组件传递给子组件的所有事件监听器的对象。利用它可以将父组件的事件监听器传递给更深层级的子组件，而无需在中间组件里重新绑定这些事件。

当存在多层嵌套的组件结构，且中间组件不需要使用某些属性或处理某些事件，仅起到传递作用时，$ attrs 和  $listeners 就能发挥很大的作用，减少中间组件的代码量，提高代码的可维护性。

### 2 使用方法
以下是一个包含父组件、子组件和孙组件的三层组件结构示例，展示了如何使用 $ attrs 和  $listeners 进行通信。

父组件 Parent.vue

```html
<template>
  <div>
    <ChildComponent
      message="这是一条消息"
      count="10"
      @click="handleClick"
      @custom-event="handleCustomEvent"
    />
  </div>
</template>
<script>
import ChildComponent from './Child.vue';
export default {
  components: {
    ChildComponent
  },
  methods: {
    handleClick() {
      console.log('点击事件被触发');
    },
    handleCustomEvent() {
      console.log('自定义事件被触发');
    }
  }
};
</script>

```

子组件 Child.vue

```html
<template>
  <div>
    <GrandChildComponent
      v-bind="$attrs"
      v-on="$listeners"
    />
  </div>
</template>
<script>
import GrandChildComponent from './GrandChild.vue';
export default {
  components: {
    GrandChildComponent
  },
  inheritAttrs: false // 禁止将未声明的属性绑定到根元素
};
</script>

```

孙组件 GrandChild.vue

```html
<template>
  <div>
    <p>接收到的消息: {{ $attrs.message }}</p>
    <p>接收到的计数: {{ $attrs.count }}</p>
    <button @click="$emit('click')">触发点击事件</button>
    <button @click="$emit('custom-event')">触发自定义事件</button>
  </div>
</template>
<script>
export default {
  inheritAttrs: false // 禁止将未声明的属性绑定到根元素
};
</script>

```

### 3 注意事项
inheritAttrs 选项：在使用 $attrs 时，通常需要设置 inheritAttrs: false，这样可以避免未声明的属性被默认绑定到组件的根元素上，从而更精确地控制属性的传递。

事件传递的层级：虽然 $listeners 可以实现跨层级的事件传递，但过多的层级可能会使代码逻辑变得复杂，降低代码的可维护性。因此，在使用时要根据实际情况进行合理设计。

事件处理函数的作用域：事件处理函数的作用域始终是定义该函数的组件，即父组件。在多层传递过程中，不会改变事件处理函数的作用域。

## 3.4.10、v-model
在 Vue 2 中，v-model 是一个非常方便的指令，常用于实现表单输入的双向数据绑定。同时，它也可以用于自定义组件之间的通信，实现父组件和子组件之间的数据双向绑定。下面详细介绍其原理和使用方法。

### （1）、基本原理
在原生表单元素上，v-model 本质上是 :value 和 @input 事件的语法糖。例如，对于一个 input 元素：

```html
<template>
  <input v-model="message">
</template>
<script>
export default {
  data() {
    return {
      message: ''
    };
  }
}
</script>

```

上面的代码等同于：

```html
<template>
  <input :value="message" @input='message = $event.target.value'>
</template>
<script>
export default {
  data() {
    return {
      message: ''
    };
  }
}
</script>

```

### （1）在自定义组件中使用v-model
在自定义组件中使用 v-model，子组件需要接收一个 value prop 并触发一个 input 事件。

子组件 ChildComponent.vue

```html
<template>
  <input :value="value" @input="$emit('input', $event.target.value)">
</template>
<script>
export default {
  props: ['value']
}
</script>

```

父组件 ParentComponent.vue

```html
<template>
  <div>
    <p>父组件中的值: {{ parentValue }}</p>
    <ChildComponent v-model="parentValue" />
  </div>
</template>
<script>
import ChildComponent from './ChildComponent.vue';
export default {
  components: {
    ChildComponent
  },
  data() {
    return {
      parentValue: ''
    };
  }
}
</script>

```

在上述代码中，父组件通过 v-model 将 parentValue 传递给子组件，子组件接收 value prop 并在输入框输入内容时触发 input 事件，将新的值传递回父组件，从而实现双向数据绑定。

此外：

v-model 使用 value 作为 prop 和 input 作为事件。但你可以通过 model 选项来自定义 prop 和事件的名称。

一个组件可以支持多个 v-model 绑定，通过不同的 prop 和事件来实现。

## 3.4.11、总结
面试题：请你说一说组件通讯？



| 组件关系 | 传递方式 |
| --- | --- |
| 父传子 | props，v-model，默认插槽，具名插槽,$parent |
| 子传父 | props，自定义事件，v-model,$ refs, $children,作用域插槽 |
| 祖先与后代 | $ attrs /  $listeners ,provide / inject |
| 任意组件通讯 | $bus,pubsub，vuex |


| 通讯方式 | 适用场景 | 特点 | 使用方法 |
| --- | --- | --- | --- |
| props | 父组件向子组件传递数据 | 单向数据流，子组件不能直接修改props | 在父组件中通过属性传递数据，在子组件中通过props接收 |
| 自定义事件 | 子组件向父组件传递数据 | 父组件绑定事件监听，子组件触发事件 | 在父组件中使用v-on绑定事件，在子组件中使用this.$emit触发事件 |
| ref | 父组件获取子组件实例 | 可以直接操作子组件的数据和方法 | 在子组件上添加ref属性，通过this.$refs.获取子组件 |
| provide/inject | 祖先组件向后代组件传递数据 | 无论组件嵌套多深，后代组件都可以接收数据 | 在祖先组件中使用provide提供数据，在后代组件中使用inject接收数据 |
| 全局事件总线 | 兄弟组件之间传递数据 | 所有组件共享一个事件总线，通过它进行事件绑定和触发 | 创建一个全局事件总线，组件通过this.$bus进行事件操作 |
| pubsub-js | 兄弟组件之间传递数据 | 使用消息订阅与发布模式，组件之间解耦 | 安装pubsub-js库，使用subscribe订阅消息，publish发布消息 |
| 插槽 | 父组件向子组件传递HTML结构 | 可以传递任意HTML结构，子组件定义插槽位置 | 在子组件中使用slot定义插槽，在父组件中传递HTML结构 |
| $ attrs/ $listeners | 非父子组件通信 | 将父组件的属性和事件监听器传递给更深层级的子组件 | 在中间组件中使用v-bind="$ attrs"和v-on=" $listeners"进行属性和事件的透传 |
| v-model | 父子组件之间双向数据绑定 | 本质上是:value和@input的语法糖，实现数据双向绑定 | 在子组件中接收value prop并触发input事件，在父组件中使用v-model绑定数据 |
