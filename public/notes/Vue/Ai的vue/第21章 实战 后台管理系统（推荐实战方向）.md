# 第21章 实战：后台管理系统（推荐实战方向）

> 目标：用 Vue2 完成一个典型后台管理系统的核心链路：
> 
> - 需求分析与模块拆分
> - Layout（侧边栏/顶栏/内容区）
> - 登录（表单校验、token 管理、路由守卫）
> - 权限（菜单权限、动态路由）
> - CRUD 页面（列表/分页/搜索/弹窗表单）
> - 数据可视化（ECharts）
> - 打包部署

---

## 21.1 需求分析与模块拆分

一个最小可用的后台系统通常包含：

- 登录/退出
- 首页仪表盘
- 用户管理（CRUD）
- 角色/权限管理（基础版可先不做全）
- 通用组件（表格、弹窗、表单、分页）

拆分建议：

- 页面模块：`views/`
- 组件模块：`components/`
- 接口模块：`api/`
- 状态模块：`store/`

---

## 21.2 页面布局设计（Layout：侧边栏/顶栏/内容区）

常见布局结构：

- 左侧：菜单（侧边栏）
- 顶部：用户信息/退出/面包屑
- 中间：`router-view` 渲染当前页面

通常会把 Layout 作为嵌套路由的父级组件：

- `/` → Layout
- `/users` `/roles` 等作为 children

---

## 21.3 登录模块

### 21.3.1 表单校验

建议做：

- 必填校验
- 最小长度/格式校验
- 提交时禁用按钮、防重复提交

### 21.3.2 token 管理

登录成功：

- 后端返回 token
- 前端存到 localStorage（或 Vuex + 持久化）

```js
localStorage.setItem('token', token)
```

退出：

- 清 token
- 清用户信息
- 跳转登录页

### 21.3.3 路由守卫

在 `router.beforeEach` 里做鉴权：

- 无 token → 去登录
- 有 token → 放行

---

## 21.4 权限系统

### 21.4.1 菜单权限

常见做法：

- 后端返回用户角色与菜单树
- 前端根据菜单渲染侧边栏

### 21.4.2 动态路由

进阶做法：

- 后端返回可访问的路由配置
- 前端运行时 `addRoutes`（Vue2 router）添加路由

注意：

- 动态路由常与权限点/角色体系绑定
- 需要考虑刷新页面后的恢复（持久化 + 重新生成路由）

---

## 21.5 CRUD 业务页面开发

一个典型 CRUD 页面包含：

- 列表
- 搜索（条件筛选）
- 分页
- 新增/编辑弹窗
- 删除确认

建议的 UI 组件抽象：

- `BaseTable`
- `BasePagination`
- `BaseDialog`

业务层只处理：

- 请求参数
- 表单数据
- 接口调用

---

## 21.6 数据可视化（ECharts）

常见需求：

- 首页折线图（趋势）
- 饼图（占比）
- 柱状图（对比）

实践建议：

- 在 `mounted` 初始化图表（DOM 已就绪）
- 在 `beforeDestroy` dispose，避免内存泄漏
- 窗口 resize 触发图表 resize

---

## 21.7 项目打包与部署

构建：

```bash
npm run build
```

部署关注点：

- 静态资源路径（publicPath）
- history 模式回退（服务端配置）
- gzip / CDN / 缓存策略（可选）

---

## 本章小结

- 后台项目的核心是：登录鉴权 + Layout + 权限 + CRUD。
- UI 组件抽象能显著提升开发效率。
- 图表与第三方库的生命周期管理要到位（初始化与销毁）。
- 部署要关注资源路径与路由回退。
