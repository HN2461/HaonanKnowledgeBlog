## 13.1 父 → 子：props

props 用于父组件向子组件传值。

### 13.1.1 基本传值

父组件：

```vue
<template>
  <UserCard :name="user.name" :age="user.age" />
</template>

<script>
import UserCard from './UserCard.vue'

export default {
  components: { UserCard },
  data() {
    return {
      user: { name: 'Tom', age: 18 }
    }
  }
}
</script>
```

子组件：

```vue
<template>
  <div>
    <p>name: {{ name }}</p>
    <p>age: {{ age }}</p>
  </div>
</template>

<script>
export default {
  props: ['name', 'age']
}
</script>
```

### 13.1.2 props 类型校验与默认值

更推荐的写法是对象形式（可读性更强，也便于排错）：

```js
export default {
  props: {
    name: {
      type: String,
      required: true
    },
    age: {
      type: Number,
      default: 0
    },
    tags: {
      type: Array,
      default() {
        return []
      }
    }
  }
}
```

说明：

- `required: true`：父组件必须传
- `default`：不传时的默认值
- 对象/数组默认值必须用函数返回，避免多个实例共享同一份引用

## 13.2 props 不可直接修改的原因

在 Vue2 中，子组件不应该直接修改 props：

```js
this.name = 'new' // 不推荐
```

原因是：

- props 的数据来源是父组件
- 子组件直接改会导致数据流混乱（“谁才是真正的来源？”）

### 13.2.1 正确做法：props 转成本地数据再改

如果子组件需要把 props 作为初始值并在内部编辑，可以：

```js
data() {
  return {
    localName: this.name
  }
}
```

但你需要注意：

- 这样 `localName` 后续不会自动跟随父组件变化
- 如果要跟随，需要再用 watch 监听 props

## 13.3 子 → 父：自定义事件 `$emit`

子组件通过 `$emit` 通知父组件：

- “我发生了某个行为”
- “我产生了某个数据”

### 13.3.1 基本用法

父组件监听子组件事件：

```vue
<UserCard @follow="onFollow" />
```

```js
methods: {
  onFollow(payload) {
    console.log('子组件通知 follow：', payload)
  }
}
```

子组件触发事件：

```vue
<template>
  <button @click="follow">关注</button>
</template>

<script>
export default {
  methods: {
    follow() {
      this.$emit('follow', { time: Date.now() })
    }
  }
}
</script>
```

## 13.4 组件 v-model 的实现方式（`value` + `input`）

在 Vue2 中，一个组件要支持 `v-model`，默认约定是：

- 父组件通过 `value` props 传入值
- 子组件通过 `input` 事件把新值传回父组件

### 13.4.1 父组件使用

```vue
<MyInput v-model="keyword" />
```

### 13.4.2 子组件实现

```vue
<template>
  <input :value="value" @input="onInput" />
</template>

<script>
export default {
  props: {
    value: {
      type: String,
      default: ''
    }
  },
  methods: {
    onInput(e) {
      this.$emit('input', e.target.value)
    }
  }
}
</script>
```

你可以把它理解为：

- `v-model="keyword"` 语法糖 ≈ `:value="keyword" @input="keyword = $event"`

## 13.5 兄弟组件通信（父组件中转）

兄弟组件 A 和 B 通信时，最推荐的方案通常是：

- A `$emit` 给父
- 父更新状态
- 父通过 props 把状态传给 B

父组件：

```vue
<template>
  <div>
    <Sender @send="msg = $event" />
    <Receiver :msg="msg" />
  </div>
</template>

<script>
import Sender from './Sender.vue'
import Receiver from './Receiver.vue'

export default {
  components: { Sender, Receiver },
  data() {
    return { msg: '' }
  }
}
</script>
```

这种方式的优点：

- 数据流清晰
- 便于维护与调试

## 13.6 跨层通信

当组件嵌套很深（爷爷 → 孙子），props 层层传会变得繁琐，这时可以考虑跨层方案。

### 13.6.1 `provide / inject`

适合：

- 祖先组件提供“依赖”
- 后代组件直接注入使用

祖先组件：

```js
export default {
  provide() {
    return {
      themeColor: '#2563eb'
    }
  }
}
```

后代组件：

```js
export default {
  inject: ['themeColor']
}
```

注意：

- `provide/inject` 更像“依赖注入”，适合共享配置/服务
- 不适合随意用于频繁变化的业务状态（那更像 Vuex 的职责）

### 13.6.2 `ref` 获取子组件实例

适合：

- 你需要调用子组件的方法（例如打开弹窗、重置表单）

父组件：

```vue
<template>
  <MyDialog ref="dialog" />
  <button @click="open">打开弹窗</button>
</template>

<script>
export default {
  methods: {
    open() {
      this.$refs.dialog.open()
    }
  }
}
</script>
```

子组件：

```js
export default {
  methods: {
    open() {
      this.visible = true
    }
  }
}
```

注意：

- `ref` 更偏“命令式”，不宜滥用
- 适合封装组件对外暴露少量方法

## 13.7 EventBus（Vue2 常见通信方案与适用场景）

EventBus 的思路：

- 创建一个空的 Vue 实例作为“事件中心”
- 任意组件都可以 `$emit` 和 `$on`

```js
// bus.js
import Vue from 'vue'
export const bus = new Vue()
```

发送方：

```js
import { bus } from './bus'

bus.$emit('msg', 'hello')
```

接收方：

```js
import { bus } from './bus'

export default {
  mounted() {
    bus.$on('msg', (val) => {
      console.log('收到：', val)
    })
  },
  beforeDestroy() {
    bus.$off('msg')
  }
}
```

适用场景：

- 小型项目
- 不想引入 Vuex，但确实存在少量跨组件事件

风险与缺点：

- 事件分散，难以追踪（谁在发、谁在听）
- 容易造成内存泄漏（忘了 `$off`）
- 项目变大后维护困难

在中大型项目里，通常更推荐：

- 用 Vuex 管理共享状态
- 用父子 props/$emit 保持数据流清晰

---

## 本章小结

- 父传子用 props（可做类型校验与默认值）。
- 子传父用 `$emit`。
- 组件 `v-model` 默认是 `value` props + `input` 事件。
- 兄弟通信优先走“父组件中转”。
- 跨层通信可用 `provide/inject` 或 `ref`（各有适用边界）。
- EventBus 适合小型项目，规模起来更推荐 Vuex。
