# 第5章 条件渲染（控制页面显示）

> 目标：掌握 Vue2 的条件渲染：
> 
> - `v-if / v-else-if / v-else`
> - `v-show`
> - 它们的性能差异与适用场景
> - `template` 配合条件渲染的写法

---

## 5.1 `v-if` / `v-else-if` / `v-else`

### 5.1.1 基本用法

`v-if` 用于“条件成立才渲染”。

```html
<div id="app">
  <p v-if="isLogin">欢迎回来，{{ username }}</p>
  <p v-else>请先登录</p>
</div>
```

这里的关键点是：

- 当 `isLogin` 为 `false` 时，`v-if` 对应的元素**不会出现在 DOM 中**

### 5.1.2 多分支：`v-else-if`

```html
<div id="app">
  <p v-if="score >= 90">优秀</p>
  <p v-else-if="score >= 60">及格</p>
  <p v-else>不及格</p>
</div>
```

注意：

- `v-else-if` 与 `v-else` 必须紧跟在对应的 `v-if` 后面，中间不能插入其他元素

---

## 5.2 `v-show`

`v-show` 的语义是“始终渲染，但用 CSS 控制显示/隐藏”。

```html
<div id="app">
  <p v-show="visible">这段文字可以隐藏</p>
  <button @click="visible = !visible">切换</button>
</div>
```

它的核心特点：

- 元素始终在 DOM 中
- 条件不满足时，Vue 会把元素设为 `display: none`

---

## 5.3 `v-if` vs `v-show` 的性能与适用场景

### 5.3.1 区别总结

- `v-if`：
  - 条件为 false 时，不创建 DOM
  - 频繁切换时开销更大（创建/销毁）

- `v-show`：
  - 永远创建 DOM
  - 频繁切换时开销更小（只是切换 `display`）

### 5.3.2 选型建议

- **优先用 `v-if` 的场景**：
  - 条件很少变化（例如权限控制、一次性判断）
  - 被隐藏的内容很大、很重（减少初次渲染成本）

- **优先用 `v-show` 的场景**：
  - 需要频繁切换显示/隐藏（例如 Tab 切换、展开收起）
  - 内容本身不算重，创建一次即可

你可以用一个直觉记忆：

- “很少出现”用 `v-if`
- “经常切换”用 `v-show`

---

## 5.4 `template` 配合条件渲染的使用方式

有时候你不想额外包一层 `div`，但又想一次性控制多个元素的显示/隐藏，可以用 `template`：

```html
<div id="app">
  <template v-if="isLogin">
    <p>欢迎回来</p>
    <p>你的消息：{{ messageCount }} 条</p>
  </template>

  <template v-else>
    <p>请先登录</p>
    <button @click="login">去登录</button>
  </template>
</div>
```

说明：

- `template` 本身不会渲染成真实 DOM
- 它只作为“逻辑容器”存在

---

## 5.5 常见坑与最佳实践

### 5.5.1 `v-if` 与 `v-for` 同时使用

虽然你可能会看到类似写法：

```html
<li v-for="item in list" v-if="item.visible">...</li>
```

但通常不推荐（可读性差、容易踩优先级坑、性能不够好）。

更推荐的写法：

- 先在计算属性里过滤
- 再渲染过滤后的结果

```html
<li v-for="item in visibleList" :key="item.id">{{ item.name }}</li>
```

---

## 本章小结

- `v-if` 是“真才渲染”，条件不满足时元素不在 DOM。
- `v-show` 是“始终渲染，用 CSS 控制隐藏”。
- 选择策略：少切换用 `v-if`，频繁切换用 `v-show`。
- `template` 可以在不增加额外 DOM 的情况下控制多段内容。
