### 5.3.2、配置vue-router
src/router/index.ts	配置路由器对象，并暴露

```javascript
import {createRouter,createWebHistory} from 'vue-router'
// 导入路由组件
import Index from '../views/index.vue'
import News from '../views/news.vue'
import Car from '../views/car.vue'
import Mine from '../views/mine.vue'
// 配置路由路径
const routes=[
    {
        path:"/index",
        component:Index
    },
    {
        path:"/news",
        component:News
    },
    {
        path:"/car",
        component:Car
    },
    {
        path:"/mine",
        component:Mine
    }
]
// 创建路由
const router =createRouter({
    // 不仅要有路由规则，还要设置监听URL的模式
    // 用于配置路由模式的选项。
    // 它指定了路由应该使用 HTML5 的历史记录 API (window.history) 来实现导航
    history:createWebHistory(),
    routes
})
// 对外暴露
export default router
```

## 5.2、 路由重定向&路由命名
### （1）、路由重定向
1. <font style="color:rgb(51, 51, 51);">作用：</font><u><font style="color:rgb(51, 51, 51);">将特定的路径，重新定向到已有路由。</font></u>
2. <font style="color:rgb(51, 51, 51);">具体编码</font>

router/index.ts

```javascript
const routes = [
    // 1、路由重定向
    {
        path: '/',//表示初始页面
        // redirect:'/index'//重定向到index组件
        redirect: { name: 'index' }
    },
    // 2、404页面,优先级最低，保底的，前面路由都没匹配上，就会走他
  // 相比vue2的 * ，vue3需要写成正则形式
    {
        path:"/:pathMatch(.*)*",
        component:NotFound
    }
  //·······
]
```

#### <font style="color:rgb(6, 6, 7);">为什么使用 </font>`/:pathMatch(.*)*`<font style="color:rgb(6, 6, 7);"> 而不是直接使用 </font>`*`<font style="color:rgb(6, 6, 7);">？</font>
1. **<font style="color:rgb(6, 6, 7);">匹配所有路径</font>**<font style="color:rgb(6, 6, 7);">：</font>`:pathMatch(.*)*`<font style="color:rgb(6, 6, 7);"> 是一个动态参数，结合了正则表达式 </font>`(.*)*`<font style="color:rgb(6, 6, 7);">，表示匹配任意字符（包括空字符），并且可以匹配多次。这确保了无论用户访问的是简单路径还是嵌套路径，都能被捕获</font><font style="color:rgb(6, 6, 7);">。</font>
2. **<font style="color:rgb(6, 6, 7);">捕获完整路径信息</font>**<font style="color:rgb(6, 6, 7);">：通过这种方式，匹配到的路径会被作为参数传递给组件。在组件中可以通过 </font>`this.$route.params.pathMatch`<font style="color:rgb(6, 6, 7);"> 获取完整的路径信息。这对于实现动态的 404 页面或日志记录等功能非常有用</font><font style="color:rgb(6, 6, 7);">。</font>
3. **<font style="color:rgb(6, 6, 7);">避免与其他路由冲突</font>**<font style="color:rgb(6, 6, 7);">：如果直接使用 </font>`*`<font style="color:rgb(6, 6, 7);">，可能会与其他路由冲突。而 </font>`/:pathMatch(.*)*`<font style="color:rgb(6, 6, 7);"> 是一个明确的通配符路由，应该始终放在所有其他路由的最后，以确保其他路由能够正确匹配。</font>

## 5.7、路由器工作模式
1. `history`模式

> 优点：`URL`更加美观，不带有`#`，更接近传统的网站`URL`。
>
> 缺点：后期项目上线，需要服务端配合处理路径问题，否则刷新会有`404`错误。
>

```javascript
const router = createRouter({
    history:createWebHistory(), //history模式
    /******/
})
```

2. `hash`模式

> 优点：兼容性更好，因为不需要服务器端处理路径。
>
> 缺点：`URL`带有`#`不太美观，且在`SEO`优化方面相对较差。
>

```javascript
const router = createRouter({
    history:createWebHashHistory(), //hash模式
    /******/
})
```

## 5.9、 路由守卫	
不同的守卫其实就是在不同的时机，不同的位置，添加鉴权（鉴别权限）代码

**需求：**并不是所有页面都可以随便访问的，如果想看购物车页面，需要先登录

<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2024/png/27167233/1734686129635-14e63cef-2fc7-421c-b394-f0940d9787d6.png)

### 5.9.1 全局前置守卫 	
1、书写位置：在创建好router之后，以及暴露router之前

2、执行时机：beforeEach()中传入一个回调函数callback，可以是普通函数或箭头函数都可以，在初始化时执行一次，然后每一次在切换任意组件之前都会被调用

:::info
router.beforeEach((to, from, next)=>{ // 翻译为：每次前（寓意：每一次切换任意路由之前执行。）

// to 去哪里(**<font style="color:rgb(255, 0, 0);">to.path、to.name</font>**)

// from 从哪来

// next 继续：调用next( )

})

:::

3、callback函数有3个参数：to form next

from参数：from是一个路由对象，表示从哪里来（从那个路由切过来的），起点

to参数：to也是一个路由对象，表示到哪里去，终点

next参数：是一个函数，调用这个函数之后，表示放行，可以继续向下走

4、给路由对象添加自定义属性的话，需要在路由对象的meta（路由元）中定义

如果路由组件较多。to.path会比较繁琐，可以考虑给需要鉴权的路由扩展一个布尔值属性，可以通过路由元来定义属性：**<font style="color:rgb(255, 0, 0);">meta:{isAuth : true}</font>**

```javascript
const router = createRouter({
    history: createWebHistory(),
    routes
})

// 1、全局前置路由守卫
// 需求：如果要去购物车页面，判断有没有登录，如果没有就不给看
router.beforeEach((to, from, next) => {
    let token = localStorage.getItem('token')
    // 不是去购物车页面，进行放行
    // 判断方式1 判断name
    // if (to.name == 'car') {
    // 判断方式2  判断path
    // if (to.path == '/car') {
    // 判断方式3  判断meta配置
    if (to.meta.isAuth) {

        if (token) {
            next()
        } else {
            alert('请登录') //给提示，让登录
            next({ path: '/login' }) //跳转登录页面
        }

    } else {
        next()
    }
    // console.log(to);
    // next()
})

export default router
```

### 5.9.2 全局后置守卫	
**需求： 每次切换完路由组件后，更换title标题栏，在meta中设置title属性属性值**

1、书写位置：router/index.js文件中拿到router对象，在创建router对象之后，暴露router对象之前写

2、执行时机：初始化执行一次，以后每一切换完任意路由之后

3、参数：只有to,from,没有next，因为没有必要了

:::info
router.afterEach((to, from)=>{ // 翻译为：每次后（寓意：每一次切换路由后执行。）

// 没有 next

document.title = to.meta.title // 通常使用后置守卫完成路由切换时title的切换。

})

:::

4、主要可以用来用户访问的路径，用来刻画用户画像

该功能也可以使用前置守卫实现：

```javascript
//需求： 每次切换完路由组件后，更换title标题栏，在meta中设置title属性属性值

// 参数：只有to,from,没有next，因为没有必要了
```

### 5.9.4 局部路由守卫之component守卫（组件守卫）	
书写位置：写在路由组件当中，也就是xxx.vue文件中

执行时机： beforeRouteEnter	进去路由组件之前执行

 		 beforeRouteLeave	离开路由组件之前执行 

注意：只有路由组件才有这两个钩子。

```javascript
<script>
export default {
    name: 'subject2',
    // 进去路由组件之前执行，此时组件实例还没有被创建
    beforeRouteEnter(to, from) {
        console.log(`进入路由组件之前“${to.meta.title}`)
    },
  // 在当前路由改变，但是该组件被复用时调用
    beforeRouteUpdate(to, from) {
        console.log(`进入路由组件之前“${to.meta.title}`);
    },
    // 离开路由组件之前执行
    beforeRouteLeave(to, from) {
        console.log(`离开路由组件之前“${from.meta.title}`);

    }
}
</script>
```
