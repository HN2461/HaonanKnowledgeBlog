# 开发必懂网络基础概念

开发时总看到 `127.0.0.1:3000`、`localhost`、`端口号` 这些东西，它们到底是什么？这篇文章帮你一次性搞清楚。

---

## 一、IP 地址：互联网上的"门牌号"

### 1.1 什么是 IP 地址

IP 地址是网络设备的唯一标识，就像现实中的门牌号。

- **IPv4**：4 组数字，每组 0-255，如 `192.168.1.1`
- **IPv6**：8 组十六进制，如 `2001:0db8:85a3::8a2e:0370:7334`

日常开发主要接触 IPv4。

### 1.2 几个特殊的 IP 地址

| IP 地址 | 含义 | 使用场景 |
|---------|------|----------|
| `127.0.0.1` | 本机回环地址 | 本地开发测试 |
| `0.0.0.0` | 所有网络接口 | 服务器监听所有 IP |
| `192.168.x.x` | 私网地址（局域网） | 家里/公司内网 |
| `10.x.x.x` | 私网地址（大企业） | 企业内网 |
| `172.16.x.x - 172.31.x.x` | 私网地址 | 企业内网 |

**私网 IP 不能直接访问互联网**，需要通过路由器 NAT 转换。

## 二、127.0.0.1 与 localhost：本机的两种叫法

### 2.1 127.0.0.1 是什么

`127.0.0.1` 是**本机回环地址**（Loopback Address）。

当你访问 `127.0.0.1` 时：
- 数据包**不离开本机**
- 不经过网卡
- 直接在操作系统内部"绕一圈"回来

就像给自己寄信，信直接送到自己手里。

### 2.2 localhost 是什么

`localhost` 是一个**主机名**，默认解析为 `127.0.0.1`。

这个映射定义在系统的 `hosts` 文件里：

```
# Windows: C:\Windows\System32\drivers\etc\hosts
# Linux/Mac: /etc/hosts

127.0.0.1    localhost
```

### 2.3 两者的区别

| 对比项 | 127.0.0.1 | localhost |
|--------|-----------|-----------|
| 类型 | IP 地址 | 主机名 |
| 是否需要解析 | 否，直接使用 | 是，需查询 hosts 或 DNS |
| 速度 | 更快（少一步解析） | 稍慢 |
| 使用场景 | 精确指定 IP | 更易读 |

**开发中两者基本等价**，`localhost` 更好记，`127.0.0.1` 更精确。

### 2.4 为什么开发要用 127.0.0.1

- 本地调试，不占用公网带宽
- 不需要联网也能测试
- 安全，外部无法访问

## 三、端口：一扇门对应一个服务

### 3.1 什么是端口

IP 地址找到"哪台电脑"，端口找到"电脑上的哪个服务"。

就像大楼地址（IP）+ 门牌号（端口）。

- 端口范围：0 - 65535
- 一个端口同一时间只能被一个程序监听

### 3.2 端口分类

| 范围 | 类型 | 说明 |
|------|------|------|
| 0 - 1023 | 知名端口 | 系统保留，需管理员权限 |
| 1024 - 49151 | 注册端口 | 应用程序常用 |
| 49152 - 65535 | 动态端口 | 系统临时分配 |

### 3.3 常用服务默认端口

| 服务 | 默认端口 | 说明 |
|------|----------|------|
| HTTP | 80 | 网页（不加密） |
| HTTPS | 443 | 网页（加密） |
| FTP | 20, 21 | 文件传输 |
| SSH | 22 | 远程登录（Linux） |
| Telnet | 23 | 远程登录（明文，不安全） |
| MySQL | 3306 | 数据库 |
| PostgreSQL | 5432 | 数据库 |
| Redis | 6379 | 缓存数据库 |
| MongoDB | 27017 | 数据库 |
| Node.js 开发服务器 | 3000, 5000, 8080 | 常见开发端口 |

### 3.4 开发中常见的端口写法

```
127.0.0.1:3000    # 本机 3000 端口
localhost:8080    # 本机 8080 端口
0.0.0.0:80        # 监听所有 IP 的 80 端口
:3000             # 简写，通常等同于 0.0.0.0:3000
```

### 3.5 为什么有时候要加端口号

- HTTP 默认 80，访问 `http://example.com` 自动用 80 端口
- 如果服务跑在 3000 端口，必须写成 `http://example.com:3000`
- HTTPS 默认 443，`https://example.com` 自动用 443

## 四、DNS：域名到 IP 的"翻译官"

### 4.1 为什么需要 DNS

人记不住 IP 地址，但能记住域名。

- 域名：`www.baidu.com`
- IP 地址：`110.242.68.66`

DNS（Domain Name System）负责把域名翻译成 IP 地址。

### 4.2 DNS 解析过程（简化版）

```
浏览器输入 www.baidu.com
    ↓
检查浏览器缓存
    ↓
检查系统缓存（hosts 文件）
    ↓
询问本地 DNS 服务器（运营商提供）
    ↓
本地 DNS 没有就去问根 DNS → 顶级 DNS → 权威 DNS
    ↓
返回 IP 地址：110.242.68.66
    ↓
浏览器用 IP 建立连接
```

### 4.3 开发中如何利用 hosts 文件

可以**手动指定域名解析**，跳过 DNS：

```
# C:\Windows\System32\drivers\etc\hosts

127.0.0.1    myproject.test
192.168.1.100    test-server.local
```

然后访问 `http://myproject.test` 就会指向本机。

**用途**：
- 本地开发多域名测试
- 屏蔽广告（把广告域名指向 127.0.0.1）
- 内网测试环境

## 五、常见开发场景对照

### 5.1 启动本地开发服务器

```bash
npm run dev
# 输出：Local: http://localhost:5173/
```

这意味着：
- 服务监听在本机 `127.0.0.1` 的 `5173` 端口
- 浏览器访问 `http://localhost:5173` 即可

### 5.2 让局域网其他设备访问

```bash
vite --host 0.0.0.0
# 或在 vite.config.js 设置 server.host: '0.0.0.0'
```

- `0.0.0.0` 表示监听所有网络接口
- 同一局域网下，其他设备用你电脑的局域网 IP（如 `192.168.1.50:5173`）访问

### 5.3 为什么有时候访问不了

检查清单：

1. 防火墙是否放行该端口
2. 服务是否真的在监听（命令行看输出）
3. IP 和端口是否正确
4. 用 `127.0.0.1` 还是 `0.0.0.0`（局域网访问必须后者）

## 六、速查表

| 场景 | 写法 | 含义 |
|------|------|------|
| 本地开发 | `http://localhost:3000` | 本机 3000 端口 |
| 本地开发 | `http://127.0.0.1:3000` | 同上 |
| 局域网访问 | `http://192.168.1.50:3000` | 局域网某台机器 |
| 监听所有 IP | `0.0.0.0:3000` | 本机 + 局域网都可访问 |
| 线上网站 | `https://example.com` | 自动用 443 端口 |
| 自定义域名测试 | `hosts` 文件 | 手动绑定域名到 IP |
