# Windows笔记本WiFi图标消失/WLAN服务无法自启

## 笔记前言

### 问题精准定义

- 核心现象：笔记本WiFi图标从任务栏消失，无法搜索/连接无线网络；手动启动`WLAN AutoConfig`服务后WiFi立即恢复，但**电脑重启后服务再次停止、WiFi再次消失**
- 历史情况：问题已出现2次，首次通过重置系统解决；本次通过「服务配置修复+管理员权限重置网络栈」根治，无需重装系统
- 根因定位：Windows系统Winsock目录/IP协议栈底层损坏、WLAN自动配置服务的自启链路异常，导致开机时服务无法正常拉起，非硬件故障

### 操作总原则

下次复现问题时，**严格按照从简单到复杂、从高频有效到低频兜底的顺序执行**，不要跳步；每完成一个阶段，重启电脑验证是否修复，修复成功即可停止，无需执行后续步骤。

---

## 第一阶段：1分钟应急恢复（先把网络连上，再做根治）

> 适用场景：WiFi突然消失，急需联网，先临时恢复，再做后续根治

1.  按下快捷键 `Win + R`，弹出运行窗口
2.  输入 `services.msc`，按下回车，打开「服务」管理器
3.  在服务列表中，找到 **WLAN AutoConfig**（也叫WLAN自动配置）
4.  右键该服务，点击「启动」；若启动按钮灰色，先点击「属性」，将「启动类型」改为「手动」，点击「应用」后再启动服务
5.  启动完成后，任务栏WiFi图标会立即出现，可正常连接网络

> 【我的实战备注】：此步骤仅临时生效，重启电脑后问题会复现，必须完成后续根治步骤

---

## 第二阶段：必做核心根治步骤（本次最终生效步骤，优先执行）

> 【我的实战验证】：本次问题最终由此步骤解决，核心生效点是**管理员权限执行网络栈重置命令**，之前执行失败是因为没有用管理员权限

### 步骤1：以管理员身份打开终端

1.  按下 `Win + S` 打开系统搜索框
2.  输入 `cmd`（或`PowerShell`，两者均可）
3.  在搜索结果中，右键「命令提示符」，选择 **以管理员身份运行**
4.  确认窗口左上角显示「管理员: 命令提示符」，再进行后续操作

### 步骤2：执行核心网络重置命令（2条，必做）

在管理员终端中，逐条输入以下命令，每输完一条按下回车，等待执行完成再输入下一条：

```cmd
netsh winsock reset
```

- 执行成功提示：`成功地重置 Winsock 目录。你必须重新启动计算机才能完成重置。`

```cmd
netsh int ip reset
```

- 执行说明：若出现少量「拒绝访问」提示，属于正常系统权限边界，不影响核心修复，只要大部分步骤显示「完成」即可
- 执行成功提示：`重新启动计算机来完成此操作。`

### 步骤3：完成重置并验证

1.  两条命令执行完成后，**必须正常重启电脑**，重置才能生效
2.  重启后验证2个核心点：
    - 任务栏WiFi图标是否自动出现，能否正常连网
    - 打开`services.msc`，查看`WLAN AutoConfig`服务是否为「启动类型：自动 + 服务状态：正在运行」
3.  若两项均正常，问题已根治，无需执行后续步骤

---

## 第三阶段：进阶排查修复（核心步骤无效时执行，本次已验证）

> 【我的实战备注】：本次已执行本阶段大部分步骤，虽未直接生效，但属于修复链路的关键补充，下次核心步骤无效时，按顺序执行

### 步骤1：强制锁定WLAN服务自动启动

1.  以管理员身份打开CMD终端
2.  执行以下命令，强制修改服务自启配置并立即启动服务：
    ```cmd
    sc config WlanSvc start= auto & sc start WlanSvc
    ```
3.  执行成功提示：2次`SUCCESS`，重启电脑验证是否生效

### 步骤2：WLAN服务依赖项全量修复

> 【我的实战备注】：本次检查大部分依赖服务正常，个别服务缺失不影响修复，无需强行处理

1.  按下 `Win + R`，输入 `services.msc` 打开服务管理器
2.  找到以下列表中的服务，逐个检查并修复，确保核心服务状态为「启动类型：自动 + 服务状态：正在运行」
    | 服务全称 | 启动类型要求 | 服务状态要求 | 核心程度 |
    |---|---|---|---|
    | Remote Procedure Call (RPC) | 自动 | 正在运行 | 必须满足 |
    | DCOM Server Process Launcher | 自动 | 正在运行 | 必须满足 |
    | RPC Endpoint Mapper | 自动 | 正在运行 | 必须满足 |
    | DHCP Client | 自动 | 正在运行 | 必须满足 |
    | Network Store Interface Service | 自动 | 正在运行 | 必须满足 |
    | Network Location Awareness | 自动 | 正在运行 | 建议满足 |
    | Windows Connection Manager | 自动 | 正在运行 | 建议满足 |
    | NDIS Usermode I/O Protocol | 手动 | 正在运行 | 建议满足 |
3.  异常服务修复方式：双击服务 → 启动类型改为要求值 → 点击「启动」→ 应用 → 确定
4.  也可直接用管理员CMD执行以下批量修复命令，逐条执行：
    ```cmd
    sc config RpcSs start= auto
    sc start RpcSs
    sc config DcomLaunch start= auto
    sc start DcomLaunch
    sc config Dhcp start= auto
    sc start Dhcp
    sc config NSI start= auto
    sc start NSI
    sc config nlasvc start= auto
    sc start nlasvc
    sc config Wcmsvc start= auto
    sc start Wcmsvc
    sc config Ndisuio start= demand
    sc start Ndisuio
    sc config WlanSvc start= auto
    sc start WlanSvc
    ```
5.  全部执行完成后，重启电脑验证

### 步骤3：注册表配置检查与修复

> 【我的实战备注】：本次检查注册表`Start`数值已为2，无需修改，若下次数值异常可按此步骤修复

1.  按下 `Win + R`，输入 `regedit`，按下回车打开注册表编辑器
2.  在左侧地址栏，直接粘贴以下路径，按下回车快速定位：
    ```
    HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\WlanSvc
    ```
3.  在右侧列表中，找到名为 **Start** 的项
4.  双击该项，查看「数值数据」：
    - 数值`2` = 自动启动（正常状态，无需修改）
    - 数值`3` = 手动启动
    - 数值`4` = 禁用
5.  若数值不是2，改为`2`，基数选择「十六进制」，点击「确定」
6.  关闭注册表，重启电脑验证

### 步骤4：系统文件完整性修复

1.  以管理员身份打开CMD终端
2.  执行以下命令，扫描并修复损坏的系统文件：
    ```cmd
    sfc /scannow
    ```
3.  等待扫描完成，提示「Windows 资源保护找到了损坏文件并成功修复」即为完成
4.  重启电脑验证

### 步骤5：全量网络配置重置

1.  以管理员身份打开CMD终端，逐条执行以下命令：
    ```cmd
    ipconfig /release
    ipconfig /renew
    ipconfig /flushdns
    ```
2.  执行完成后重启电脑验证

## 第四阶段：第三方软件拦截排查（以上步骤全无效时执行）

> 【我的实战备注】：本次因软件较多未执行，若下次所有步骤都无效，优先排查此项

1.  问题根源：360安全卫士、电脑管家、火绒安全、驱动精灵、各类优化软件、VPN软件、虚拟机软件，会在开机时禁用WLAN服务，导致自启失效
2.  排查方式：
    - 临时完全退出所有上述软件，部分软件需右键托盘图标选择「完全退出/关闭防护」
    - 重启电脑，查看WLAN服务是否自动启动、WiFi是否正常
    - 若恢复正常，逐个重新开启/安装软件，找到导致拦截的程序
    - 在该软件的「启动项管理/服务管理/优化记录」中，找到`WlanSvc`/`WLAN AutoConfig`服务，设置为「允许开机启动/恢复优化」

## 第五阶段：终极兜底方案（100%生效，以上全无效时用）

> 【我的实战备注】：本次未使用，逻辑完全可行，无需重装系统，彻底杜绝服务不启动问题

### 方案说明

通过创建开机自动执行的脚本，电脑开机时强制执行启动WLAN服务的命令，无论服务自启链路是否异常，都能保证WiFi正常启动。

### 详细操作步骤

1.  在电脑桌面空白处，右键 → 新建 → 文本文档
2.  打开新建的文本文档，粘贴以下一行内容：
    ```cmd
    net start WlanSvc
    ```
3.  点击左上角「文件」→ 「保存」，关闭文档
4.  选中文档，按下`F2`重命名，将文件名改为 **启动WiFi.bat**
    - 关键：必须把文件后缀`.txt`改成`.bat`，若看不到后缀，打开文件资源管理器顶部「查看」，勾选「文件扩展名」
5.  按下 `Win + R`，输入以下内容，按下回车，打开开机启动文件夹：
    ```
    shell:startup
    ```
6.  将改好名的`启动WiFi.bat`文件，拖入这个开机启动文件夹中
7.  重启电脑验证，开机后WiFi会自动出现，无需任何手动操作

## 验证问题是否根治的标准

连续3次正常重启电脑，每次重启后都满足以下2个条件，即为彻底根治：

1.  任务栏WiFi图标自动出现，无需手动操作，可正常搜索并连接无线网络
2.  打开`services.msc`，`WLAN AutoConfig`服务状态为「启动类型：自动 + 服务状态：正在运行」

## 专属避坑提醒

1.  ⚠️ 所有网络修复命令，**必须用管理员身份**运行终端，否则会出现「拒绝访问」，导致修复不完整、不生效
2.  ⚠️ 所有网络重置命令执行完成后，**必须重启电脑**，否则配置不会生效
3.  ⚠️ 找不到Fn+WiFi快捷键，可直接跳过，不影响任何核心修复步骤
4.  ⚠️ 不要盲目重置系统，99%的情况通过以上步骤都能解决，重置系统是最后选择
5.  ⚠️ 尽量不要用优化软件禁用系统服务，这是导致此问题反复出现的最常见原因
