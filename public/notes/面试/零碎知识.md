## 1. Vue2 和 Vue3 的区别

### 1.1 核心区别

- 响应式原理：Vue2 基于 `Object.defineProperty`，Vue3 基于 `Proxy`
- API 风格：Vue2 主要是选项式 API，Vue3 同时强化了组合式 API
- TypeScript 支持：Vue3 对 TS 更友好
- 性能：Vue3 在编译和运行时都做了更多优化
- 全局 API 与部分语法：Vue3 对一些旧写法做了调整或移除

### 1.2 Vue2 的缺点

1. 响应式系统基于 `Object.defineProperty`，需要递归遍历对象属性，初始化开销较大
2. 无法监听新增/删除属性，需要借助 `Vue.set`/`Vue.delete` 手动触发更新
3. 无法直接监听数组索引变化和 `length` 属性修改，需要重写数组方法
4. Options API 容易让同一块业务逻辑分散在不同选项中，代码组织不够灵活
5. 深层对象需要递归劫持，初始化性能开销较大
6. TypeScript 支持相对薄弱，类型推导能力有限

### 1.3 Vue3 的优势

1. 使用 `Proxy` 重构响应式系统，支持动态增删属性，也能监听 `Map`、`Set`
2. 数组索引变化可以被直接追踪
3. 更适合组合式封装，逻辑复用更自然
4. 更好地支持 TypeScript
5. 编译优化更多，例如静态提升、`Patch Flag`、Block Tree
6. 更容易做 Tree Shaking，减小打包体积

### 1.4 Vue2 和 Vue3 的响应式原理

- Vue2：通过 `Object.defineProperty` 劫持对象属性的 `getter` 和 `setter`
- Vue3：通过 `Proxy` 代理整个对象，能力更完整

### 1.5 数据劫持

数据劫持是 Vue 响应式的底层思想之一。Vue2 会把 `data` 中的数据通过 `Object.defineProperty` 转成带 `getter` 和 `setter` 的属性。当数据被读取或修改时，就能在中间插入依赖收集与视图更新逻辑，所以叫“劫持”。

更通用地说：

- Vue2 主要依赖 `Object.defineProperty`
- Vue3 主要依赖 `Proxy`
- 本质都是在数据变化时自动通知视图更新

## 2. 虚拟 DOM、Diff 和 MVVM

### 2.1 什么是虚拟 DOM

虚拟 DOM 是对真实 DOM 结构的 JavaScript 对象描述。框架在数据变化后，先生成新的虚拟 DOM，再对比新旧差异，最后把最小更新同步到真实 DOM。

### 2.2 Diff 算法原理

Diff 算法用于比较新旧虚拟 DOM 的差异，以更高效地更新真实 DOM。

核心思路：

- 同层比较，不做复杂的跨层级对比
- 优先借助 `key` 判断节点是否可复用
- 只更新变化的部分，不整棵树重建

### 2.3 Diff 算法特征

- Key 优先匹配
- 同层级比较
- 最小化 DOM 操作
- 通过同层比较和 key 优化，将传统树对比的 `O(n³)` 复杂度降低到 `O(n)`

### 2.4 Vue2 和 Vue3 的 Diff 差异

- Vue2 常见描述是双端比较策略
- Vue3 对动态子节点处理做了进一步优化
- Vue3 会结合最长递增子序列（LIS）减少无意义移动
- Vue3 还会结合静态提升、Patch Flag、Block Tree 缩小比较范围

### 2.5 为什么 `v-for` 不建议直接用 `index` 作为 `key`

- 列表顺序变动时，容易错误复用旧节点
- 可能导致组件内部状态错乱
- 会带来额外更新，影响性能和正确性

### 2.6 `v-if` 和 `v-for` 的区别

- `v-if` 是条件渲染，决定某个节点是否创建
- `v-for` 是循环渲染，决定渲染多少个节点
- 一般不建议把两者直接写在同一个元素上

### 2.7 MVVM 架构

MVVM 全称是 `Model - View - ViewModel`。

- `Model`：数据层，负责业务数据
- `View`：视图层，负责展示页面
- `ViewModel`：连接 View 和 Model，实现数据绑定和交互桥接

Vue 常被认为是典型的 MVVM 思想实践。

### 2.8 虚拟 DOM 简单示例

下面这个例子展示了如何用一个普通对象描述 DOM 结构，并递归渲染成真实 DOM：

```javascript
const vnode = {
  tag: 'div',
  attrs: { id: 'app', class: 'container' },
  children: [
    { tag: 'h1', attrs: {}, children: ['Hello'] },
    { tag: 'p', attrs: {}, children: ['Virtual DOM'] }
  ]
}

function render(vnode) {
  if (typeof vnode === 'string') {
    return document.createTextNode(vnode)
  }

  const el = document.createElement(vnode.tag)

  for (let key in vnode.attrs) {
    el.setAttribute(key, vnode.attrs[key])
  }

  vnode.children.forEach(child => {
    el.appendChild(render(child))
  })

  return el
}

document.body.appendChild(render(vnode))
```

## 4. Vuex 中如何调用 getters

### 4.1 直接通过 `this.$store.getters` 访问

在组件的方法或计算属性中，可以直接通过 `this.$store.getters` 获取 getter 的返回值。

```javascript
export default {
  computed: {
    userDetail() {
      return this.$store.getters.getUserDetail
    }
  },
  created() {
    console.log(this.userDetail)
  }
}
```

也可以像这样直接使用：

```javascript
this.$store.getters.xxx
```

### 4.2 使用 `mapGetters` 辅助函数

`mapGetters` 可以把 store 中的 getters 映射为当前组件的计算属性，使用起来更简洁。

```javascript
import { mapGetters } from 'vuex'

export default {
  computed: {
    ...mapGetters(['someGetter'])
  }
}
```

上面例子中，`someGetter` 是 Vuex 中定义的 getter 名称，映射后可以直接通过 `this.someGetter` 访问。

### 4.2.1 带命名空间的 getter 调用

如果使用了模块化并开启了命名空间（`namespaced: true`），需要指定模块路径：

```javascript
import { mapGetters } from 'vuex'

export default {
  computed: {
    // 方式一：使用 mapGetters 指定命名空间
    ...mapGetters('user', ['userInfo', 'isLogin']),
    
    // 方式二：直接访问
    userInfo() {
      return this.$store.getters['user/userInfo']
    }
  }
}
```

### 4.3 用户列表示例

```vue
<template>
  <div>
    <ul>
      <li v-for="user in users" :key="user.id">
        {{ user.name }}

<script>
```

### 4.4 用户详情示例

```vue
<template>
  <div>
```

### 4.5 参数化 getters

getter 也可以返回一个函数，这样就能接收参数，适合做筛选类逻辑。

```javascript
// store.js
getters: {
  todosByStatus: (state) => (status) => {
    return state.todos.filter(todo => todo.done === status)
  }
}

// 组件中使用
computed: {
  doneTodos() {
    return this.$store.getters.todosByStatus(true)
  },
  pendingTodos() {
    return this.$store.getters.todosByStatus(false)
  }
}
```

## 5. Vue 生命周期与路由

### 5.1 Vue2 生命周期

- 创建阶段：`beforeCreate`、`created`
- 挂载阶段：`beforeMount`、`mounted`
- 更新阶段：`beforeUpdate`、`updated`
- 销毁阶段：`beforeDestroy`、`destroyed`
- 其他：`activated`、`deactivated`、`nextTick`

如果对应到 Vue3：

- `created` 前后的初始化逻辑很多会前移到 `setup`
- `beforeDestroy`、`destroyed` 变成 `onBeforeUnmount`、`onUnmounted`

### 5.2 父子组件生命周期执行顺序

父组件和子组件在挂载阶段的常见执行顺序是：

父 `beforeCreate` -> 父 `created` -> 父 `beforeMount` -> 子 `beforeCreate` -> 子 `created` -> 子 `beforeMount` -> 子 `mounted` -> 父 `mounted`

更新阶段通常是：

父 `beforeUpdate` -> 子 `beforeUpdate` -> 子 `updated` -> 父 `updated`

销毁阶段通常是：

父 `beforeDestroy` -> 子 `beforeDestroy` -> 子 `destroyed` -> 父 `destroyed`

### 5.3 路由守卫

- 全局前置守卫：`router.beforeEach`
- 全局解析守卫：`router.beforeResolve`
- 全局后置守卫：`router.afterEach`
- 路由独享守卫：`beforeEnter`
- 组件内守卫：`beforeRouteEnter`、`beforeRouteUpdate`、`beforeRouteLeave`

### 5.4 路由的两种传参方式

- `params`：通常配合 `name` 使用，刷新后可能丢失
- `query`：可以配合 `path` 或 `name` 使用，参数会展示在 URL 上

## 7. WebSocket、长轮询与网络基础

### 7.1 WebSocket 是什么

WebSocket 是一种支持客户端和服务端双向通信的协议。不同于 HTTP 一问一答的请求响应模式，WebSocket 建立连接后，服务端也可以主动推送消息给客户端。

### 7.2 WebSocket 常用方法

- `onopen`
- `onmessage`
- `onclose`
- `onerror`
- `send`
- `close`

### 7.3 WebSocket 心跳机制

为了保证连接可用，客户端通常会每隔一段时间发送一个心跳包，服务器收到后返回响应。若一定时间内没有收到响应，通常会认为连接已断开，并触发重连逻辑。

#### 心跳机制实现示例

```javascript
let heartbeatTimer = null
let reconnectTimer = null
let missedHeartbeats = 0
const MAX_MISSED_HEARTBEATS = 3

const ws = new WebSocket('ws://example.com')

ws.onopen = () => {
  console.log('WebSocket 连接成功')
  missedHeartbeats = 0
  
  // 每 30 秒发送一次心跳
  heartbeatTimer = setInterval(() => {
    if (ws.readyState === WebSocket.OPEN) {
      ws.send(JSON.stringify({ type: 'ping' }))
      missedHeartbeats++
      
      // 如果连续多次未收到响应，认为连接断开
      if (missedHeartbeats >= MAX_MISSED_HEARTBEATS) {
        console.log('心跳超时，准备重连')
        ws.close()
      }
    }
  }, 30000)
}

ws.onmessage = (event) => {
  const data = JSON.parse(event.data)
  
  if (data.type === 'pong') {
    // 收到心跳响应，重置计数器
    missedHeartbeats = 0
    console.log('心跳响应正常')
  } else {
    // 处理业务消息
    console.log('收到消息:', data)
  }
}

ws.onclose = () => {
  console.log('WebSocket 连接关闭')
  clearInterval(heartbeatTimer)
  
  // 5 秒后尝试重连
  reconnectTimer = setTimeout(() => {
    console.log('尝试重新连接...')
    // 这里可以递归调用连接函数
  }, 5000)
}

ws.onerror = (error) => {
  console.error('WebSocket 错误:', error)
}
```

### 7.4 单向通信方案

如果不使用 WebSocket，但需要服务端有数据时再返回，可以考虑 HTTP 长轮询。

基本思路：

1. 客户端发起请求
2. 服务端暂时挂起连接，直到有数据可返回
3. 客户端拿到结果后立刻再次发起请求

### 7.5 浏览器跨域问题

当协议、域名、端口有一个不同，就会产生跨域问题。常见解决方案有：

- 后端设置 CORS
- 前端开发环境配置代理
- 使用 Nginx 反向代理
- JSONP

### 7.6 浏览器三次握手和四次挥手

#### 三次握手

1. 客户端发送 SYN，请求建立连接
2. 服务端返回 SYN + ACK，表示收到并同意连接
3. 客户端再发送 ACK，连接建立完成

#### 四次挥手

1. 客户端发送 FIN，表示要关闭连接
2. 服务端返回 ACK，确认收到
3. 服务端处理完剩余数据后，再发送 FIN
4. 客户端返回 ACK，连接彻底关闭

### 7.7 前端防止泄密的常见措施

- 使用 `HTTPS`
- 使用 `Token` 做鉴权
- 避免把敏感信息长时间存到 `localStorage` 或 `sessionStorage`
- 代码混淆、资源压缩、文件名哈希

## 8. JavaScript 基础高频题

### 8.1 宏任务和微任务

JavaScript 运行机制基于事件循环。

- 宏任务：`setTimeout`、`setInterval`、I/O、`requestAnimationFrame`
- 微任务：`Promise.then`、`catch`、`finally`、`async/await`

微任务会在当前宏任务执行结束后、下一个宏任务开始前优先执行。

### 8.6 闭包及缺点

闭包指函数可以访问其外部函数作用域中的变量。

**优点**：
- 可以延长变量生命周期
- 实现数据封装和私有变量
- 适合模块化开发

**缺点**：
- 如果闭包引用的外部变量一直不被释放（如事件监听器未移除、定时器未清除），会导致内存无法回收，形成内存泄漏
- 过度使用闭包会增加内存占用

### 8.7 原型和原型链

每个函数都有一个 `prototype` 属性，对应一个原型对象；对象在查找属性时，如果自身没有，就会沿着原型继续向上查找，这个查找链路就叫原型链。

### 8.8 深拷贝和浅拷贝

- 浅拷贝：只复制第一层，更深层对象依旧共享引用
- 深拷贝：递归复制各层数据，新旧对象互不影响

常见方式：

- 浅拷贝：`slice()`、`concat()`、`Object.assign()`
- 深拷贝：`JSON.parse(JSON.stringify(obj))`、递归、自定义方法、`_.cloneDeep`

### 8.9 防抖和节流

#### 防抖

```javascript
function debounce(fn, delay) {
  let timer = null
  return function(...args) {
    clearTimeout(timer)
    timer = setTimeout(() => {
      fn.apply(this, args)
    }, delay)
  }
}
```

#### 节流

```javascript
function throttle(fn, interval) {
  let lastTime = 0
  return function(...args) {
    const now = Date.now()
    if (now - lastTime >= interval) {
      lastTime = now
      fn.apply(this, args)
    }
  }
}
```

## 9. CSS、布局与浏览器渲染

### 9.1 Flexbox 中 `flex: 1` 的含义

`flex: 1` 常被理解为以下三个属性的简写：

- `flex-grow: 1`
- `flex-shrink: 1`
- `flex-basis: 0`

它的常见效果是：

- 尽量占据剩余空间
- 空间不足时允许收缩
- 更偏向按比例分配而不是优先看内容本身大小

### 9.2 `flex: 1` 和 `flex: auto` 的区别

- `flex: 1` 等价于 `flex: 1 1 0`
- `flex: auto` 等价于 `flex: 1 1 auto`

区别在于：

- `flex: 1` 更强调平均分配
- `flex: auto` 会更参考内容初始尺寸

### 9.3 如何将 `1px` 转成 `0.5px`

可以利用 CSS 的缩放能力模拟细线：

```css
.half-px {
  background-color: #000;
  height: 1px;
  transform: scale(0.5);
  transform-origin: 0 0;
}
```

### 9.4 回流和重绘

- 回流：元素尺寸、位置、结构变化引起布局重新计算
- 重绘：只改变外观，不影响布局

结论：

- 回流一定会触发重绘
- 重绘不一定会触发回流
- 回流的性能开销通常更大

### 10.2 ECharts 柱状图倒置的两种方法

常见有两种思路：

- **方法一（推荐）**：通过 `yAxis.inverse = true` 直接反转 Y 轴
- **方法二**：通过 `reverse()` 调整数据顺序，实现倒序展示

#### 方法一：Y 轴倒置（推荐）

这种方式最简单直接，只需要设置 `yAxis.inverse = true`：

```javascript
var chartDom = document.getElementById('main')
var myChart = echarts.init(chartDom)
var option = {
  xAxis: {
    type: 'category',
    data: ['Mon', 'Tue', 'Wed', 'Thu', 'Fri', 'Sat', 'Sun']
  },
  yAxis: {
    type: 'value',
    inverse: true  // 关键配置：Y 轴倒置
  },
  series: [{
    data: [120, 200, 150, 80, 70, 110, 130],
    type: 'bar'
  }]
}

myChart.setOption(option)
```

#### 方法二：调整数据顺序

通过反转数据数组来实现倒序展示：

```javascript
var chartDom = document.getElementById('main')
var myChart = echarts.init(chartDom)
var originalData = ['Mon', 'Tue', 'Wed', 'Thu', 'Fri', 'Sat', 'Sun']
var originalValues = [120, 200, 150, 80, 70, 110, 130]

// 反转数据
var reversedData = originalData.slice().reverse()
var reversedValues = originalValues.slice().reverse()

var option = {
  xAxis: {
    type: 'category',
    data: reversedData
  },
  yAxis: {
    type: 'value'
  },
  series: [{
    data: reversedValues,
    type: 'bar'
  }]
}

myChart.setOption(option)
```

#### 综合示例：X 轴置顶 + Y 轴倒置

如果需要横向柱状图并且倒置，可以这样配置：

```javascript
var chartDom = document.getElementById('main')
var myChart = echarts.init(chartDom)
var option = {
  xAxis: {
    type: 'value',
    position: 'top'  // X 轴置顶
  },
  yAxis: {
    type: 'category',
    data: ['Mon', 'Tue', 'Wed', 'Thu', 'Fri', 'Sat', 'Sun'],
    inverse: true  // Y 轴倒置
  },
  series: [{
    type: 'bar',
    data: [120, 200, 150, 80, 70, 110, 130]
  }]
}

myChart.setOption(option)
```
