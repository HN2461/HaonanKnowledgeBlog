# 前端面试知识点整理
## 1. Vue2 和 Vue3 的区别
+ **响应式原理**：Vue2 基于 `Object.defineProperty`，Vue3 基于 `Proxy`。
+ **API 风格**：Vue2 是选项式 API（Options API），Vue3 是组合式 API（Composition API）。
+ **TypeScript 支持**：Vue3 更好地支持 TypeScript。
+ **源码**：Vue3 源码升级，占用内存更小。
+ **内置组件和生命周期钩子**：Vue3 有新的内置组件和调整后的生命周期钩子。
+ **修饰符等移除**：Vue3 移除 `native` 修饰符，移除 `keyCode` 支持作为 `v-on` 的修饰符 ，自定义指令等也有不同，过度类名更改（增加了 `from` ），全局 API 转移。

## 2. Vue2 的缺点和 Vue3 的优势
### Vue2 的缺点
1. 占用内存高，虚拟 DOM 全量对比导致不必要的组件重渲染，相对 Vue3 反应速度慢。
2. Options API 导致逻辑分散：同一功能的代码被拆分到 `data`、`methods`、`mounted` 等选项中，维护复杂。
3. 动态属性监听缺失：无法自动检测对象属性的新增/删除，必须使用 `Vue.set/Vue.delete`；数组监听缺陷：直接通过索引修改数组元素（如 `arr[0] = 1` ）或改变长度（`arr.length = 0` ）不会触发更新；深层对象性能损耗：深度监听嵌套对象需递归遍历，影响性能。 
4. TypeScript 支持薄弱。

### Vue3 的优势
1. 响应式系统重构（`Proxy`）：支持动态增删属性、监听 `Map/Set` 等复杂结构；数组索引修改可直接触发更新，无需特殊处理。 
2. 源码升级，更好地支持 TypeScript。 
3. 性能全面优化：
    - 编译优化：静态节点提升（`HoistStatic`）减少重复创建；`Patch Flag` 标记动态节点，加速 Diff 过程 。
    - Tree - shaking：未使用的模块（如 `v - model` 指令）不打包，减小产物体积。

## 8. v - for 的 key 值作用
给元素添加唯一标识符，用于提高 Vue 渲染性能。当 `data` 变化，Vue 用 `diff` 算法对比新旧虚拟 DOM，`key` 值相同考虑复用元素，不同则强制更新元素。一般设 `key` 为 `id` ，保证更新时最大限度复用相同 `key` 值元素。 

## 9. 虚拟 dom 转化为真实的 dom
虚拟 DOM 到真实 DOM 的转换，即根据虚拟节点结构，递归创建元素、设置属性并添加到页面。示例如下：

```javascript
const vnode = {
    tag: 'div',
    attrs: { id: 'app', class: 'container' },
    children: [
        { tag: 'h1', attrs: {}, children: ['Hello'] },
        { tag: 'p', attrs: {}, children: ['Virtual DOM'] }
    ]
};

function render(vnode) {
    if (typeof vnode === 'string') {
        return document.createTextNode(vnode); // 处理文本节点
    }
    const el = document.createElement(vnode.tag); // 创建元素节点
    // 设置属性
    for (let key in vnode.attrs) {
        el.setAttribute(key, vnode.attrs[key]);
    }
    // 递归处理子节点
    vnode.children.forEach(child => {
        el.appendChild(render(child));
    });
    return el;
}

// 使用
document.body.appendChild(render(vnode));
```

## 11. 宏任务和微任务
JavaScript 的执行是单线程的事件循环机制（`Event Loop`），任务分两类：

+ **宏任务（Macro Task）**：主线程中执行的任务，每次事件循环只能执行一个宏任务。包括 `setTimeout`、`setInterval`、`setImmediate`（Node）、`requestAnimationFrame`、I/O 操作等 。
+ **微任务（Micro Task）**：当前宏任务执行完后、渲染前执行的任务。包括 `Promise.then / catch / finally`、`async/await` 等。微任务优先级高于宏任务。

## 13. Diff 算法及特征
Diff 算法是比较新旧虚拟 DOM（`VNode`）差异，高效更新真实 DOM 的算法，避免直接操作整个 DOM 树，提升性能。

## 14. v - if 和 v - for 的区别
`v - if` 是条件渲染，控制是否创建元素，用于条件判断；`v - for` 是循环渲染，控制创建多少元素，用于列表渲染。`v - for` 优先级比 `v - if` 高，不建议混用，会造成页面卡顿。

## 18. MVVM 架构
MVVM 全称是 `Model - View - ViewModel`，实现 Vue 数据双向绑定。分别是：

+ **Model（模型）**：数据层，存储数据和业务逻辑。 
+ **View（视图）**：UI 层，展示数据给用户，响应用户交互。 
+ **ViewModel（视图模型）**：连接 `Model` 和 `View` 的桥梁，处理业务逻辑，实现数据双向绑定。当用户操作 `View` 变化时，同步到 `ViewModel` 处理并更新 `Model`；若服务端数据变化，同步到 `ViewModel` 处理并更新 `View` 展现给用户。

## 22. 防抖和节流
### 防抖（Debounce）
将多次执行变为最后一次执行，常用于搜索输入、窗口缩放等场景。

```javascript
function debounce(fn, delay) {
    let timer = null;
    return function(...args) {
        clearTimeout(timer);
        timer = setTimeout(() => {
            fn.apply(this, args);
        }, delay);
    };
}
```

### 节流（Throttle）
规定时间内只允许执行一次函数，适合滚动监听、按钮频繁点击等场景。

```javascript
function throttle(fn, interval) {
    let lastTime = 0;
    return function(...args) {
        const now = Date.now();
        if (now - lastTime >= interval) {
            lastTime = now;
            fn.apply(this, args);
        }
    };
}
```

## 32. 小程序的支付流程
1. **前端发起支付请求**：用户点击支付后，前端通过 `uni.request` 向后端发起统一下单请求，传入 `openid`、订单号和金额等信息。 
2. **后端调用微信统一下单接口**：后端接收到请求后，调用微信的「统一下单 API」，使用商户号、`AppID`、签名等参数生成 `prepay_id`，再将所需支付参数（如 `timeStamp`、`nonceStr`、`package`、`paySign` ）返回给前端。 
3. **前端调起微信支付**：前端拿到支付参数后，调用 `uni.requestPayment` 发起支付流程，用户完成支付后微信会回调结果。 
4. **微信服务器回调（notify）**：微信支付完成后，异步通知后端服务器。后端校验签名，确认支付成功后更新订单状态，并返回“`SUCCESS`”响应。

## 41. 如何将 1px 转化为 0.5px

## 42. 深拷贝和浅拷贝
+ **浅拷贝（shallow copy）**：只拷贝一层，对于更深层次的对象，仅拷贝引用（地址 ）。新对象发生改变时，原对象也会跟着改变，因为它们指向同一块内存地址。常见方法有 `Slice()`（数组 ）、`concat`（数组 ）、`Object.assign()`（对象 ）等。  
+ **深拷贝（deep copy）**：会递归拷贝对象的每一级数据，新对象和原对象内容独立，一方改变不影响另一方。常用方法如 `JSON.parse(JSON.stringify(obj))`（但无法处理函数、正则等特殊类型 ），也可自己实现递归遍历对象进行拷贝，或者使用 Lodash 库的 `_.cloneDeep` 方法。

## 43. 父组件子组件执行的顺序（Vue2 生命周期角度 ）
父组件和子组件生命周期钩子执行顺序如下：

## 44. 如何解决跨域问题
当协议、域名、端口有一个不一样时就产生跨域，解决方法：

+ **后端设置 CORS**：后端在响应头中添加 `Access-Control-Allow-Origin` 等相关 CORS 头信息，允许指定来源的跨域请求。  
+ **前端配置代理服务器**：如在 Vue 项目中，通过 `vue.config.js` 配置 `devServer.proxy` ，将前端请求代理到后端接口，绕过浏览器的跨域限制（开发环境常用 ）。  
+ **使用 JSONP**：利用 `<script>` 标签不受跨域限制的特点，通过动态创建 `script` 标签，请求服务端返回指定格式（一般是函数调用形式 ）的数据，实现跨域获取数据，但仅支持 GET 请求。  
+ **使用 Nginx 反向代理**：在服务器端配置 Nginx，将前端的请求转发到后端对应的接口，让浏览器认为是同域请求，从而解决跨域问题（生产环境常用 ）。

## 46. 数据劫持
**<font style="color:#FF0000;">数据劫持</font>**：Vue底层使用了Object.defineProperty，将data中的数据放到了_data中，在_data中配置了setter方法，当去修改属性值时，setter方法则被自动调用，**<font style="color:#FF0000;">setter方法中不仅修改了属性值，而且还做了其他的事情</font>**，例如：重新渲染页面。setter方法就像半路劫持一样，所以称为数据劫持。

数据劫持就是通过 `Object.defineProperty`（Vue2 ）或 `Proxy`（Vue3 ）等方式，劫持对象属性的 `getter` 和 `setter` ，当数据发生变化时自动通知视图更新，是实现响应式数据的核心机制。 

## 47. 原型和原型链
我们所创建的每一个函数，解析器都会向函数中添加一个属性prototype,这个属性对应着一个对象，这个对象就是我们所谓的原型对象

![](https://cdn.nlark.com/yuque/0/2025/png/50923934/1749822033350-bd0b3ef9-b3e5-4e1a-b5d9-0734fd0bc6be.png)

+ **原型**：在 JavaScript 中，每个对象创建时会关联到另一个对象，这个对象就是它的原型，可通过 `__proto__` 访问（非标准但常用 ），标准上通过 `Prototype` 属性访问（构造函数的 `prototype` 指向原型对象 ）。原型对象包含可以被对象实例共享的属性和方法。  
+ **原型链**：当访问一个对象的属性时，若对象本身没有该属性，JavaScript 引擎会沿着其原型对象继续查找，如此递归直到找到属性或到达原型链尽头（`null` ），这种链式查找机制就是原型链。

## 48. 浏览器的三次握手和四次挥手
“三次握手”是建立TCP连接

:::color1
第一步，客户端向服务器发送一个包含同步序列号（SYN）的请求报文；

第二步，服务器接收到请求后，回复一个包含SYN和确认号（ACK）的报文，表示已收到请求并同意连接；

第三步，客户端再发送一个确认报文（ACK），至此连接建立完成。

:::



“四次挥手”则是在TCP连接关闭时所经历的四个步骤。

:::color1
第一步，客户端发送一个结束报文（FIN），表示要关闭连接；

第二步，服务器收到FIN后，回复一个确认报文（ACK），此时客户端到服务器的连接关闭，但服务器到客户端的连接仍可传输数据；

第三步，服务器处理完数据后，也发送一个FIN报文给客户端；

第四步，客户端收到后，回复一个ACK报文，至此双方连接完全关闭。这两个过程确保了网络通信中连接的可靠建立与关闭。 

:::

+ **三次握手（建立可靠的 TCP 连接 ）**：
    1. 第一次握手：客户端向服务器发送 SYN（同步序列编号 ）包，请求建立连接。  
    2. 第二次握手：服务器收到 SYN 包后，回复 SYN + ACK（同步 + 确认 ）包，同意建立连接。  
    3. 第三次握手：客户端收到服务器的 SYN + ACK 包后，发送 ACK 包，确认连接建立。此时 TCP 连接建立完成，双方可开始数据传输。
+ **四次挥手（优雅地断开 TCP 连接 ）**：
    1. 第一次挥手：客户端发送 FIN（结束 ）包，请求关闭连接。  
    2. 第二次挥手：服务器收到 FIN 包后，发送 ACK 包，确认收到关闭请求（此时服务器可能还有数据未发送完，不会马上关闭连接 ）。  
    3. 第三次挥手：服务器数据发送完毕后，发送 FIN 包，请求关闭连接。  
    4. 第四次挥手：客户端收到服务器的 FIN 包后，发送 ACK 包，确认关闭连接。此时 TCP 连接断开。

## 49. 重绘和回流
+ **回流（Reflow）**：当元素的大小、结构或位置发生变化时，浏览器会重新计算元素的几何属性和布局，触发整个页面或部分页面的重新渲染。会触发回流的操作如改变元素尺寸（`width`/`height` ）、位置（`top`/`left`/`margin` ）、增删 DOM 元素、读取某些属性（如 `offsetTop`、`clientHeight` ，可能强制回流 ）等。回流会带来较大性能开销，因为涉及到布局重新计算。  
+ **重绘（Repaint）**：当元素样式改变但不影响布局时，浏览器会重新绘制元素的外观。会触发重绘的操作如改变颜色（`color`、`background-color` ）、设置 `visibility: hidden` 、修改 `box-shadow`、`border-color` 等视觉属性。重绘的性能开销相对回流较小，但也应尽量减少频繁触发。  
简单来说，回流涉及布局变化，重绘仅涉及外观变化，回流必然会引发重绘，重绘不一定引发回流，且回流性能影响更大，开发中应避免频繁触发回流。

## 50. 项目从 0 到 1 的流程
1. **需求对接与评审**：和产品经理对接需求，参与 UI 设计评审，明确项目功能、交互逻辑、页面布局等需求细节。  
2. **技术选型**：根据项目需求（如功能复杂度、性能要求、团队技术栈等 ），选择合适的技术栈，包括前端框架（Vue/React 等 ）、UI 组件库、状态管理工具、请求库等；后端技术栈（如 Node.js + Express/Koa 、Java + SpringBoot 等 ）、数据库（MySQL、MongoDB 等 ）等。  
3. **项目初始化**：搭建项目脚手架（如使用 `vue-cli`、`create-react-app` 或自定义 Webpack 配置 ），初始化项目目录结构；配置代码规范（如 ESLint、Prettier ）、Git 版本控制（初始化仓库、设置 `.gitignore` 等 ）；配置环境变量（开发、测试、生产环境区分 ）。  
4. **基础模块开发**：开发项目基础功能模块，如用户登录注册、权限管理（路由守卫、动态路由、按钮权限控制 ）、通用组件封装（如弹窗、表格、表单等 ）、请求拦截器和响应拦截器（统一处理请求头、错误响应等 ）。  
5. **业务功能开发**：按照需求，开发具体业务页面和功能，如数据列表展示、增删改查操作、图表可视化、websocket 实时通信等功能，过程中注重组件化、模块化开发，提高代码复用性和可维护性。  
6. **联调测试**：前后端联调，确保接口数据交互正常；进行功能测试（单元测试、集成测试、E2E 测试等 ，根据项目要求选择 ），检查功能是否符合需求，修复发现的 bug 。  
7. **性能优化**：对项目进行性能优化，如代码分割（Code Splitting ）、懒加载（路由懒加载、组件懒加载 ）、优化首屏加载速度（预加载关键资源、减小打包体积 ）、内存泄漏检测与修复等。  
8. **部署上线**：配置生产环境打包，将项目部署到服务器（如使用 Nginx 、Docker 等 ）；配置域名、HTTPS 等；上线后进行线上验证，确保项目正常运行，持续关注线上反馈和性能监控，及时处理问题。
