## 三：闭包与垃圾回收
### 一、内存管理、垃圾回收与内存泄漏
1. 垃圾回收机制  
JavaScript 引擎通过自动垃圾回收机制管理内存，核心目标是识别并释放程序中不再使用的内存。

主要算法：

- 标记清除（主流）
  - 标记阶段：从根对象（全局变量等）出发，遍历所有可达对象，并标记为“活动”
  - 清除阶段：回收所有未被标记为“活动”的对象所占用的内存
- 标记整理  
在标记清除基础上增加整理阶段，将所有存活对象移动到内存一端，消除碎片，使空闲内存连续，提高后续分配效率
- 引用计数（较少使用）  
跟踪每个值被引用次数，当计数变为 `0` 时回收。缺点是无法处理循环引用，容易造成内存无法释放

2. 内存泄漏  
指程序中已分配内存由于某些原因未能被垃圾回收机制释放，导致可用内存逐渐减少。

常见原因及规避：

- 意外的全局变量：未声明变量变成全局对象属性，长期不回收
- 被遗忘的定时器或事件监听器：持有引用且未清理
- 脱离 DOM 的引用：DOM 已移除但 JS 仍保存引用
- 闭包不合理使用（见下文）

内存泄漏 vs. 内存溢出：

- 内存泄漏是长期运行后内存使用逐渐增长的问题
- 内存溢出是某一时刻申请内存超过系统可提供上限，导致崩溃
- 长期内存泄漏最终可能引发内存溢出

### 二、闭包
精准定义：  
闭包是一个函数（内部函数）与其创建时词法作用域（外部函数）的组合。内部函数可以记住并访问外部作用域变量，即使外部函数已经执行完毕。

形成闭包的三个核心条件：

1. 函数嵌套：外部函数内部定义内部函数
2. 内部函数引用外部函数的变量或参数
3. 内部函数被暴露出去（例如通过 `return`、赋值给全局变量、作为事件处理器）

为什么闭包可能导致内存泄漏：  
闭包会长期持有外部作用域变量引用，导致本应在外部函数结束后可回收的变量无法释放。

如何释放闭包占用内存：  
关键是切断引用。

```javascript
function outer() {
  let largeData = /* ...一个很大的数据... */
  return function inner() {
    console.log('I remember the scope.')
  }
}

let closureFn = outer() // 此时闭包形成，largeData 不会被释放

// ...使用 closureFn ...

closureFn = null // 关键步骤：切断对闭包函数的引用
// 下次垃圾回收时，inner 及其引用的 largeData 如果不再可达，将被回收
```

总结：  
闭包是 JavaScript 的重要能力，但要关注生命周期。不再需要时应主动断开引用，便于垃圾回收。
