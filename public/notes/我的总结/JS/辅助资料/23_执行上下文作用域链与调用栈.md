# 第23篇：执行上下文、作用域链与调用栈

这篇不是为了多背一个名词，而是为了补上 JavaScript 面试里最容易断层的底层主线。

如果你能把这篇讲清楚，很多零散题都会突然串起来：

- 为什么会有变量提升
- 为什么闭包能记住外层变量
- 为什么函数调用会形成栈
- 为什么报错里总看到 call stack
- 为什么 `this`、作用域、闭包不能混为一谈

---

## 一、这篇的目标

学完后，你至少要能回答下面 5 个问题：

1. 什么是执行上下文
2. 创建阶段和执行阶段分别做什么
3. 什么是词法作用域，什么是作用域链
4. 什么是调用栈，函数为什么会一层层入栈
5. 闭包、变量提升、`this` 分别和哪一层机制有关

## 二、先记住一张总图

```plain
代码被执行
-> 创建执行上下文
-> 建立词法环境 / 变量环境
-> 确定 this 绑定
-> 进入调用栈执行代码
-> 查变量时沿作用域链向外找
```

面试里如果先说出这张总图，整体会显得非常稳。

## 三、什么是执行上下文

### 1. 通俗定义

执行上下文可以理解成：

> JavaScript 在“准备执行某段代码时”创建的一份运行环境说明书。

这份说明书里至少会记录：

- 当前代码能访问哪些变量
- 当前 `this` 是谁
- 外层作用域在哪里
- 这段代码执行时要往哪一层栈里放

### 2. 三种常见执行上下文

1. 全局执行上下文
2. 函数执行上下文
3. `eval` 执行上下文

实际开发和面试里，前两种最重要。

---

## 四、执行上下文的两个阶段

### 1. 创建阶段

这一阶段还没正式逐行执行代码，但引擎已经先做准备：

- 建立词法环境
- 处理函数声明
- 处理变量声明
- 确定 `this`
- 建立外部环境引用

### 2. 执行阶段

这时才开始逐行运行代码：

- 变量被真正赋值
- 表达式开始计算
- 函数被调用
- 新的函数执行上下文继续被创建

### 3. 面试高频连接点

- 变量提升，本质发生在创建阶段
- 函数声明能先用，也是因为创建阶段已经处理好了
- `let` / `const` 的暂时性死区，也和创建阶段有关

---

## 五、词法作用域到底是什么

### 1. 一句话定义

词法作用域由“代码写在哪里”决定，不由“代码在哪里调用”决定。

```javascript
const outer = 'global'

function fn() {
  const outer = 'local'

  function inner() {
    console.log(outer)
  }

  return inner
}

const g = fn()
g() // 'local'
```

### 2. 这一点为什么关键

因为它直接决定：

- 闭包为什么能记住外层变量
- 为什么普通函数的 `this` 和作用域是两套规则
- 为什么把函数拿到别处调用，作用域链也不会变

---

## 六、什么是作用域链

### 1. 定义

当代码访问一个变量时，如果当前作用域找不到，就会沿着外层作用域一层层往外找，这条查找路径就叫作用域链。

### 2. 查找顺序

```plain
当前作用域
-> 外层函数作用域
-> 更外层作用域
-> 全局作用域
-> 找不到则报错
```

### 3. 一道必会口述版

> 作用域链是变量查找机制，不是函数调用链；它由词法作用域决定，写代码时就基本确定了。

---

## 七、调用栈是什么

### 1. 通俗理解

调用栈就是 JavaScript 管理函数执行顺序的一摞栈结构。

- 调用函数时，压栈
- 函数执行完，出栈
- 永远先执行最上面的那一层

### 2. 最简单例子

```javascript
function a() {
  b()
}

function b() {
  c()
}

function c() {
  console.log('end')
}

a()
```

大致过程：

```plain
全局上下文入栈
-> a 入栈
-> b 入栈
-> c 入栈
-> c 出栈
-> b 出栈
-> a 出栈
```

### 3. 为什么会有栈溢出

如果递归过深，函数不断压栈却没有及时出栈，就可能出现：

```plain
Maximum call stack size exceeded
```

## 八、执行上下文、作用域链、调用栈三者关系

最容易混的是这三个概念，其实它们关注点不同：

| 概念 | 回答什么问题 |
| --- | --- |
| 执行上下文 | 当前这段代码运行环境里有什么 |
| 作用域链 | 当前变量该去哪里找 |
| 调用栈 | 当前代码执行到哪一层函数了 |

建议直接背这句：

> 执行上下文是运行环境，作用域链是变量查找规则，调用栈是函数执行顺序管理结构。

## 九、和高频面试题怎么串起来

### 1. 变量提升

- 属于执行上下文创建阶段
- 不是“代码真的跑上去了”
- 而是声明先被收集和建立绑定

### 2. 闭包

- 本质是函数和其词法作用域的组合
- 外层函数执行结束后，内部函数仍能通过作用域链访问外部变量

### 3. `this`

- `this` 不是作用域链里的变量查找规则
- 普通函数的 `this` 主要看调用方式
- 箭头函数的 `this` 看定义时外层上下文

### 4. 异步

- `setTimeout`、Promise 回调不是“跳出作用域”
- 它们只是稍后再进调用栈执行
- 作用域还是写代码时确定的那套作用域

## 十、面试最爱追问的 6 个问题

### Q1：执行上下文和作用域是一回事吗

不是。

- 执行上下文是更大的运行时概念
- 作用域更偏变量可访问范围

### Q2：作用域链和调用栈是一回事吗

不是。

- 作用域链解决“变量去哪找”
- 调用栈解决“函数先执行谁”

### Q3：为什么会有变量提升

因为在执行上下文创建阶段，引擎会先处理声明并建立绑定。

### Q4：闭包为什么不会立刻释放外部变量

因为内部函数仍然持有对外层词法环境的引用。

### Q5：把函数传出去后，作用域会变吗

不会。

词法作用域看定义位置，不看调用位置。

### Q6：递归爆栈爆的是什么

爆的是调用栈深度，不是作用域链。

## 十一、30 秒面试回答模板

如果面试官问你“执行上下文、作用域链、调用栈怎么理解”，可以直接这样答：

> JavaScript 执行代码时会先创建执行上下文，里面会建立变量环境、词法环境和 this 绑定。变量查找时会沿着作用域链向外层查找，这个链条由词法作用域决定。函数调用时则会进入调用栈，调用一个压一层，执行完再出栈。所以变量提升看执行上下文创建阶段，闭包看词法作用域和作用域链，递归爆栈看调用栈深度，这三个概念相关但不是一回事。
