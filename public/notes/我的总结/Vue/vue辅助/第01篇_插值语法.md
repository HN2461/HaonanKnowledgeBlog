## 一、插值语法概述

### 1.1 什么是插值语法

Vue 的插值语法使用双大括号 `{{ }}`（Mustache 语法）包裹变量或表达式，它会自动将 Vue 实例（或组件）`data` 中的数据渲染到页面对应位置。

根据 [Vue 官方文档](http://vuejs.org/v2/guide/syntax)，插值语法是数据绑定最常见的形式，Vue 会将模板编译成虚拟 DOM 渲染函数，结合响应式系统智能地计算出最小的重新渲染代价。

### 1.2 表达式 vs 语句

**核心区别：**

1. **表达式**：能算出一个确定的值（数字、字符串、布尔值等）
2. **语句**：执行一个动作/操作，无返回结果

**Vue 插值规则：**

- ✅ 只支持**单个简单表达式**
- ❌ 不能放 `if`/`for` 语句、多语句、函数声明等
- ✅ 单个变量是最常用的合法表达式（本质是"无运算的表达式"）
- 💡 复杂逻辑优先用**计算属性**（`computed`）或**方法**（`methods`），再在插值里调用

## 二、核心用法示例

### 2.1 基础示例（完整可运行代码）

```vue
<template>
  <div id="app">
    <!-- 1. 直接渲染变量 -->
    <p>用户名：{{ username }}</p>
    
    <!-- 2. 渲染表达式结果 -->
    <p>13 + 12.2 = {{ 13 + 12.2 }}</p>
    
    <!-- 3. 渲染对象属性 -->
    <p>用户年龄：{{ user.age }}</p>
    
    <!-- 4. 渲染计算结果（简单表达式） -->
    <p>年龄加5：{{ user.age + 5 }}</p>
    
    <!-- 5. 渲染方法返回值 -->
    <p>格式化后的用户名：{{ formatName() }}</p>
  </div>
</template>

<script>
export default {
  name: 'App',
  data() {
    return {
      username: '张三',
      user: {
        age: 20
      }
    }
  },
  methods: {
    formatName() {
      return this.username.toUpperCase(); // 转为大写
    }
  }
}
</script>
```

### 2.2 代码解释

| 插值表达式 | 说明 | 页面显示 |
|-----------|------|---------|
| `{{ username }}` | 直接渲染 `data` 中的变量 | 用户名：张三 |
| `{{ 13 + 12.2 }}` | 支持简单的算术表达式 | 13 + 12.2 = 25.2 |
| `{{ user.age }}` | 支持访问对象属性 | 用户年龄：20 |
| `{{ user.age + 5 }}` | 支持简单运算 | 年龄加5：25 |
| `{{ formatName() }}` | 支持调用方法并渲染返回值 | 格式化后的用户名：张三 |

## 三、注意事项与限制

### 3.1 仅支持单个表达式

不能写多行代码、`if`/`else` 语句、循环等复杂逻辑。

```vue
<!-- ❌ 错误：不能写if语句 -->
{{ if (user.age > 18) { return '成年' } }}

<!-- ✅ 正确：三元表达式（简单逻辑） -->
{{ user.age > 18 ? '成年' : '未成年' }}
```

### 3.2 响应式更新

当 `data` 中的数据变化时，插值语法渲染的内容会**自动更新**（Vue 的响应式特性）。

### 3.3 不能在 HTML 属性中使用

如果要给 HTML 属性绑定数据，需要用 `v-bind` 指令。

```vue
<!-- ❌ 错误：属性中不能用插值语法 -->
<input value="{{ username }}">

<!-- ✅ 正确：使用v-bind（可简写为:） -->
<input v-bind:value="username">
<input :value="username"> <!-- 简写形式 -->
```

### 3.4 渲染纯文本

插值语法会将内容作为**纯文本**渲染，不会解析 HTML 标签。

```vue
<script>
export default {
  data() {
    return {
      htmlStr: '<h1>这是标题</h1>'
    }
  }
}
</script>

<!-- 页面会显示纯文本：<h1>这是标题</h1>，不会渲染成h1标签 -->
<p>{{ htmlStr }}</p>

<!-- 若要解析HTML，需用v-html指令 -->
<p v-html="htmlStr"></p>
```

⚠️ **安全警告**：使用 `v-html` 时要注意 XSS 攻击风险，不要渲染用户输入的 HTML 内容。

## 四、最佳实践

### 4.1 性能优化建议

1. **避免在插值中调用复杂方法**：每次重新渲染都会执行，建议改用计算属性
2. **使用计算属性缓存结果**：计算属性有缓存机制，依赖不变时不会重新计算

```vue
<!-- ❌ 不推荐：每次渲染都执行 -->
<p>{{ expensiveOperation() }}</p>

<!-- ✅ 推荐：使用计算属性 -->
<p>{{ computedValue }}</p>

<script>
export default {
  computed: {
    computedValue() {
      return this.expensiveOperation();
    }
  }
}
</script>
```

### 4.2 可读性建议

1. **保持插值表达式简洁**：复杂逻辑移到 `computed` 或 `methods`
2. **避免过长的链式调用**：如 `{{ user.profile.settings.theme.color }}`，建议用计算属性封装

## 五、总结

### 核心要点

1. **Vue 插值语法核心**：`{{ 变量/表达式 }}`，用于将数据动态渲染到页面，只支持单个简单表达式
2. **渲染规则**：
   - 插值语法渲染纯文本
   - 解析 HTML 需用 `v-html`（注意 XSS 风险）
   - 绑定 HTML 属性需用 `v-bind`（简写 `:`）
3. **响应式特性**：数据变化时，插值内容会自动更新，这是 Vue 的核心特性之一

### 对比表格

| 场景 | 使用方式 | 示例 |
|-----|---------|------|
| 渲染文本内容 | 插值语法 `{{ }}` | `<p>{{ message }}</p>` |
| 绑定 HTML 属性 | `v-bind` 或 `:` | `<img :src="imageUrl">` |
| 渲染 HTML 内容 | `v-html` | `<div v-html="htmlContent"></div>` |
| 双向绑定表单 | `v-model` | `<input v-model="username">` |
