# 📚 个人技术博客使用手册

## 🎯 项目简介

这是一个基于 Vue3 + Vite 构建的静态个人技术博客网站，专门用于展示和管理 Markdown 格式的技术笔记。

**核心特性：**
- 📝 支持 Markdown 格式笔记展示
- 🗂️ 自动识别文件夹结构，生成分类导航
- 🔍 全文搜索功能
- 🎨 浅色/深色主题切换
- 📱 响应式设计，支持移动端
- 🚀 静态网站，快速部署

---

## 🚀 快速开始

### 1. 环境准备
确保您的电脑已安装：
- Node.js (版本 16 或以上)

## 📁 笔记管理

### 文件存储结构
```
public/notes/
├── JavaScript/          # 分类文件夹
│   ├── 基础语法.md      # 笔记文件
│   └── ES6新特性.md
├── Vue/
│   └── Vue3组合式API.md
├── React/
│   └── React Hooks.md
└── CSS/
    └── Flexbox布局.md
```

### 文件命名规范

#### ✅ 推荐做法
- 使用有意义的英文命名：`javascript-basics.md`
- 使用中文命名：`JavaScript基础语法.md`
- 避免特殊字符：`!@#$%^&*()`

#### ❌ 避免做法
- 使用空格：`file name.md`
- 使用特殊符号：`file@name.md`
- 过长的文件名

### Markdown 文件格式

每个笔记文件支持 frontmatter 元信息：

```markdown
---
title: 自定义标题（可选）
date: 2025-01-15（可选）
tags: [JavaScript, 前端, 基础]（可选）
category: JavaScript（可选）
description: 文章简短描述（可选）
---

# 文章标题
这里是文章正文内容...

## 二级标题
内容...

### 三级标题
内容...
```

#### Frontmatter 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| title | string | ❌ | 文章标题，默认使用文件名 |
| date | string | ❌ | 发布日期，默认使用文件修改时间 |
| tags | array | ❌ | 标签列表，用于分类和搜索 |
| category | string | ❌ | 分类名称，默认使用文件夹名 |
| description | string | ❌ | 文章描述，用于搜索结果展示 |

## 🔄 内容更新流程

### 添加新笔记

1. **创建文件夹**（如果需要）
   ```bash
   # 在 public/notes/ 下创建新分类
   mkdir public/notes/新分类名
   ```

2. **创建笔记文件**
   ```bash
   # 在文件夹中创建 .md 文件
   touch public/notes/新分类名/笔记名称.md
   ```

3. **编写内容**
   - 使用 Markdown 语法编写
   - 添加 frontmatter 元信息（可选）

4. **更新索引**
   ```bash
   npm run generate:index
   ```

5. **查看效果**
   - 刷新浏览器页面
   - 新笔记应该出现在侧边栏和首页

### 修改现有笔记

1. 直接编辑 `public/notes/` 目录下的 `.md` 文件
2. 保存文件
3. 刷新浏览器查看效果（不需要重新生成索引）

### 删除笔记

1. 删除 `public/notes/` 目录下的文件或文件夹
2. 运行 `npm run generate:index` 更新索引
3. 刷新浏览器

## 🎨 网站功能使用

### 1. 首页功能

#### 搜索框
- 输入关键词搜索笔记
- 支持标题、内容、标签搜索
- 按回车键或点击搜索按钮执行搜索

#### 统计信息
- 显示总笔记数
- 显示分类数量
- 显示标签数量

#### 最近更新
- 显示最近 6 篇笔记
- 按时间倒序排列

#### 分类卡片
- 显示所有分类
- 点击进入分类页面
- 显示每个分类的笔记数量

### 2. 侧边栏导航

#### 文件树
- 📁 文件夹：点击展开/收起
- 📄 文件：点击查看笔记内容
- 📍 当前页面：高亮显示

#### 移动端
- 侧边栏默认隐藏
- 点击左上角菜单按钮显示
- 点击遮罩层或关闭按钮隐藏

### 3. 笔记详情页

#### 文章信息
- 标题、日期、分类、标签
- 阅读时间估算
- 文件大小信息

#### 目录导航（右侧）
- 自动生成文章目录
- 点击跳转到对应章节
- 滚动时高亮当前章节

#### 导航按钮
- 上一篇/下一篇笔记
- 返回顶部按钮

### 4. 搜索页面

#### 搜索功能
- 实时搜索建议
- 搜索历史记录
- 关键词高亮显示

#### 搜索历史
- 保存最近 10 次搜索
- 点击历史记录快速搜索
- 支持清除搜索历史

### 5. 主题切换

#### 切换方式
- 点击右上角主题按钮
- 自动保存用户偏好
- 支持系统主题检测

#### 主题类型
- 🌞 浅色主题：适合白天使用
- 🌙 深色主题：适合夜间使用

## 🌐 部署指南

### 部署到 Netlify

1. **构建项目**
   ```bash
   npm run build
   ```

2. **上传到 Netlify**
   - 登录 Netlify
   - 拖拽 `dist` 文件夹到部署区域
   - 等待部署完成

3. **自动部署（可选）**
   - 连接 Git 仓库
   - 设置构建命令：`npm run build`
   - 设置发布目录：`dist`

### 部署到 Vercel

1. **安装 Vercel CLI**
   ```bash
   npm i -g vercel
   ```

2. **部署**
   ```bash
   vercel --prod
   ```

### 部署到 GitHub Pages

1. **安装 gh-pages**
   ```bash
   npm install --save-dev gh-pages
   ```

2. **修改 package.json**
   ```json
   {
     "scripts": {
       "deploy": "npm run build && gh-pages -d dist"
     }
   }
   ```

3. **部署**
   ```bash
   npm run deploy
   ```

## ⚠️ 常见问题

### Q: 添加新笔记后网站没有显示？
A: 需要运行 `npm run generate:index` 重新生成索引，然后刷新浏览器。

### Q: 图片不显示怎么办？
A: 
1. 确保图片放在 `public/images/` 目录
2. 使用正确的路径：`![描述](/images/图片名.jpg)`
3. 检查文件名是否正确（大小写敏感）

### Q: 搜索功能不工作？
A: 
1. 确保已经生成了索引文件
2. 检查笔记内容是否包含搜索关键词
3. 尝试刷新页面重新加载

### Q: 移动端显示异常？
A: 
1. 确保使用现代浏览器
2. 尝试清除浏览器缓存
3. 检查是否有 CSS 冲突

### Q: 如何备份笔记？
A: 
1. 备份整个 `public/notes/` 目录
2. 使用 Git 进行版本控制
3. 定期导出 Markdown 文件

## 📊 性能优化建议

### 笔记内容优化

1. **文件大小**
   - 单个笔记文件建议 < 100KB
   - 过长的文章考虑分章节

2. **图片优化**
   - 使用 WebP 格式
   - 压缩图片大小
   - 添加合适的 alt 文本

3. **代码块**
   - 避免过长的代码片段
   - 使用语法高亮插件

### 网站性能

1. **构建优化**
   - 定期运行 `npm run build` 测试
   - 检查构建报告

2. **部署优化**
   - 启用 Gzip 压缩
   - 配置 CDN 加速
   - 设置缓存策略

## 🔧 自定义配置

### 修改网站标题

编辑 `index.html` 文件：
```html
<title>您的网站名称</title>
```

### 修改样式主题

编辑 `src/styles/index.css` 文件中的 CSS 变量：
```css
:root {
  --primary-color: #409eff;  /* 主题色 */
  --bg-primary: #ffffff;     /* 背景色 */
  --text-primary: #303133;   /* 文字色 */
}
```

### 添加自定义组件

1. 在 `src/components/` 创建新组件
2. 在页面中导入使用
3. 添加相应的样式
