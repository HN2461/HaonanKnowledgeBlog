## 一、表单交互类事件（以 input/change 为核心）
### 1. 基础输入事件：input /change（网页核心内容 + 实战补充）
**核心区别**：触发时机与条件完全不同，适配 “实时响应” 与 “确认后响应” 两类需求

| 事件 | 触发时机 | 核心特点 | 面试场景举例 |
| --- | --- | --- | --- |
| input | <font style="color:#DF2A3F;">输入框内容发生任何变化瞬间（敲键 / 粘贴 / 删改 / 输入法选字上屏）</font> | 实时性，无附加条件，内容变即触发；输入法候选阶段不触发 | 实时搜索联想、输入字数统计、表单内容即时预览 |
| change | <font style="color:#DF2A3F;">满足双条件：① 元素失去焦点（blur）；② 内容与获焦时相比有变更</font> | 非实时性，依赖 “失焦 + 真变更”；不受输入法阶段影响，只看最终结果 | 表单项失焦校验（如手机号格式验证）、下拉框选值确认、复选框状态变更后提交 |


**补充面试考点**：

+ 跨元素表现：change 事件并非仅适用于输入框 —— 下拉框（select）选值切换、复选框（checkbox）勾选状态变更时均会触发，而 input 仅针对可输入元素；
+ 手动触发坑点：通过prop()修改复选框checked属性时，不会自动触发 change 事件，需手动调用.change()或.trigger('change')；`<font style="color:rgba(0, 0, 0, 0.85);">prop()</font>`<font style="color:rgba(0, 0, 0, 0.85);"> 是 </font>**<font style="color:rgb(0, 0, 0) !important;">jQuery 框架提供的核心方法</font>**<font style="color:rgba(0, 0, 0, 0.85);">，作用是「获取或设置 DOM 元素的</font>**<font style="color:rgb(0, 0, 0) !important;">固有属性</font>**<font style="color:rgba(0, 0, 0, 0.85);">（也叫 “DOM 属性”）」</font>
+ 框架差异：在 nicegui 等事件驱动框架中，on_change参数绑定的函数需注意 “不主动调用” 特性，通过sender.value获取最新值可提升函数通用性。

### 2. 焦点事件：blur /focus/focusin /focusout（与 change 强关联）
**核心区别**：是否支持事件冒泡，决定能否监控子元素焦点变化

| 事件 | 触发时机 | 核心特点 | 面试场景举例 |
| --- | --- | --- | --- |
| focus | 元素获取焦点瞬间（点击 / Tab 切换） | 不冒泡，仅能监听自身焦点 | 单个输入框获取焦点时高亮边框 |
| blur | 元素失去焦点瞬间（点击外部 / Tab 切换） | 不冒泡，仅能监听自身失焦 | 单个输入框失焦后校验格式；是 change 触发的必要条件之一 |
| focusin | 同 focus（元素获取焦点） | 支持冒泡，父元素能监听到子元素焦点 | 表单整体监控：任意输入框获焦时显示 “正在编辑” 提示 |
| focusout | 同 blur（元素失去焦点） | 支持冒泡，父元素能监听到子元素失焦 | 表单整体校验：任意输入框失焦后统一检查必填项 |


**补充面试考点**：

focus/blur 无法通过事件委托实现（不冒泡），focusin/focusout 可通过事件委托监听多个子元素焦点变化，减少事件绑定数量。

### 3. 文本选择事件：select（与 input 互补）
**核心区别**：input 监听 “内容变更”，select 监听 “文本选中”

| 事件 | 触发时机 | 核心特点 | 面试场景举例 |
| --- | --- | --- | --- |
| input | 输入框内容发生任何变化（敲键 / 粘贴 / 删改） | 关注 “内容是否变”，选字不变不触发 | 实时搜索联想、输入字数统计 |
| select | 输入框 / 文本域中选中文本时（鼠标拖选 / 快捷键选） | 关注 “文本是否被选”，内容不变也触发 | 选中文本后弹出 “复制 / 翻译” 菜单、富文本编辑器选字高亮 |


**补充面试考点**：

select 事件触发后，可通过event.target.selectionStart和event.target.selectionEnd获取选中的文本范围。

## 二、鼠标操作类事件（高频交互场景）
### 1. 点击拆解事件：click /mousedown/mouseup
**核心区别**：触发时机不同，对应 “点击操作的不同阶段”

| 事件 | 触发时机 | 核心特点 | 面试场景举例 |
| --- | --- | --- | --- |
| mousedown | 鼠标按下瞬间（左键 / 右键均可） | 无需松开，操作开始时触发 | 拖拽功能：按下鼠标后开始移动元素 |
| mouseup | 鼠标松开瞬间（与按下元素无关） | 操作结束前触发 | 拖拽功能：松开鼠标后停止移动元素 |
| click | 完成 “mousedown+mouseup” 完整操作（需在同一元素） | 需 “按下 + 松开” 双条件，操作完成时触发 | 按钮点击提交、弹窗关闭、导航跳转 |


**面试常考顺序**：

同一元素上触发顺序 → mousedown → mouseup → click

### 2. 单次 / 双击事件：click /dblclick
**核心区别**：触发条件不同（单次完整点击 vs 连续两次完整点击）

| 事件 | 触发时机 | 核心特点 | 面试场景举例 |
| --- | --- | --- | --- |
| click | 单次 “按下 + 松开” 操作 | 单次触发，无时间间隔要求 | 列表项点击查看详情 |
| dblclick | 短时间内（约 300ms）连续两次 click | 需两次完整操作，有时间阈值 | 表格单元格双击编辑、图片双击放大 |


**补充面试考点**：

dblclick 触发前，会先触发两次 click（浏览器默认行为），开发中需做防抖处理（如双击时屏蔽前两次 click 的逻辑）。

## 三、键盘输入类事件（输入框交互辅助）
### 键盘事件：keydown /keyup/keypress（keypress 已废弃）
**核心区别**：触发阶段不同，是否识别功能键 / 可打印字符

| 事件 | 触发时机 | 核心特点 | 面试场景举例 |
| --- | --- | --- | --- |
| keydown | 键盘按下瞬间（任意按键） | 识别功能键（Tab/Enter/Shift），输入法候选阶段也触发 | 输入框回车提交、快捷键（Ctrl+C）监听 |
| keyup | 键盘松开瞬间（任意按键） | 同 keydown，松开时触发，可用于 “松开后执行” 逻辑 | 游戏操作：松开方向键停止移动角色 |
| keypress | 键盘按下（仅可打印字符，如字母 / 数字） | 不识别功能键，已被浏览器废弃，替代方案：keydown + event.key 判断 | （不推荐使用）早期字符输入监控 |


**与 input 的搭配场景**：

+ 实时限制输入（如仅允许数字）：keydown（按下时拦截非法字符）+ input（兜底监控粘贴的非法内容）
+ 避免频繁触发：input 用于内容变更，keydown 用于功能键（如回车），分工明确。

## 四、输入法兼容类事件：compositionstart /compositionend（与 input 配合）
**核心区别**：监控 “输入法候选字阶段”，解决 input 的输入法兼容问题

| 事件 | 触发时机 | 核心特点 | 面试场景举例 |
| --- | --- | --- | --- |
| compositionstart | 输入法开始输入候选字（如打拼音敲首字母） | 标记 “进入输入法阶段”，此时 input 不触发 | 实时搜索：候选字阶段不发送请求，避免无效接口调用 |
| compositionend | 输入法选字上屏 / 取消候选 | 标记 “退出输入法阶段”，之后会触发 input | 实时搜索：选字完成后，触发 input 再发送搜索请求 |


**补充面试考点**：

做实时搜索、实时翻译时，必须用这组事件判断是否在输入法阶段，否则会出现 “拼音还没选字就触发搜索” 的问题。

## 五、所有事件通用区分维度（面试万能思路）
不管遇到哪组易混淆事件，按以下 3 点分析，轻松答清区别：

1. **触发时机**：是 “操作开始”（mousedown）、“操作中”（input）、“操作完成”（click）还是 “状态变更”（focus）？
2. **触发条件**：是否需要附加条件（如 change 需 “失焦 + 内容变”，click 需 “按下 + 松开同元素”）？
3. **特殊属性**：是否支持冒泡（focus 不冒泡，focusin 冒泡）、是否识别特殊场景（输入法、功能键）、是否专属某类元素（如 select 仅表单输入框）？
