# Cursor 学习笔记系列（精简版）

> 更新时间：2026-04-14
> 这一轮已将 Cursor 专题从 28 篇压缩为 8 篇。整理依据以 Cursor 官方 docs、pricing、dashboard 与 troubleshooting 页面为主，只保留仍稳定、仍值得单独理解的能力簇。

以前这组文章拆得太细，目录里同时出现 Tab、Inline Edit、Terminal、Tabs、Checkpoints、快捷键、Shell Commands、CLI Shell Mode 等多个相互交叉的小题，会让读者花很多时间“找位置”，而不是快速建立工作流。现在改成 8 篇主线文章，目标是两件事：

- 保留官方口径仍明确、仍值得单独理解的功能边界
- 把本来就应该一起理解的能力收敛成能力簇，减少重复阅读成本

## 这次缩编的原则

- 安装、迁移、索引、忽略文件合并成一篇，因为它们决定的是“第一次打开 Cursor 时要不要先把地基打好”
- Agent、Ask、Tab、Inline Edit、Tabs、Checkpoints 合并成一篇，因为它们共同组成日常高频交互闭环
- Rules、AGENTS.md、Memories、@ 上下文、Notepads、Commands 合并成一篇，因为它们解决的是“上下文怎么长期治理”
- Terminal、Shell Commands、Cursor CLI、Git 工作流合并成一篇，因为它们都属于本地执行与交付闭环
- Background Agents、GitHub、Bugbot、Slack 合并成一篇，因为它们本质上都在讲“异步协作自动化”
- MCP、Tools、Shell Mode、Permissions 合并成一篇，因为它们共同定义能力边界和安全护栏
- 模型、隐私、团队后台、Analytics、Admin API 合并成一篇，因为它们解决的是成本、合规和组织治理
- Data Science、远程环境与常见问题合并成一篇，作为进阶与排障收口
