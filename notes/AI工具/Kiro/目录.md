## 覆盖主题

**Specs 规格系统**

- requirements.md / design.md / tasks.md 三层文档结构
- Feature Spec 与 Bugfix Spec 两种类型
- Requirements-First 与 Design-First 两种工作流
- 任务状态追踪（not_started / in_progress / completed）

**Steering 上下文管理**

- 工作区 / 全局 / 团队三种作用域
- always / fileMatch / manual / auto 四种 inclusion 模式
- 三个基础文件：product.md / tech.md / structure.md
- 文件引用语法 `#[[file:path]]`

**Hooks 自动化机制**

- 10 种触发事件：promptSubmit / agentStop / fileEdited / fileCreated / fileDeleted / userTriggered / preToolUse / postToolUse / preTaskExecution / postTaskExecution
- 两种动作类型：askAgent / runCommand
- 工具类型过滤：read / write / shell / web / spec

**MCP 工具扩展**

- 工作区级（.kiro/settings/mcp.json）与用户级（~/.kiro/settings/mcp.json）配置
- 本地服务器（command + args）与远程服务器（url + headers）
- 关键字段：autoApprove / disabledTools / env / disabled
- 环境变量引用语法 `${VAR_NAME}`

**工作流实战**

- Spec + Steering + Hooks + MCP 四者协同
- Kiro vs Claude Code vs Codex 横向对比
- 适合与不适合 Kiro 的场景分析
- 常见坑与解决方案
