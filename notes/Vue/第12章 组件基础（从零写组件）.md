## 12.1 组件是什么、为什么要组件化

组件（Component）可以理解为：

- 一段可以复用的 UI + 逻辑
- 有自己的数据、模板、样式、行为
- 可以像搭积木一样组合成页面

### 12.1.1 为什么需要组件化

当项目变大时，如果所有代码都写在一个页面里，会出现：

- 代码越来越长，难以维护
- 结构和逻辑高度耦合
- UI 复用困难（复制粘贴容易引入 bug）

组件化带来的收益：

- **复用**：同一个按钮/卡片/弹窗在多个页面复用
- **拆分复杂度**：页面拆成多个小组件，各自负责一块
- **协作**：多人开发时可按组件边界分工

## 12.2 组件注册方式

Vue2 里常见组件有两种注册方式：

- 全局组件
- 局部组件

### 12.2.1 全局组件

全局组件注册后，任何地方都可以直接使用。

```html
<div id="app">
  <my-button></my-button>
</div>

<script src="https://cdn.jsdelivr.net/npm/vue@2/dist/vue.js"></script>
<script>
  Vue.component('my-button', {
    template: '<button>我是全局组件按钮</button>'
  })

  new Vue({
    el: '#app'
  })
</script>
```

适用场景：

- 基础组件（Button、Icon、Input）
- 需要在多处使用且命名明确

注意：

- 全局注册太多会造成命名冲突与体积膨胀

### 12.2.2 局部组件

局部组件只在当前组件作用域内可用。

```html
<div id="app">
  <app-root></app-root>
</div>

<script src="https://cdn.jsdelivr.net/npm/vue@2/dist/vue.js"></script>
<script>
  const MyButton = {
    template: '<button>我是局部组件按钮</button>'
  }

  Vue.component('app-root', {
    components: { MyButton },
    template: '<div><my-button /></div>'
  })

  new Vue({ el: '#app' })
</script>
```

适用场景：

- 业务组件（只在某个页面/模块使用）
- 避免全局污染

## 12.3 单文件组件 SFC：`.vue` 文件结构

在 Vue CLI 项目中，最常见的是单文件组件（Single File Component）：

```vue
<template>
  <div class="box">Hello</div>
</template>

<script>
export default {
  name: 'HelloBox',
  data() {
    return {
      msg: 'Hello'
    }
  }
}
</script>

<style scoped>
.box {
  color: #2563eb;
}
</style>
```

三部分职责：

- `<template>`：写模板结构
- `<script>`：写逻辑（数据、方法、生命周期等）
- `<style>`：写样式（可选 `scoped` 限定作用域）

## 12.4 组件命名规范与目录组织建议

### 12.4.1 命名规范

建议：

- 组件文件名使用 **PascalCase**：`UserCard.vue`
- 组件 `name` 也用 PascalCase：`name: 'UserCard'`
- 在模板使用时，可用：
  - `<UserCard />`（在 SFC 环境常见）
  - 或 `<user-card />`（HTML 更传统的写法）

### 12.4.2 目录组织建议（简单好用）

一个常见结构：

```
src/
├── components/        # 通用组件
│   ├── base/          # 基础组件（BaseButton/BaseInput）
│   └── business/      # 业务通用组件（UserCard/OrderTable）
└── views/             # 页面级组件
```

## 12.5 组件中 `data` 为什么必须是函数

在组件里，`data` 必须是函数：

```js
export default {
  data() {
    return {
      count: 0
    }
  }
}
```

原因：

- 组件是可以被复用的
- 如果 `data` 是对象，那么多个组件实例会共享同一份数据
- 数据共享会导致互相影响，出现非常难排查的问题

### 12.5.1 错误示例（不要这样写）

```js
export default {
  data: {
    count: 0
  }
}
```

### 12.5.2 你要建立的直觉

- 每次创建一个组件实例，都应该拿到一份“全新的 data”
- 函数返回对象，正好满足这个需求

---

## 本章小结

- 组件化让大型项目变得可拆分、可复用、易协作。
- 组件注册分为全局/局部，业务优先用局部。
- Vue CLI 项目里主要写 `.vue` 单文件组件。
- 组件 `data` 必须是函数，确保每个实例数据独立。
