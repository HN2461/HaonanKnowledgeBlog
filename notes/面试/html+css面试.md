# HTML CSS 面试题详解

这篇文章主要整理 HTML 和 CSS 里非常常见的一批面试题。和单纯背答案不同，本文尽量用小白也能看懂的方式去讲：先说结论，再说为什么，最后补一点面试时容易被追问的细节。这样您不是死记硬背，而是真的能理解。

## 1、主流浏览器的内核分别是什么？

先直接记结论：

- IE：Trident 内核
- Firefox：Gecko 内核
- Safari：WebKit 内核
- Chrome：Blink 内核
- Opera：现在主流版本也是 Blink 内核

### 简单理解

浏览器内核，可以理解成浏览器内部真正负责“解析网页、渲染页面”的核心程序。我们平时写的 HTML、CSS、JavaScript，最终都要交给浏览器内核去理解和显示。

所以面试官问这个问题，本质上是在看您对浏览器基础有没有最起码的了解。

### 补充说明

- Chrome 早期其实是基于 WebKit，后来谷歌把它分出来做成了 Blink
- Opera 更早的时候用过 Presto，后面切换成了 Blink
- 现在说主流浏览器内核，基本答上面那一组就够了

### 面试回答模板

可以这样说：

“主流浏览器里，IE 是 Trident，Firefox 是 Gecko，Safari 是 WebKit，Chrome 和现在的 Opera 主要是 Blink。Chrome 早期基于 WebKit，后来演进成 Blink。”

## 2、每个 HTML 文件开头都有 `<!DOCTYPE html>`，它的作用是什么？

### 先说结论

`<!DOCTYPE html>` 的作用，是告诉浏览器当前页面按照 HTML5 标准来解析，并尽量进入标准模式，而不是怪异模式。

### 简单理解

它虽然看起来像一个标签，但其实不是普通的 HTML 标签，而是一个文档类型声明。

浏览器在打开网页的时候，第一件事就是先看这个声明。如果这个声明写对了，浏览器就更容易按统一标准渲染页面；如果不写，或者写错了，浏览器可能进入怪异模式，这时候很多盒模型、尺寸、边距的表现都会和标准模式不一样。

### 为什么它很重要

前端开发最怕的事情之一，就是“同一份代码在不同浏览器显示不一样”。  
而 `DOCTYPE` 的存在，就是为了尽量减少这种混乱。

### 面试里怎么说更稳

不要只说“它告诉浏览器用什么规范”。  
更完整一点可以说：

“`<!DOCTYPE html>` 是 HTML5 的文档类型声明，写在文档最前面，用来告诉浏览器按标准模式解析当前页面。如果缺失或格式错误，浏览器可能进入怪异模式，造成渲染差异。”

## 3、`div + css` 布局较 `table` 布局有什么优点？

### 先说结论

`div + css` 更适合做页面布局，因为它更灵活、结构更清晰，也更符合现代前端开发方式。  
`table` 更适合展示“表格数据”，而不是拿来做整页布局。

### 为什么以前很多人喜欢用 `table`

因为早年做网页时，CSS 布局能力没现在这么强，很多人就用 `table` 来拼页面。它的好处是“排版看起来比较规整”，尤其在老浏览器时代，效果有时还挺稳定。

但问题是：

- 页面结构会变得很臃肿
- 嵌套层级很多
- 修改起来很麻烦
- 不利于响应式布局
- 语义也不合适

### `div + css` 的主要优点

1. 结构和样式分离，代码更清晰
2. 后期修改布局更灵活
3. 更利于语义化和 SEO
4. 更适合响应式设计、组件化开发

### `table` 的适用场景

要注意，这不是说 `table` 没用。

如果页面里展示的是课程表、财务表、统计表这种“天然就是表格”的数据，那就应该优先考虑 `table`。  
不建议的是：拿 `table` 去做整个页面框架布局。

### 面试回答模板

“`div + css` 布局最大的优点是结构和样式分离，维护性更好，布局调整更灵活，也更利于语义化和响应式开发。`table` 更适合展示表格型数据，不适合作为整页布局方案。”

## 4、`img` 的 `alt` 属性与 `title` 属性有何异同？

### 先说结论

- `alt`：图片的替代文本
- `title`：补充提示文本

### 小白理解法

您可以把 `alt` 理解成“图片的替身文字”。

什么意思？

如果图片加载失败了，或者用户用的是读屏软件、看不到图片内容，那这个时候 `alt` 就能代替图片，告诉别人这张图本来是什么。

而 `title` 更像什么呢？

更像一个“鼠标移上去时弹出来的小提示”。

### `alt` 为什么更重要

因为 `alt` 不只是“图片裂开时显示的字”这么简单，它还有两个很重要的作用：

1. 无障碍访问  
   屏幕阅读器会把 `alt` 读出来，帮助视障用户理解图片内容。
2. 搜索引擎理解图片  
   搜索引擎不能像人一样直接看懂图片，所以 `alt` 能帮助它理解图像语义。

### `title` 的作用

`title` 更多是补充说明。  
比如一张风景图，`alt` 可以写“海边日落照片”，`title` 可以写“拍摄于夏季傍晚”。

### 示例代码

```html
<img src="image.jpg" alt="海边日落照片" title="拍摄于夏季傍晚">
```

### 面试回答模板

“`alt` 是图片的替代文本，图片加载失败时可能显示出来，同时对无障碍和 SEO 也很重要；`title` 是补充提示信息，通常在鼠标悬停时显示。开发中 `alt` 的语义价值通常比 `title` 更高。”

## 5、`strong` 标签与 `em` 标签的异同

### 先说结论

- 两者都是语义化的强调标签
- `em` 偏语气上的强调
- `strong` 偏更强的重要性强调

### 不要只把它们当样式标签

很多初学者会把：

- `strong` = 粗体
- `em` = 斜体

这样记其实不够准确。

因为粗体、斜体只是浏览器的默认表现，真正重要的是它们表达的“语义”。

### 怎么理解两者区别

比如一句话：

“这个按钮千万不要乱点。”

如果您只是想语气上强调一下某个词，可以用 `em`。  
如果您是想表达“这个内容非常重要”，那更适合用 `strong`。

### 面试回答模板

“`em` 和 `strong` 都是强调标签，但重点在语义。`em` 表示语气上的强调，`strong` 表示更强的重要性或严重性。它们默认通常显示为斜体和粗体，但面试里更应该强调语义区别。”

## 6、渐进增强和优雅降级之间的不同

### 先说结论

- 渐进增强：先保证基本可用，再逐步增强体验
- 优雅降级：先做完整功能，再向下兼容低版本环境

### 渐进增强怎么理解

可以把它理解成：

“先做最基础的版本，让所有用户至少都能正常使用，然后再给高版本浏览器增加更好的动画、交互、样式效果。”

这种思路更稳，更照顾兼容性和可访问性。

### 优雅降级怎么理解

它的思路是反过来的：

“先按照最理想、最完整的效果开发，然后再考虑低版本浏览器如果不支持，就做一些退化处理。”

### 举个生活化的例子

比如一个页面按钮有很炫的动画：

- 渐进增强：先保证按钮能点、能提交，再给支持的浏览器加动画
- 优雅降级：先把动画、阴影、特效全部做好，再想办法让旧浏览器至少别崩

### 面试回答模板

“渐进增强是先保证核心功能可用，再针对高版本浏览器增强体验；优雅降级是先实现完整效果，再兼容低版本浏览器。前者更强调基础可用性，后者更强调现代环境下的完整体验。”

## 7、CSS3 新增伪类举例

### 先说什么是伪类

伪类就是：根据元素的状态、位置、结构关系，去选中某类元素。

它不是给元素真的加了一个类名，而是 CSS 根据规则“判断你符不符合这个条件”。

### 常见结构伪类

- `p:first-of-type`：选中同类型中的第一个 `<p>`
- `p:last-of-type`：选中同类型中的最后一个 `<p>`
- `p:nth-of-type(2)`：选中同类型中的第 2 个 `<p>`
- `p:only-of-type`：选中同类型里唯一的 `<p>`
- `p:first-child`：如果这个 `<p>` 是父元素的第一个子元素，就选中它
- `p:last-child`：如果这个 `<p>` 是父元素的最后一个子元素，就选中它
- `p:nth-child(2)`：如果这个 `<p>` 是父元素的第 2 个子元素，就选中它

### 常见表单伪类

- `:disabled`：选中禁用的表单控件
- `:checked`：选中被勾选的单选框或复选框
- `:focus`：选中获取焦点的元素
- `:enabled`：选中可用状态的表单控件

### 容易混淆的点

很多人会把“伪类”和“伪元素”混了。

比如：

- `:hover`、`:focus`、`:nth-child()` 这些是伪类
- `::before`、`::after` 这些才是伪元素

## 9、简述一下 `src` 与 `href` 的区别

### 先说结论

- `src`：表示把外部资源引入当前元素中使用
- `href`：表示建立当前文档和目标资源之间的关联关系

### 小白理解法

`src` 更像“把资源本体拿过来用”。  
比如：

- `<img src="...">`
- `<script src="...">`
- `<iframe src="...">`

这些标签都是真的要去拿那个资源本身。

而 `href` 更像“建立一个关系”。

比如：

- `<a href="...">` 是建立跳转关系
- `<link href="...">` 是建立外部资源引用关系

### 为什么很多人会背错

以前有些资料喜欢一股脑地说：

“`src` 会阻塞，`href` 不会阻塞。”

这个说法太粗糙了。

更准确地说：

- 不同标签的行为不完全一样
- 比如 `<script src>` 是否阻塞解析，还和 `async`、`defer` 有关
- `<img src>` 会发请求，但不等于像同步脚本那样阻塞整个页面解析

### 面试回答模板

“`src` 一般用于把外部资源本身引入当前元素，比如 `img`、`script`、`iframe`；`href` 一般用于建立当前文档和目标资源之间的关联关系，比如 `a` 和 `link`。两者语义不同，不能简单粗暴地按‘谁一定阻塞谁不阻塞’来背。”

## 10、网页制作会用的图片格式有哪些？

### 常见格式

- `jpg/jpeg`
- `png`
- `gif`
- `svg`
- `webp`

### 分别怎么理解

#### 1. `jpg/jpeg`

特点：

- 有损压缩
- 体积通常较小
- 不支持透明

适合：

- 照片
- 风景图
- 色彩丰富的大图

#### 2. `png`

特点：

- 无损压缩
- 支持透明
- 画质比较稳定

适合：

- Logo
- 图标
- 边缘要求比较高的图片

#### 3. `gif`

特点：

- 支持简单动画
- 颜色数量有限

适合：

- 简单动图
- 表情图

#### 4. `svg`

特点：

- 矢量图
- 放大缩小不失真
- 非常适合图标和线稿

适合：

- 图标
- 图形
- 简单插画

#### 5. `webp`

特点：

- 支持有损和无损压缩
- 通常体积更小
- 现代浏览器兼容性已经比较好了

适合：

- 需要兼顾画质和体积的大多数网页图片场景

### 额外补充

`base64` 严格来说不是图片格式，而是一种编码方式。  
它常被用来把很小的图片直接写进代码里，从而减少额外请求，但体积一大就不划算。

## 13、一个页面上有大量图片，加载很慢，你有哪些方法优化这些图片的加载？

### 先说思路

图片优化不要只盯着“压缩图片”这一件事，而是要从：

- 什么时候加载
- 加载什么尺寸
- 用什么格式
- 怎么让用户感觉更快

这几个方向一起考虑。

### 常见优化方式

#### 1. 图片懒加载

滚动到可视区域附近再加载图片。  
这样首屏不会一次性请求太多资源。

#### 2. 合理预加载

像轮播图、相册这种场景，可以提前加载下一张图，提高切换体验。

#### 3. 选择合适格式

- 照片类优先考虑 `jpg` 或 `webp`
- 图标类优先考虑 `svg`
- 简单透明图可以用 `png`

#### 4. 大图不要小用

明明页面只显示 300px 的图，却加载 3000px 的原图，这就是典型浪费。

#### 5. 使用缩略图、占位图

先给用户看一个模糊小图或者缩略图，再慢慢替换成高清图，感知速度会更好。

#### 6. 小图标方案优化

可以根据场景考虑：

- SVG Sprite
- Iconfont
- CSS Sprite

#### 7. 配合缓存和 CDN

图片资源配合缓存策略和 CDN，能明显减轻服务器压力，也能提升用户访问速度。

### 面试回答模板

“图片优化可以从懒加载、预加载、合适格式选择、多尺寸适配、缩略图占位、图标方案优化、缓存和 CDN 几方面入手。核心目标是减少首屏压力、减少无效资源、提升用户感知速度。”

## 15、以前端角度出发，有哪些注意事项有利于 SEO？

### 先说结论

SEO 不是只靠“堆关键词”，而是要让搜索引擎更容易理解页面内容，同时让页面本身质量更高、速度更快、结构更清晰。

### 常见做法

#### 1. 保持内容质量和持续更新

内容才是 SEO 的基础。  
如果页面本身没价值，光做标签和关键词优化意义有限。

#### 2. 页面结构清晰

层级不要太乱，导航、标题、正文最好一眼能看懂。

#### 3. 合理使用语义化标签

比如：

- `header`
- `main`
- `article`
- `section`
- `nav`
- `h1` 到 `h6`

这样更有助于搜索引擎理解页面结构。

#### 4. 写好 `title` 和 `meta description`

- `title` 概括页面主题
- `description` 概括页面内容摘要

#### 5. 图片写好 `alt`

搜索引擎不直接“看懂”图片，所以 `alt` 能帮助它理解图像内容。

#### 6. 提升页面加载速度

减少不必要请求、压缩资源、优化首屏速度，这些都对 SEO 有帮助。

#### 7. 保证重要内容不是完全靠图片或脚本表达

如果核心内容全藏在图片里，或者必须执行脚本才能看到，那搜索引擎理解起来会更困难。

### 容易踩坑的点

- `meta keywords` 现在价值已经很低，不要把它当重点
- `display: none` 不是绝对不能用，但不要拿它隐藏大量关键词做作弊

## 16、对网页设置 CSS 样式的方式有哪些？

### 三种常见方式

#### 1. 外部样式表

通过 `<link>` 引入外部 CSS 文件。

优点：

- 最常用
- 可复用
- 结构清晰
- 维护方便

#### 2. 内部样式表

在页面的 `<style>` 标签里直接写 CSS。

适合：

- 单页测试
- 小型 demo
- 局部演示

#### 3. 内联样式

直接写在元素的 `style` 属性里。

适合：

- 非常少量、临时的样式

缺点：

- 不利于维护
- 样式复用差

### 面试回答模板

“CSS 设置方式主要有三种：外部样式表、内部样式表、内联样式。实际开发中最常用的是外部样式表，因为更利于维护和复用。”

## 17、CSS 选择器有哪些，选择器的权重优先级是什么？

### 常见选择器

1. ID 选择器：`#id`
2. 类选择器：`.class`
3. 标签选择器：`p`
4. 通配选择器：`*`
5. 属性选择器：`[type="text"]`
6. 伪类选择器：`a:hover`
7. 伪元素选择器：`::before`、`::after`
8. 子选择器：`ul > li`
9. 相邻兄弟选择器：`h1 + p`
10. 后代选择器：`li a`

### 权重怎么理解

权重可以理解成“谁说了更算数”。

常见优先级从高到低大致是：

- `!important`
- 内联样式
- ID 选择器
- 类 / 属性 / 伪类
- 标签 / 伪元素
- 通配符

常见记法：

- 内联样式：`1000`
- ID：`100`
- class / 属性 / 伪类：`10`
- 标签 / 伪元素：`1`
- 通配符：`0`

### 一个容易误解的点

权重相同的时候，不是谁“离得近”谁就一定赢，而是通常后写的会覆盖先写的。  
所以很多老资料里那句“就近原则”，最好理解成“在权重相同前提下，后面的样式通常覆盖前面的样式”。

## 18、CSS 中，对 DOM 元素设置不显示的方式有哪些？

### 常见方式

#### 1. `display: none`

特点：

- 元素直接不参与布局
- 不占位置

#### 2. `visibility: hidden`

特点：

- 元素看不见
- 但还占着原来的位置

#### 3. `opacity: 0`

特点：

- 元素完全透明
- 还占位置
- 通常还可以响应事件

#### 4. 宽高设为 `0`

可以配合 `overflow: hidden` 使用，用于某些特殊隐藏场景。

#### 5. 移出可视区域

通过绝对定位把元素移到屏幕外，常见于某些无障碍场景。

### 面试里容易加分的说法

不要只会背“隐藏”。  
最好顺手补一句：

“它们的区别主要在于是否占位、是否参与布局、是否还能响应交互。”

## 19、超链接访问过后，`hover` 样式就不出现的问题是什么，如何解决？

### 原因

通常是因为 `a` 标签几个伪类的书写顺序不对。

### 正确顺序

- `:link`
- `:visited`
- `:hover`
- `:active`

也就是常说的 `LVHA`。

### 怎么记

您可以把它理解成：

先是普通链接，再是访问过的链接，然后是鼠标移上去，最后是点击那一瞬间。

## 21、行内元素和块级元素的具体区别是什么？行内元素的 `padding` 和 `margin` 可设置吗？

### 块级元素的特点

- 默认独占一行
- 默认会尽量撑满父容器可用宽度
- 可以设置宽高、内外边距

### 行内元素的特点

- 默认不独占一行
- 宽度通常由内容决定
- 普通情况下 `width`、`height` 不太生效

### 行内元素的 `padding` 和 `margin` 能不能设置？

能设置。

但是要注意：

- 左右方向通常表现更明显
- 上下方向即使设置了，很多时候对布局影响不像块元素那样直观

所以不能简单粗暴说“行内元素上下 `margin` 和 `padding` 不能设置”，更准确的说法应该是“可以设置，但布局表现有限”。

### `inline-block`

它可以理解成一种折中：

- 不独占一行
- 但又可以设置宽高

常见行内块元素有：

- `img`
- `input`
- `button`
- `textarea`
- `select`

## 22、什么是外边距重叠？重叠的结果是什么？

### 先说结论

外边距重叠，也叫 margin collapse，通常发生在垂直方向上的相邻块级元素之间。发生后，两个外边距不会简单相加，而是会合并。

### 常见场景

#### 1. 相邻兄弟元素

上下两个块级元素相邻时，垂直外边距可能发生合并。

#### 2. 父子元素

如果父元素没有：

- `border`
- `padding`
- 行内内容

那子元素的垂直外边距可能和父元素发生塌陷。

### 重叠后的结果

通常会取较大的那个外边距值，而不是两个值直接相加。

### 解决方式

1. 让父元素形成 BFC
2. 给父元素加 `padding`
3. 给父元素加 `border`
4. 用伪元素或内容把它们隔开

### 示例

```css
.mc::before {
  content: '';
  display: table;
}
```

## 23、`rgba()` 和 `opacity` 的区别

### `opacity`

它是一个 CSS 属性。

例如：

```css
.box {
  opacity: 0.5;
}
```

它的特点是：整个元素最终渲染出来都会一起变透明，包括里面的文字、图片、子元素。

### `rgba()`

它不是一个单独属性，而是一种颜色写法。

例如：

```css
.box {
  background-color: rgba(0, 0, 0, 0.5);
}
```

这样只会让背景色半透明，不会让里面的文字也跟着一起透明。

### 一句话总结

- `opacity`：整个元素一起透明
- `rgba()`：只让某个颜色值带透明度

- 水平方向：`text-align: center`

## 25、如何水平垂直居中一个元素？

这个问题是面试高频题，常见方法很多。

### 方法一：绝对定位 + `transform`

```css
.parent {
  position: relative;
}

.child {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
}
```

这是最常见也最好记的一种方式。

### 方法二：绝对定位 + 负边距

前提是子元素宽高已知。

```css
.parent {
  position: relative;
}

.child {
  width: 100px;
  height: 100px;
  position: absolute;
  top: 50%;
  left: 50%;
  margin-top: -50px;
  margin-left: -50px;
}
```

### 方法三：绝对定位 + `margin: auto`

```css
.parent {
  position: relative;
}

.child {
  width: 100px;
  height: 100px;
  position: absolute;
  top: 0;
  right: 0;
  bottom: 0;
  left: 0;
  margin: auto;
}
```

### 方法四：Flex 布局

```css
.parent {
  display: flex;
  justify-content: center;
  align-items: center;
}
```

这是现代开发里最常用的一种。

### 方法五：`table-cell`

```css
.parent {
  display: table-cell;
  vertical-align: middle;
  text-align: center;
}

.child {
  display: inline-block;
}
```

### 面试建议

如果面试官问这个题，您最稳的答法是：

“常见方案有绝对定位配合 `transform`、Flex 居中、绝对定位加 `margin: auto` 等。现代项目里最常用的是 Flex。”

## 27、`px` 和 `em`、`rem` 的区别

### `px`

`px` 是绝对长度单位。  
写多少，通常就按多少来计算，直观、好理解。

### `em`

`em` 是相对单位，它相对于当前元素的字体大小来计算。  
如果当前元素自己没有设置字体大小，就会继续往父级那里找。

这就导致一个特点：`em` 容易受嵌套影响。

### `rem`

`rem` 也是相对单位，但它永远相对于根元素 `html` 的字体大小来计算。

这就让它比 `em` 更适合做整站统一缩放。

### 简单理解

- `px`：固定、直观
- `em`：跟当前元素字号走
- `rem`：跟根元素字号走

### 补充

浏览器默认根字号通常是 `16px`，所以很多场景下：

- `1rem = 16px`

但这是默认情况，不是永远固定不变。

## 28、Sass、LESS 是什么？大家为什么要使用它们？

### 先说结论

Sass 和 Less 都是 CSS 预处理器。

它们本质上是：在 CSS 基础上增加了一些更方便开发的能力，最后再编译成浏览器能识别的普通 CSS。

### 它们带来什么好处

#### 1. 变量

比如颜色、字号、间距这些值，可以统一管理。

#### 2. 嵌套

写起来更有层级感，结构更清楚。

#### 3. 混入

能把重复样式封装起来复用。

#### 4. 更适合大项目

当项目样式很多时，预处理器能明显减少重复代码，提高维护效率。

### 现在还要不要用

要看团队和项目。  
因为现代 CSS 自身也越来越强了，比如变量、`calc()`、层级能力都在提升。

所以面试回答时不必把 Sass、Less 吹得太神，重点说它们“提高开发效率、便于维护”就够了。

## 29、CSS 中 `link` 和 `@import` 的区别

### 先说结论

1. `<link>` 是 HTML 标签，`@import` 是 CSS 规则
2. `<link>` 通常在 HTML 解析阶段就参与资源加载
3. `@import` 是在解析到对应 CSS 时再触发加载
4. 实际开发里一般更推荐用 `<link>`

### 为什么一般推荐 `link`

因为它更直接、兼容性更稳定，也更符合工程化开发习惯。

### 一个常见误区

不要再背：

“`@import` 一定要等页面加载完成后才加载。”

这个说法太绝对。  
更准确的说法是：它依赖 CSS 的解析过程触发加载，不如 `<link>` 那么直接。

### 另一个常见误区

“`link` 引入的权重大于 `@import`”

这也不能这么说。  
样式最终谁生效，核心还是看：

- 选择器优先级
- 是否有 `!important`
- 样式书写顺序

而不是单纯因为“引入方式不同”就天然谁权重大。

## 30、简介盒子模型

### 盒子模型是什么

CSS 会把页面中的元素都看成一个个矩形盒子。  
这个盒子主要由四部分组成：

1. `content` 内容区
2. `padding` 内边距
3. `border` 边框
4. `margin` 外边距

### 标准盒模型

在默认情况下，`width` 和 `height` 只表示内容区大小，不包含 `padding` 和 `border`。

### `border-box`

如果设置：

```css
box-sizing: border-box;
```

那 `width` 和 `height` 就会把 `padding` 和 `border` 也算进去。  
这在现代开发里非常常见，因为更容易控制布局尺寸。

### 面试回答模板

“盒子模型由内容区、内边距、边框、外边距组成。默认标准盒模型下，`width` 和 `height` 只表示内容区大小；如果使用 `box-sizing: border-box`，则会把 `padding` 和 `border` 也计算在内。”

## 31、为什么要初始化样式？

### 原因

不同浏览器对某些标签会有默认样式差异。  
比如：

- `h1` 的大小
- `p` 的上下边距
- `ul` 的缩进

如果不做统一处理，不同浏览器里页面就可能出现细微差别。

### 常见方案

- `reset.css`
- `normalize.css`

### 两者简单区别

- `reset.css` 更激进，基本把默认样式都清掉
- `normalize.css` 更温和，尽量保留合理默认值，同时统一差异

## 32、对 BFC 规范（块级格式化上下文）的理解

### 先说什么是 BFC

BFC 全称是 Block Formatting Context，中文常叫“块级格式化上下文”。

刚学的时候这个词会觉得很绕，您可以先把它简单理解成：

“一个相对独立的布局区域。”

在这个区域里，内部元素怎么排版，和外部元素之间会有一定隔离。

### BFC 常见作用

#### 1. 解决垂直外边距重叠

也就是常说的 margin 塌陷问题。

#### 2. 包含内部浮动元素

有时候父元素里全是浮动子元素，父元素高度会塌掉。  
让父元素形成 BFC，往往就能把浮动包住。

#### 3. 隔离布局影响

内部布局一般不会直接影响到外部同级区域。

### 常见触发方式

1. `float` 不为 `none`
2. `position: absolute`
3. `position: fixed`
4. `overflow` 不为 `visible`
5. `display: inline-block`
6. `display: table-cell`
7. `display: flow-root`

### 容易背错的点

很多旧资料会写：

“`position` 不是 `static` 或 `relative` 就行”

这种说法不够严谨。  
更安全的回答方式，是直接记上面这些常见触发条件。

## 39、列出 `position` 的值，注意它们的定位参照原点

默认值，不开启定位。

### 2. `relative`

相对于元素原本所在的位置进行偏移。

相对于最近的已定位祖先元素定位。

相对于浏览器视口定位。

### 5. `sticky`

可以理解成“粘性定位”。  
在达到某个阈值前像 `relative`，达到后会表现出类似固定效果。

## 44、从浏览器地址栏输入 URL 到显示页面的步骤

## 46、简单描述一下什么是 Flex 布局及常用属性说明

### 什么是 Flex

Flex 是弹性布局，它特别适合处理“一维方向上的排列和对齐问题”。

比如：

- 一行排多个按钮
- 左右分布
- 垂直居中
- 元素自动分配剩余空间

这些场景，用 Flex 都非常方便。

### 容器常见属性

#### 1. `flex-direction`

决定主轴方向：

- `row`
- `row-reverse`
- `column`
- `column-reverse`

#### 2. `flex-wrap`

决定一行放不下时要不要换行：

- `nowrap`
- `wrap`
- `wrap-reverse`

#### 3. `flex-flow`

是 `flex-direction` 和 `flex-wrap` 的简写。

#### 4. `justify-content`

控制主轴方向对齐方式。

#### 5. `align-items`

控制交叉轴方向对齐方式。

#### 6. `align-content`

多行内容时，在交叉轴上的整体分布方式。

### 子项常见属性

#### 1. `flex-grow`

有剩余空间时，项目放大的比例。

#### 2. `flex-shrink`

空间不足时，项目缩小的比例。

#### 3. `flex-basis`

项目在分配剩余空间之前的基础尺寸。

#### 4. `align-self`

单个子项覆盖容器的对齐方式。

#### 5. `order`

控制项目排序。

#### 6. `flex`

是 `flex-grow`、`flex-shrink`、`flex-basis` 的简写。

### `flex: 1` 怎么理解

很多面试会问这个。

通常可以理解成：

- 这个元素可以放大
- 可以缩小
- 会积极参与剩余空间分配

常见记法会写成：

```css
flex: 1 1 0%;
```

### 面试里一句话总结

“Flex 是一种一维弹性布局模型，擅长解决元素排列、对齐、剩余空间分配的问题。常用属性有 `flex-direction`、`justify-content`、`align-items`、`flex-wrap`、`flex-grow`、`flex-shrink`、`flex-basis` 等。”

## 45、用 CSS 画一个三角形

### 原理

把一个元素的宽高都设为 `0`，然后只保留某一边的边框颜色，其他边透明，就会形成三角形。

### 示例

```css
.triangle {
  width: 0;
  height: 0;
  border: 50px solid transparent;
  border-left-color: yellowgreen;
}
```

### 为什么能形成三角形

因为当元素本身没有内容区大小时，边框在视觉上会向四个方向展开，保留一边颜色，其他边透明，看起来就像一个三角形。

## 46、清除浮动有哪些方式？

### 为什么要清除浮动

因为子元素浮动之后，可能脱离普通文档流，导致父元素高度塌陷。

### 常见方式

#### 1. 让父元素形成 BFC

比如给父元素设置：

```css
overflow: hidden;
```

#### 2. 在后面加一个清除元素

新增一个块级元素并设置：

```css
clear: both;
```

#### 3. 使用伪元素清除浮动

这是开发里非常常见的写法。

```css
.clearfix::after {
  content: '';
  display: block;
  clear: both;
}
```

### 面试建议

如果面试官问“清除浮动”，您最好顺带说一下“为什么要清除”，这样回答会显得更完整。

## 48、什么是 CSS Sprite？有什么优缺点？

### 什么是 CSS Sprite

CSS Sprite 通常叫“雪碧图”。

它的核心思路是：

把多张小图合成一张大图，然后通过 `background-position` 去显示其中某一块区域。

### 优点

1. 可以减少 HTTP 请求数量
2. 适合一批固定小图标的管理

### 缺点

1. 制作和维护麻烦
2. 图标改动时可能要重新合图
3. 在高分辨率、多倍图、自适应场景下管理成本更高

### 现代项目里常见替代方案

- SVG
- SVG Sprite
- Iconfont
- 组件化图标方案

所以现在雪碧图不像以前那样普遍了，但面试里还是会问，因为它代表的是一种经典优化思路。

## 49、`line-height` 和 `height` 有什么区别？

### `line-height`

表示行高，控制的是一行文字所占的高度。

### `height`

表示元素内容区本身的高度。

### 一个典型应用

单行文字垂直居中时，经常把：

- `height`
- `line-height`

设置成一样。

但要注意，这更适合单行文本，不适合复杂多行文本布局。

## 50、CSS 的三大特性

CSS 的三大特性是：

1. 层叠性
2. 继承性
3. 优先级

### 1. 层叠性

多个样式都作用在同一个元素上时，最终会按照规则决定谁生效。

### 2. 继承性

某些样式会从父元素传给子元素，比如字体、颜色。

### 3. 优先级

不同选择器命中同一个元素时，要比较谁的优先级更高。

### 一句话总结

层叠性解决“都在改时谁生效”，继承性解决“父亲的样式孩子会不会继承”，优先级解决“谁更有话语权”。

## 51、`defer` 和 `async` 的区别

### 先说结论

它们都是给 `<script>` 用的属性，作用都是尽量减少脚本对页面解析的阻塞，但行为不一样。

- `async`：下载脚本时不阻塞 HTML 解析，下载完马上执行
- `defer`：下载脚本时不阻塞 HTML 解析，等 HTML 解析完成后再按顺序执行

### 为什么会有这两个属性

默认情况下，浏览器解析到普通 `<script>` 时，通常会停下来先下载并执行脚本，然后再继续解析后面的 HTML。  
这就可能影响页面首屏渲染速度。

所以后来就有了 `async` 和 `defer`，用来减少这种阻塞。

### `async` 的特点

- 谁先下载完谁先执行
- 执行顺序不一定和书写顺序一致
- 更适合彼此独立、不依赖别的脚本的资源

比如统计脚本、广告脚本这类场景，常常可以考虑 `async`。

### `defer` 的特点

- 会等 HTML 全部解析完成后再执行
- 多个 `defer` 脚本会按照它们在文档中的顺序执行
- 更适合主业务脚本

### 面试回答模板

“`async` 和 `defer` 都能减少脚本阻塞页面解析。区别是 `async` 下载完就立即执行，执行顺序不一定按书写顺序；`defer` 会等 HTML 解析完成后按顺序执行，所以主业务脚本更常用 `defer`。”

## 52、HTML5 语义化标签有哪些？为什么要用？

### 常见语义化标签

- `header`
- `nav`
- `main`
- `article`
- `section`
- `aside`
- `footer`

### 什么叫语义化

语义化不是“标签看起来高级”，而是“让标签本身就能表达内容含义”。

比如：

- 页面顶部区域用 `header`
- 导航区域用 `nav`
- 主体内容区域用 `main`
- 一篇独立文章用 `article`
- 侧边栏用 `aside`

这样别人一看结构就知道每块内容大概是干什么的。

### 为什么要用语义化标签

#### 1. 代码可读性更好

后面维护的人更容易看懂页面结构。

#### 2. 更利于 SEO

搜索引擎更容易理解页面内容层次。

#### 3. 更利于无障碍访问

辅助技术更容易识别页面区域和重点内容。

### 面试回答模板

“HTML5 语义化标签有 `header`、`nav`、`main`、`article`、`section`、`aside`、`footer` 等。使用语义化标签能提高代码可读性，也更利于 SEO 和无障碍访问。”

## 53、`meta viewport` 的作用是什么？

### 先说结论

`meta viewport` 主要用于移动端页面适配，它告诉浏览器页面的可视区域应该怎么计算和缩放。

### 常见写法

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

### 这句配置怎么理解

- `width=device-width`：让页面宽度跟设备屏幕宽度一致
- `initial-scale=1.0`：初始缩放比例为 1

### 为什么移动端需要它

如果不写这个，移动浏览器可能会把页面按一个较大的默认虚拟宽度来渲染，结果页面就会看起来缩小、布局怪异。

### 面试回答模板

“`meta viewport` 用于移动端适配，控制页面的可视区域和缩放行为。常见写法是 `width=device-width, initial-scale=1.0`，这样页面宽度会和设备宽度一致，更适合响应式布局。”

## 55、什么是空元素？常见空元素有哪些？

### 什么是空元素

空元素就是本身不包裹内容、没有闭合内容部分的元素。

## 56、`srcset` 和 `sizes` 是做什么的？

### 先说结论

它们主要是为了响应式图片服务的，让浏览器根据设备宽度、屏幕像素密度等条件，自动选择更合适的图片资源。

### 为什么会需要它

同一张图片，在手机、小平板、大屏电脑上实际需要的尺寸往往不一样。  
如果永远只加载一张大图，会浪费流量；如果永远只加载一张小图，在高清屏上又可能发虚。

### `srcset`

用来告诉浏览器：“我这边有几张不同规格的图，你自己选一个更合适的。”

### `sizes`

用来告诉浏览器：“这张图在不同视口下，大概会占多宽。”

### 示例

```html
<img
  src="small.jpg"
  srcset="small.jpg 480w, medium.jpg 800w, large.jpg 1200w"
  sizes="(max-width: 600px) 100vw, 50vw"
  alt="示例图片"
>
```

### 面试回答模板

“`srcset` 和 `sizes` 主要用于响应式图片。`srcset` 提供多种图片资源备选，`sizes` 告诉浏览器图片在不同视口下大概会显示多宽，浏览器再自动选择更合适的资源。”

## 57、`cookie`、`localStorage`、`sessionStorage` 的区别

### 先说结论

这三个都能在浏览器端存数据，但用途和特性不一样。

### `cookie`

特点：

- 体积较小
- 会随着 HTTP 请求自动带给服务器
- 常用于登录状态、会话标识等场景

### `localStorage`

特点：

- 长期保存，除非手动清除
- 不会自动随请求发送到服务器
- 适合存一些前端持久化数据

### `sessionStorage`

特点：

- 生命周期通常跟当前标签页会话一致
- 关闭当前标签页后一般会消失
- 也不会自动随请求发送到服务器

### 面试回答模板

“`cookie`、`localStorage`、`sessionStorage` 都能存数据。`cookie` 会自动随请求发送给服务器，常用于会话相关；`localStorage` 持久保存，除非手动清除；`sessionStorage` 通常在当前会话或标签页关闭后失效。”

## 58、`iframe` 有什么作用？它有什么缺点？

### 作用

`iframe` 可以在当前页面里嵌入另一个页面。

常见场景：

- 第三方页面嵌入
- 某些独立内容隔离展示
- 历史系统兼容

### 缺点

#### 1. 性能开销

它相当于又嵌了一个独立页面，会增加资源加载和渲染成本。

#### 2. SEO 不友好

搜索引擎对 `iframe` 内内容的处理通常不如普通正文直接。

#### 3. 交互和样式控制麻烦

父子页面之间不是像普通 DOM 那样直接操作。

#### 4. 安全问题

如果来源不可信，可能有安全风险，所以经常要配合 `sandbox`、同源策略等一起考虑。

### 面试回答模板

“`iframe` 的作用是在当前页面嵌入另一个页面，但它会带来性能、SEO、样式控制和安全方面的问题，所以现代开发中通常会谨慎使用。”

## 59、`box-sizing: content-box` 和 `border-box` 的区别

### `content-box`

这是默认盒模型。

在这种模式下：

- `width` / `height` 只表示内容区
- `padding` 和 `border` 要额外往外加

### `border-box`

在这种模式下：

- `width` / `height` 会把 `padding` 和 `border` 也算进去

### 为什么现代开发常用 `border-box`

因为布局更容易控制。

比如一个盒子您就想让它总宽度是 `200px`，那用 `border-box` 更直观，不容易一不小心算超。

### 示例

```css
* {
  box-sizing: border-box;
}
```

### 面试回答模板

“默认的 `content-box` 下，`width` 和 `height` 只表示内容区；`border-box` 下，`width` 和 `height` 会包含 `padding` 和 `border`。现代项目里经常统一使用 `border-box`，这样布局更好控制。”

## 60、`position: sticky` 是什么？和 `fixed` 有什么区别？

### `sticky`

`sticky` 可以理解成“带条件的固定”。

它在没到阈值前，表现有点像 `relative`；  
当滚动到指定位置后，它会像吸附住一样。

### 典型写法

```css
.nav {
  position: sticky;
  top: 0;
}
```

### 和 `fixed` 的区别

- `fixed`：直接相对视口固定
- `sticky`：先跟着正常文档流走，到了阈值后才表现出吸附效果

### 典型场景

- 吸顶导航
- 分组标题吸附

## 61、回流和重绘是什么？有什么区别？

### 先说结论

- 回流：布局重新计算
- 重绘：外观重新绘制

一般来说，回流的成本比重绘更高。

### 怎么理解回流

如果一个改动影响到了元素的几何信息，比如：

- 宽高
- 位置
- 布局

那浏览器就可能要重新计算布局，这就是回流。

### 怎么理解重绘

如果只是外观变了，比如：

- 颜色
- 背景
- 边框颜色

但元素大小和位置没变，那通常主要是重绘。

### 为什么面试爱问

因为它和性能优化关系很大。  
如果频繁触发布局变化，就更容易造成页面卡顿。

### 面试回答模板

“回流是浏览器重新计算元素布局，重绘是浏览器重新绘制元素外观。回流通常比重绘成本更高，因为它往往伴随着重新计算位置和尺寸。”

## 62、`z-index` 为什么有时候不生效？

### 常见原因

#### 1. 元素没有形成定位上下文

很多情况下，`z-index` 需要配合定位元素使用，比如：

- `position: relative`
- `position: absolute`
- `position: fixed`
- `position: sticky`

#### 2. 受层叠上下文影响

有时候不是您的 `z-index` 数值小，而是它被困在了父级的层叠上下文里。

#### 3. 兄弟元素和父级关系判断错了

同一个层叠上下文里的比较，和跨层叠上下文的比较，逻辑不一样。

### 小白理解法

不是所有元素都在同一张“平面图”上比大小。  
有时候它们已经被分到不同的“楼层”里了，您在小楼层里把数字写很大，也不一定能盖过大楼层外面的元素。

## 63、`overflow: hidden`、`auto`、`scroll` 的区别

### `hidden`

- 超出内容会被裁掉
- 不显示滚动条

### `auto`

- 超出时才显示滚动条
- 不超出时通常不显示

### `scroll`

- 无论内容有没有超出，通常都会预留滚动机制

### 面试回答模板

“`overflow: hidden` 会裁剪溢出内容，`auto` 会在内容超出时自动出现滚动条，`scroll` 通常总是提供滚动机制。”

## 64、`transition` 和 `animation` 的区别

### `transition`

更适合“状态变化时的过渡”。

比如：

- 鼠标移入按钮，颜色平滑变化
- 宽度从小变大

它通常需要一个触发条件。

### `animation`

更适合“主动播放的一整段动画”。

它依赖 `@keyframes`，可以做更复杂、更完整的动画过程。

### 一句话区别

- `transition`：从一个状态过渡到另一个状态
- `animation`：按关键帧播放一段完整动画

## 65、媒体查询是什么？有什么用？

### 先说结论

媒体查询是响应式设计的核心手段之一，它可以让我们根据不同设备条件，写不同的 CSS。

### 常见场景

- 手机和电脑布局不同
- 小屏和大屏字号不同
- 某些区域在窄屏下改成单列

### 示例

```css
@media (max-width: 768px) {
  .container {
    flex-direction: column;
  }
}
```

### 怎么理解

可以把它理解成：

“如果屏幕满足某个条件，就应用这一套样式。”

## 66、响应式布局和自适应布局的区别

### 响应式布局

一套页面结构，随着屏幕尺寸变化自动流动和调整。

特点是更灵活、更连续。

### 自适应布局

通常是针对几个固定尺寸区间，准备几套相对明确的布局方案。

特点是分档更明显。

### 简单理解

- 响应式：像水一样，连续变化
- 自适应：像换挡一样，到某个区间切一次方案

### 面试回答模板

“响应式布局通常是一套布局随屏幕连续变化，自适应布局则更像针对几个固定区间切换不同方案。现代前端里两者常常会结合使用。”

## 67、圣杯布局和双飞翼布局是什么？

### 它们解决的核心问题

这两种都是经典三栏布局方案，目标一般是：

- 左右两栏固定宽度
- 中间主内容自适应
- 还希望主内容优先加载或优先展示

### 圣杯布局

一般通过：

- 浮动
- 负外边距
- 父元素 `padding`

等方式实现。

### 双飞翼布局

思路和圣杯布局很像，但通常会在中间主内容里再包一层，借此处理左右两栏位置问题。

### 现在还常用吗

现在实际开发中，Flex 和 Grid 更常见。  
但圣杯布局、双飞翼布局在面试里仍然常被问，因为它们能反映您对传统布局思路是否理解。

## 68、Grid 布局是什么？和 Flex 有什么区别？

### Grid 是什么

Grid 是网格布局，它更适合做二维布局，也就是同时处理行和列。

### Flex 和 Grid 的区别

- Flex 更适合一维布局，主要处理一行或者一列
- Grid 更适合二维布局，同时处理多行多列

### 什么时候用 Flex

比如：

- 一排按钮
- 水平垂直居中
- 导航排列

### 什么时候用 Grid

比如：

- 宫格卡片
- 仪表盘
- 比较复杂的整体页面分区

### 面试回答模板

“Flex 更偏一维布局，擅长解决一条主轴上的排列和对齐；Grid 更偏二维布局，适合同时控制行和列。简单线性布局更适合 Flex，复杂网格布局更适合 Grid。”
