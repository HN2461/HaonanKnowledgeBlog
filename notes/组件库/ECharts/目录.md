# ECharts 目录

## 为什么单独开这个目录

`ECharts` 和 `Element Plus`、`uv-ui` 不太一样。

前两个更偏“页面组件库”，而 `ECharts` 更偏“数据可视化能力库”。

它重点解决的是：

- 数据怎么变成图
- 一张图里怎么放坐标轴、提示框、图例、缩放、联动
- 图表怎么在后台、看板、报表、移动端页面里稳定运行

所以这里单独开一个专题，把图表这条线系统收起来。

## 当前内容

- 第一篇《快速认识 ECharts：安装、核心概念与常见图表》
  - 先讲清 `ECharts` 是什么、适合什么场景、最新版资料怎么看、怎么安装
  - 再讲 `option`、`dataset`、动态更新、事件交互、样式、性能、Vue 里怎么接

- 第二篇《常见四类图表：折线图、柱状图、饼图、散点图怎么选》
  - 先把最常用的四类图表场景讲透，再补上组合图、堆叠、环形图和双轴图的实战判断

- 第三篇《dataset、dimensions、encode、transform：把后端数据整理成能复用的图表数据》
  - 重点讲数据组织思路，解决“接口数据怎么喂给图表、怎么一份数据复用多张图”的问题

- 第四篇《Vue 3 + Vite 项目落地：把 ECharts 封成能直接上项目的图表组件》
  - 重点讲 Vue ECharts 的推荐接法、按需引入、自动适配、主题、loading、事件、手动更新和常见坑

- 第五篇《交互与联动：tooltip、legend、dataZoom、事件监听、dispatchAction 怎么配合》
  - 重点讲后台分析页最常用的交互能力，让图表从“能看”升级成“能分析、能联动、能钻取”

- 第六篇《项目开发速查清单：从安装到上线排查的 ECharts 实战路线》
  - 把官方资料校准后的安装、按需引入、Vue 封装、dataset、交互、性能、安全、无障碍和排错流程串成一张项目落地清单

## 建议阅读顺序

1. 先看第一篇，把整体地图建立起来
2. 再看第二篇，把最常用的四类图和组合图吃透
3. 再看第三篇，把 `dataset`、`encode`、`transform` 这条数据组织线吃透
4. 再看第四篇，把项目里真正会用到的封装和接法搞定
5. 再看第五篇，把交互、联动、缩放和程序控制这条线吃透
6. 最后看第六篇，把前面知识压缩成项目开发检查表，后面写页面时可以直接照着走
7. 后面如果继续扩写，再按“性能优化 -> 地图与进阶图表”往下学
