## 按场景选择入口

### 场景一：我想把现有 uni-app 微信小程序改成钉钉小程序

按顺序读：

1. **先看方案选型**
   `小程序/uni-app微信小程序迁移钉钉小程序-技术选型与分阶段改造方案`
   → 单仓改造 vs 复制 vs 重写，哪条路最合适，分几个阶段推进

2. **再看能力清单**
   `小程序/uni-app迁移钉钉小程序-能力速查与适配清单`
   → 哪些能直接复用，哪些必须适配，哪些不能机械替换

3. **搞清楚 ID 体系**
   `小程序/uni-app迁移钉钉时最容易搞混的各种id`
   → corpId / agentId / clientId / userId 分别是什么，别搞混

4. **接入编译链路**
   `小程序/uni-app接入钉钉小程序编译链路-配置与首次编译报错处理`
   → package.json 怎么配，首次编译报错 CE1000.01 怎么修

5. **wx 替换速查**
   `小程序/uni-app项目里最常见的wx替换清单_附代码模板`
   → wx.* 逐条对应改法，附代码模板

### 场景二：我想快速让项目在钉钉里跑起来（H5 微应用）

按顺序读：

1. **先看全流程**
   `H5微应用/uni-app项目接入钉钉H5微应用-从部署到登录跑通全流程`
   → 为什么优先走 H5 微应用，后台怎么配，H5 怎么部署，登录怎么实现

2. **深入登录实现**
   `H5微应用/钉钉双链路登录完整实现-JSAPI免登与OAuth授权`
   → SDK 懒加载、dd.ready 等待、state 防伪、hash 路由参数残留、App WebView 拦截

3. **避开常见误区**
   `H5微应用/钉钉H5微应用登录常见误区与官方规则核验`
   → requestAuthCode 需不需要先 dd.config，state 能不能省，回调字段名兼容

4. **接入扫码能力**
   `H5微应用/钉钉H5微应用-扫一扫JSAPI-前后端对接清单`
   → 钉钉原生扫码替代 uni.scanCode，前后端签名对接
