# 第三方登录 — 术语与场景速查

> 阅读本目录其他文档前，先读这篇。把容易混淆的词一次说清楚。

---

## 一、最容易混淆的三个场景

这三个场景名字相近，但运行环境完全不同，登录实现也不同。

### 1. H5 微应用（移动端）

**运行容器**：手机钉钉 / 企业微信客户端的内置浏览器（WebView）

**用户路径**：用户在手机上打开钉钉 → 工作台 → 点击应用图标 → 钉钉用内置浏览器加载你的网页

**登录方式**：

- 钉钉：加载 JSAPI SDK，等 `dd.ready`，调 `dd.runtime.permission.requestAuthCode({ corpId })` → 拿到 `authCode` → 发给后端换 token
- 企业微信：构造 OAuth URL（`snsapi_base` 静默授权）跳转 → 回调拿 `code` → 发给后端换 token

**特征**：用户完全无感知，打开即登录，不需要手动扫码或输密码。

**UA 特征**：`navigator.userAgent` 包含 `DingTalk`（钉钉）或 `wxwork`（企业微信）

---

### 2. PC 微应用（桌面端）

**运行容器**：电脑上的钉钉 / 企业微信桌面客户端的内置浏览器（WebView）

**用户路径**：用户在电脑上打开钉钉客户端 → 工作台 → 点击应用图标 → 钉钉桌面客户端用内置浏览器加载你的网页

**登录方式**：和 H5 微应用完全一样，也是免登（`requestAuthCode`），只是运行在 PC 客户端的 WebView 里而不是手机上。

**特征**：同样是打开即登录，无需扫码。

**和 H5 微应用的区别**：只是设备不同（手机 vs 电脑），登录技术实现几乎一致，差异主要在屏幕尺寸适配和部分 JSAPI 能力上。

---

### 3. 普通浏览器扫码登录（PC 管理后台的主要场景）

**运行容器**：系统浏览器（Chrome / Edge / Safari 等），和钉钉 / 企业微信客户端没有关系

**用户路径**：用户用浏览器打开学校 Web 系统 → 登录页展示一个二维码 → 用手机钉钉扫码 → 扫码成功后浏览器完成登录

**登录方式**：OAuth 授权码流程

1. 前端构造授权 URL，嵌入二维码组件（钉钉：`DTFrameLogin`；企业微信：`ww.createWWLoginPanel` 或构造登录链接）
2. 用户扫码，钉钉 / 企业微信服务器回调 `redirect_uri`，URL 带 `code`
3. 前端回调页解析 `code`，发给后端换 token

**特征**：需要用户主动扫码，有明显的"扫码"交互步骤。

## 二、三者对照表

| 维度         | H5 微应用（手机）        | PC 微应用（桌面）        | 普通浏览器扫码          |
| ------------ | ------------------------ | ------------------------ | ----------------------- |
| 运行容器     | 手机钉钉/企微内置浏览器  | 桌面钉钉/企微内置浏览器  | 系统浏览器              |
| 用户设备     | 手机                     | 电脑                     | 电脑（扫码用手机）      |
| 登录方式     | 免登（无感知）           | 免登（无感知）           | OAuth 扫码              |
| 需要扫码     | 否                       | 否                       | 是                      |
| UA 特征      | 含 `DingTalk`/`wxwork`   | 含 `DingTalk`/`wxwork`   | 普通浏览器 UA           |
| 钉钉登录 API | `requestAuthCode`        | `requestAuthCode`        | `DTFrameLogin` 组件     |
| 企微登录 API | OAuth `snsapi_base` 跳转 | OAuth `snsapi_base` 跳转 | `ww.createWWLoginPanel` |

## 三、"微应用"这个词是什么意思

"微应用"是钉钉 / 企业微信开放平台的官方叫法，指的是**嵌入在钉钉 / 企业微信客户端工作台里的 Web 应用**。

本质上就是一个普通网页，只不过：

- 它被注册到了钉钉 / 企业微信开放平台
- 用户从客户端工作台入口打开，运行在客户端内置浏览器里
- 可以调用平台提供的 JSAPI（扫码、选人、免登等）

所以"H5 微应用"= 在手机客户端里跑的网页，"PC 微应用"= 在桌面客户端里跑的网页。

## 四、OAuth2.0 授权码流程（通用原理）

无论钉钉还是企业微信，扫码登录的底层都是 OAuth2.0 授权码流程：

```
1. 前端构造授权链接（含 appid、redirect_uri、state 等参数）
2. 跳转到第三方授权页面（或展示二维码）
3. 用户同意授权（扫码 / 静默自动同意）
4. 回调 redirect_uri，URL 携带 code 参数
5. 前端把 code 发给后端
6. 后端用 code + appSecret 换取 access_token
7. 后端用 access_token 调 API 获取用户信息
8. 后端验证/创建本地用户，返回业务 token
```

**关键点**：`code` 是一次性的，有效期通常 5 分钟，用完即废。`state` 必须校验，防 CSRF 攻击。

## 五、常见 ID 速查

| ID 名称  | 含义                       | 钉钉         | 企业微信                |
| -------- | -------------------------- | ------------ | ----------------------- |
| corpId   | 企业唯一标识               | 字母数字混合 | `ww` 开头               |
| agentId  | 应用在企业内的标识         | 数字         | 纯数字                  |
| clientId | 应用的 OAuth 客户端 ID     | 即 appKey    | 即 corpId（网页授权时） |
| userId   | 用户在企业内的唯一标识     | 字符串       | 字符串（小写）          |
| unionId  | 用户跨企业的唯一标识       | 有           | 有                      |
| openId   | 用户在某个应用下的唯一标识 | 有           | 有                      |

## 六、常见登录场景分类（通用）

接入第三方登录时，一个项目通常会同时涉及以下几种场景，每种场景的实现方式不同，需要分别处理：

| 场景 | 触发方式 | 登录实现 | 典型入口页 |
| ---- | -------- | -------- | ---------- |
| 移动端客户端内免登（H5微应用） | 用户从手机钉钉/企微工作台打开 | 钉钉：`requestAuthCode`；企微：OAuth `snsapi_base` | 独立登录中转页 |
| 桌面端客户端内免登（PC微应用） | 用户从桌面钉钉/企微工作台打开 | 同上，实现一致 | 独立登录中转页 |
| PC 浏览器扫码登录 | 用户在系统浏览器打开管理后台 | 钉钉：`DTFrameLogin`；企微：`ww.createWWLoginPanel` | 登录页内嵌二维码组件 |
| App 原生端 OAuth | 用户在 App 内触发第三方登录 | 弹出 WebView 加载授权页，拦截回调 URL 提取 code | 独立登录中转页 |
| 账号绑定 | 已登录用户主动绑定第三方账号 | 同 OAuth 流程，但绑定而非登录 | 个人中心绑定入口 |

> 实际项目中，同一套代码可能需要同时支持多个场景，建议按场景拆分独立页面，不要揉进一个"万能登录页"。

## 补充场景（UniApp H5 微应用）

### 场景：钉钉 H5 微应用（移动端）

- **运行环境**：手机钉钉内置浏览器
- **技术栈**：UniApp 编译 H5，嵌入钉钉微应用容器
- **登录方式**：钉钉 JSAPI 免登（`dd.runtime.permission.requestAuthCode({ corpId })`）→ 拿到 `authCode` → 后端换取用户信息
- **关键文档**：`01-钉钉/01-H5微应用/01-知识文档/`

### 场景：企业微信 H5 微应用（移动端）

- **运行环境**：手机企业微信内置浏览器
- **技术栈**：UniApp 编译 H5，嵌入企业微信微应用容器
- **登录方式**：企业微信 OAuth 静默授权（`snsapi_base` 跳转）→ 回调拿 `code` → 后端换取用户信息
- **注意**：`wx.config` + `agentConfig` 是 JSSDK 能力初始化（扫码、选人等），和登录是两件独立的事，不要混淆
- **关键文档**：`02-企业微信/01-H5微应用/01-知识文档/`
