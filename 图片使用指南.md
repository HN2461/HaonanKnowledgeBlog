# 🖼️ 图片使用指南

## 📁 图片存储位置

所有图片都应该存放在 `public/images/` 目录下：

```
public/
├── images/
│   ├── javascript/          # JavaScript 相关图片
│   ├── vue/                # Vue 相关图片
│   ├── react/              # React 相关图片
│   ├── css/                # CSS 相关图片
│   ├── screenshots/        # 截图
│   └── common/             # 通用图片
```

## 📝 图片引用方法

在 Markdown 文件中引用图片：

### 1. 相对路径引用（推荐）
```markdown
![JavaScript Logo](/images/javascript/js-logo.png)
![Vue 组件图](/images/vue/component-lifecycle.png)
```

### 2. 带尺寸的图片
```markdown
![图片描述](/images/common/example.png){width=300 height=200}
```

### 3. 带标题的图片
```markdown
![Vue 生命周期](/images/vue/lifecycle.png)

*图：Vue 组件生命周期钩子*
```

## 🎨 图片类型建议

### 1. 截图类
- **格式**: PNG 或 WebP
- **用途**: 代码截图、界面截图
- **命名**: 使用描述性名称

### 2. 图标类
- **格式**: SVG 或 PNG
- **用途**: 技术图标、Logo
- **命名**: 简洁明了

### 3. 示例图片
- **格式**: JPG 或 WebP
- **用途**: 效果展示、案例图片
- **命名**: 包含技术栈名称

## 📏 图片尺寸建议

| 用途 | 宽度 | 高度 | 格式 |
|------|------|------|------|
| 代码截图 | 800px | 自动 | PNG |
| 图标 | 64px | 64px | SVG |
| 效果展示 | 600px | 自动 | JPG |
| 架构图 | 800px | 600px | PNG |

## ⚠️ 注意事项

1. **文件命名**: 使用英文和连字符，避免中文和空格
2. **文件大小**: 单张图片建议不超过 500KB
3. **格式选择**: 
   - 截图用 PNG（保持清晰度）
   - 照片用 JPG（较小体积）
   - 图标用 SVG（可缩放）
4. **路径规范**: 始终以 `/images/` 开头

## 🔄 图片更新流程

1. 将图片复制到 `public/images/` 对应目录
2. 在 Markdown 中引用图片
3. 刷新浏览器查看效果（无需重新生成索引）
