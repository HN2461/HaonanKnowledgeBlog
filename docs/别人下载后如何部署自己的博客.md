# 如何部署自己的博客到 GitHub Pages（详细版）

> 📖 **文档说明**：本文档是详细版部署指南，适合完全没有接触过 GitHub Pages 的新手。如果你已经熟悉基本流程，可以查看 [README.md](../README.md) 的快速版本。

---

## 🎯 最终效果

部署成功后，你将拥有一个完全属于自己的在线博客网站，地址格式为：

```
https://你的GitHub用户名.github.io/你的仓库名/
```

**示例**：
- 如果你的 GitHub 用户名是 `zhangsan`
- 仓库名是 `my-blog`
- 那么网站地址就是：`https://zhangsan.github.io/my-blog/`

在开始之前，请确保你已经准备好以下内容：

### 必需项

| 项目 | 说明 | 下载地址 |
|------|------|----------|
| ✅ GitHub 账号 | 用于托管代码和网站 | [注册地址](https://github.com/signup) |
| ✅ Git | 版本控制工具 | [下载地址](https://git-scm.com/downloads) |
| ✅ Node.js | JavaScript 运行环境（建议 v20+） | [下载地址](https://nodejs.org/) |

```bash
# 应该输出类似：git version 2.40.0

# 检查 Node.js 版本
node --version
# 应该输出类似：v20.11.0

npm --version
# 应该输出类似：10.2.4
```

如果命令执行失败，说明对应工具未安装，请先安装。

## 🏗️ 第一步：在 GitHub 创建你的仓库

### 操作步骤

1. **登录 GitHub**  
   访问 [github.com](https://github.com) 并登录你的账号

2. **创建新仓库**  
   点击右上角的 `+` 号，选择 `New repository`

3. **填写仓库信息**
   - **Repository name**（仓库名）：建议用英文，例如 `my-blog`、`tech-notes` 等
   - **Description**（描述）：可选，例如"我的个人技术博客"
   - **Public/Private**：选择 `Public`（公开）
   - ⚠️ **不要勾选** "Add a README file"、"Add .gitignore"、"Choose a license"

4. **点击 `Create repository`**

### 记录重要信息

创建完成后，请记住以下两个信息（后面会用到）：

- **你的 GitHub 用户名**：例如 `zhangsan`
- **你的仓库名**：例如 `my-blog`

---

## 📥 第二步：获取项目代码

有两种方式获取项目代码，选择其中一种即可：

---

### 方式 A：Fork 本项目（⭐ 推荐）

**优点**：可以随时同步原项目的更新

#### 操作步骤

1. **Fork 仓库**  
   在本项目的 GitHub 页面，点击右上角的 `Fork` 按钮

2. **Fork 到你的账号**  
   在弹出页面中，确认信息后点击 `Create fork`

3. **克隆到本地**  
   打开命令行，执行：

```bash
# 克隆你 Fork 的仓库（注意替换用户名）
git clone https://github.com/你的GitHub用户名/HaonanKnowledgeBlog.git

# 进入项目目录
cd HaonanKnowledgeBlog
```

**示例**：
```bash
git clone https://github.com/zhangsan/HaonanKnowledgeBlog.git
cd HaonanKnowledgeBlog
```

---

### 方式 B：下载 ZIP 后上传到新仓库

**优点**：不需要保持与原项目的关联

#### 操作步骤

1. **下载项目**  
   在本项目页面，点击 `Code` -> `Download ZIP`

2. **解压文件**  
   将下载的 ZIP 文件解压到你想要的位置

3. **初始化 Git 仓库**  
   打开命令行，进入解压后的项目目录，执行：

```bash
# 初始化 Git 仓库
git init

# 添加所有文件
git add .

# 创建第一次提交
git commit -m "初始化博客项目"

# 设置主分支名为 main
git branch -M main

# 关联远程仓库（注意替换用户名和仓库名）
git remote add origin https://github.com/你的GitHub用户名/你的仓库名.git

# 推送到 GitHub
git push -u origin main
```

**示例**：
```bash
git init
git add .
git commit -m "初始化博客项目"
git branch -M main
git remote add origin https://github.com/zhangsan/my-blog.git
git push -u origin main
```

> 💡 **提示**：如果推送时要求输入用户名和密码，建议配置 [SSH 密钥](https://docs.github.com/zh/authentication/connecting-to-github-with-ssh) 或使用 [Personal Access Token](https://docs.github.com/zh/authentication/keeping-your-account-and-data-secure/creating-a-personal-access-token)。

## ⚙️ 第三步：修改配置（⚠️ 最重要的一步）

> ⚠️ **警告**：这一步是最容易出错的地方！如果不修改，部署后会出现白屏或资源 404 错误。

### 需要修改的文件

打开项目根目录下的 `vite.config.js` 文件。

### 修改前（原始内容）

```js
export default defineConfig({
  base: '/HaonanKnowledgeBlog/',  // 👈 这是原作者的仓库名
  // ...其他配置
})
```

### 修改后（你的配置）

```js
export default defineConfig({
  base: '/你的仓库名/',  // 👈 改成你在第一步创建的仓库名
  // ...其他配置
})
```

### 具体示例

如果你的仓库名是 `my-blog`，应该改成：

```js
export default defineConfig({
  base: '/my-blog/',
  // ...其他配置
})
```

### 注意事项

- ✅ 必须以 `/` 开头和结尾，例如：`/my-blog/`
- ❌ 不能写成：`my-blog`、`/my-blog`、`my-blog/`
- ✅ 仓库名区分大小写，必须与 GitHub 上的仓库名完全一致

## 🚀 第五步：配置 GitHub Actions 自动部署

GitHub Actions 可以在你每次推送代码时，自动构建并部署网站。

### 创建配置文件

在项目根目录创建文件夹和文件：

```
.github/
└── workflows/
    └── deploy-pages.yml
```

### 文件内容

将以下内容复制到 `deploy-pages.yml` 文件中：

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: ['main']
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: 'pages'
  cancel-in-progress: true

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Generate notes index
        run: npm run generate:index

      - name: Build
        run: npm run build

      - name: Setup Pages
        uses: actions/configure-pages@v5

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: dist

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
      - name: Deploy to Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

### 配置说明

- `on.push.branches: ['main']`：当推送到 `main` 分支时触发
- `workflow_dispatch`：允许手动触发部署
- `npm ci`：安装依赖（比 `npm install` 更快更稳定）
- `npm run generate:index`：生成笔记索引
- `npm run build`：构建生产版本

## 🔧 第六步：在 GitHub 启用 Pages

### 操作步骤

1. **进入仓库设置**  
   在你的 GitHub 仓库页面，点击 `Settings`（设置）

2. **找到 Pages 设置**  
   在左侧菜单中找到 `Pages`

3. **配置部署源**  
   在 `Build and deployment` 部分：
   - **Source**（源）：选择 **`GitHub Actions`**
   - 不要选择 "Deploy from a branch"

4. **保存**  
   配置会自动保存

### 配置截图说明

你应该看到类似这样的界面：

```
Build and deployment
├─ Source: GitHub Actions  ← 选择这个
└─ Custom domain: (可选)
```

## 📝 第八步：后续如何更新网站

部署成功后，你可以随时更新博客内容。

### 添加新文章

1. **创建 Markdown 文件**  
   在 `public/notes/` 目录下创建或编辑 `.md` 文件

2. **添加 frontmatter**（推荐）  
   在文件开头添加元信息：

```markdown
---
title: 我的第一篇文章
date: 2026-05-12
category: JavaScript
tags:
  - 基础
  - 入门
description: 这是一篇关于 JavaScript 基础的文章
---

# 我的第一篇文章

这里是正文内容...
```

3. **重新生成索引**  
   在命令行执行：

```bash
npm run generate:index
```

4. **提交并推送**

```bash
git add .
git commit -m "新增文章：我的第一篇文章"
git push origin main
```

### 自动部署

推送后，GitHub Actions 会自动：
1. 安装依赖
2. 生成索引
3. 构建网站
4. 部署到 GitHub Pages

整个过程通常需要 1-3 分钟，完成后你的网站就会更新。

## ❓ 常见问题与解决方案

### Q1: 部署后网站白屏怎么办？

**原因**：99% 是因为 `vite.config.js` 中的 `base` 配置不正确。

**解决方法**：

1. 打开 `vite.config.js`
2. 检查 `base` 配置是否正确：
   - ✅ 正确：`base: '/my-blog/'`（你的仓库名）
   - ❌ 错误：`base: '/HaonanKnowledgeBlog/'`（原作者的仓库名）
3. 修改后重新提交并推送

```bash
git add vite.config.js
git commit -m "修复 base 配置"
git push origin main
```

---

### Q2: 新文章不显示怎么办？

**原因**：添加文章后忘记执行 `npm run generate:index`。

**解决方法**：

```bash
# 重新生成索引
npm run generate:index

# 提交更新
git add .
git commit -m "更新索引"
git push origin main
```

---

### Q3: GitHub Actions 部署失败怎么办？

**排查步骤**：

1. **查看错误日志**  
   进入仓库的 `Actions` 页面，点击失败的任务，查看详细日志

2. **常见错误及解决方法**

| 错误信息 | 原因 | 解决方法 |
|---------|------|---------|
| `npm ci` 失败 | `package-lock.json` 不匹配 | 本地执行 `npm install` 后重新提交 |
| `npm run build` 失败 | 代码语法错误 | 本地执行 `npm run build` 查看错误 |
| `Permission denied` | Pages 权限未配置 | 检查仓库 Settings -> Pages 配置 |
| `404 on deployment` | `base` 配置错误 | 检查 `vite.config.js` 中的 `base` |

3. **重新触发部署**  
   修复问题后，可以手动触发部署：
   - 进入 `Actions` 页面
   - 选择 "Deploy to GitHub Pages" 工作流
   - 点击 `Run workflow` -> `Run workflow`

---

### Q4: 如何自定义域名？

**步骤**：

1. **在 GitHub 配置**  
   进入仓库 `Settings` -> `Pages` -> `Custom domain`，输入你的域名（例如 `blog.example.com`）

2. **在域名提供商配置 DNS**  
   添加 CNAME 记录：
   - **类型**：CNAME
   - **名称**：`blog`（或 `@` 用于根域名）
   - **值**：`你的GitHub用户名.github.io`

3. **等待 DNS 生效**  
   通常需要几分钟到几小时

4. **启用 HTTPS**  
   DNS 生效后，在 GitHub Pages 设置中勾选 `Enforce HTTPS`

---

### Q5: 如何同步原项目的更新？

**前提**：你使用了 Fork 方式获取项目

**步骤**：

```bash
# 添加原项目为上游仓库（只需执行一次）
git remote add upstream https://github.com/原作者用户名/HaonanKnowledgeBlog.git

# 拉取上游更新
git fetch upstream

# 合并到你的 main 分支
git merge upstream/main

# 推送到你的仓库
git push origin main
```

```bash
# 构建生产版本
npm run build

# 预览构建结果
npm run preview
```

---

### Q7: 部署成功但样式丢失怎么办？

**原因**：资源路径不正确

**检查项**：

1. `vite.config.js` 中的 `base` 配置是否正确
2. 浏览器开发者工具（F12）-> Network 标签，查看哪些资源 404
3. 确保所有资源引用都使用相对路径或正确的 `base` 路径

---

### Q8: 如何删除已部署的网站？

**步骤**：

1. 进入仓库 `Settings` -> `Pages`
2. 在 `Source` 下拉菜单中选择 `None`
3. 或者直接删除整个仓库
